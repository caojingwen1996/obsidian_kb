# 冰冰小美体系三要素独立技能设计

## 1. 背景

当前项目把体系三要素的具体检查项混在 `risk-identification`、`equity-research` 和交易复盘技能中，导致“三要素分析”没有可直接调用的独立入口，也容易在多个技能之间产生规则漂移。

本次把三要素分析拆成独立项目技能，并重新明确它与风险识别的边界。

## 2. 目标

新增 `bbxm-three-factor-analysis`，用于直接分析市场、指数、板块或个股的：

1. 竞争格局与比较优势；
2. 流动性辩证分析；
3. 情绪位置变化；
4. 三要素综合状态；
5. `buy / sell / reduce / wait / observe / review` 行动结论。

新技能应能独立触发，也能由 `bbxm-expert` 路由调用。

## 3. 非目标

新技能不负责：

- 判断风险正在增强还是减弱；
- 代替完整的公司基本面、财务与估值研究；
- 代替新闻、公告或政策的信息过滤；
- 代替交割单和仓位行为复盘；
- 因单一利好、单日涨跌或主观叙事直接给出交易结论。

## 4. 技能边界

### 4.1 `bbxm-three-factor-analysis`

只负责体系三要素及其综合行动映射：

```text
竞争格局
→ 流动性辩证分析
→ 情绪位置变化
→ 三要素综合状态
→ buy / sell / reduce / wait / observe / review
```

### 4.2 `risk-identification`

只负责风险方向变化：

```text
风险增强
风险持平
风险减弱
风险重新增强
证据不足
```

`risk-identification` 不判断买入窗口，不调用三要素技能，不输出三要素结论，也不给出具体买卖动作。

### 4.3 两者关系

两个技能平行、互不调用。用户要求综合分析时，由 `bbxm-expert` 分别路由并汇总：

```text
风险增强或减弱 → risk-identification
三要素状态与行动 → bbxm-three-factor-analysis
```

## 5. 触发与路由

以下请求直接触发 `bbxm-three-factor-analysis`：

- “用冰冰小美三要素分析某个市场、指数、板块或个股”；
- “竞争格局、流动性、情绪位置现在是否有利”；
- “三要素是否共振”；
- “按三要素判断应当买、卖、减仓、等待、观察还是复盘”。

相邻请求继续按以下路径处理：

| 用户主动作 | 技能 |
|---|---|
| 判断风险增强、持平、减弱或重新增强 | `risk-identification` |
| 过滤新闻、公告、政策或研报 | `information-filter-flow` |
| 复盘交割单、买卖记录或仓位行为 | `trade-ticket-review` |
| 完整公司研究、估值、DCF 或目标价 | `equity-research` |
| 直接分析体系三要素 | `bbxm-three-factor-analysis` |

## 6. 输入契约

执行前明确：

- 分析对象及市场；
- 对象层级：市场、指数、板块或个股；
- 分析截止时间；
- 用户的核心决策问题；
- 已提供材料与仍需核验的数据。

涉及当前市场状态时必须核验最新可得数据。无法取得的数据写明“未获取到”、检查范围及其对置信度的影响，不以推测补空。

## 7. 分析流程

### 7.1 检索体系依据

先检索当前 wiki 中与三要素、分析对象和相关产业方向有关的正式页面与来源。体系术语、判断顺序和证据要求以当前 wiki 为依据。

### 7.2 竞争格局与比较优势

至少检查：

- 是否符合安全与发展方向；
- 产业链利润流向与议价权；
- 国内外竞争位置和替代关系；
- 供给扩张、出清或格局修复；
- 比较优势是否可持续；
- 支撑结论的证据及反证。

### 7.3 流动性辩证分析

至少检查：

- 宏观、中观和微观流动性；
- 资金来自真实增量、机构配置、ETF、融资盘还是短期接力；
- 承接是否从少数标的扩散到板块内部；
- 股票供给、减持、融资和解禁压力；
- 流动性是否可持续以及退出路径是否清晰；
- 支撑结论的证据及反证。

### 7.4 情绪位置变化

至少检查：

- 当前处于冰点、试错、扩散、拥挤、退潮还是修复；
- 市场广度、接力、赚钱效应和亏钱效应；
- 利好与利空出现后的价格反应；
- 情绪标、跟风标和套利行为是否分化；
- 情绪位置正在改善、钝化还是恶化；
- 支撑结论的证据及反证。

### 7.5 综合判断

每个要素使用统一状态：

```text
有利 / 中性 / 不利 / 证据不足
```

综合结论使用：

```text
同步有利
部分有利
尚未形成有利共振
同步不利
证据不足
```

不得用一个强要素机械抵消另外两个不利要素。结论必须说明主导要素、拖累要素、下一验证节点和失效条件。

### 7.6 行动映射

行动语言限定为：

```text
buy / sell / reduce / wait / observe / review
```

行动必须由三要素证据支持，并区分新增现金与已有持仓。证据不足时使用 `observe` 或 `review`，不得伪装成确定性买卖建议。

## 8. 输出结构

新技能使用同目录 `template.md`，至少包含：

1. 对象、层级、截止时间与核心问题；
2. 一句话结论；
3. 三要素状态总表；
4. 竞争格局证据、反证与失效条件；
5. 流动性证据、反证与失效条件；
6. 情绪位置证据、反证与失效条件；
7. 综合状态与主导矛盾；
8. 新增现金与已有持仓的行动映射；
9. 下一验证节点；
10. 来源、数据缺口与置信度。

## 10. 验证方案

### 10.1 静态验证

自动检查：

- 新技能的 frontmatter、目录和配套文件完整；
- `bbxm-expert` 能把直接三要素请求路由到新技能；
- `risk-identification` 不再包含三要素分析、买入窗口、具体买卖动作或独立行动建议；
- `equity-research` 和交易复盘引用新技能而不是复制规则；
- 中文文件不存在典型乱码字符。

### 10.2 场景测试

至少覆盖：

1. “用三要素分析某板块”应直接进入新技能；
2. “判断某股风险是在增强还是减弱”只进入风险识别；
3. “完整研究某公司并给出估值和三要素结论”由完整研报调用新技能；
4. “复盘这笔买入是否符合三要素”仍以交易复盘为主，并引用新技能；
5. 数据不足时输出缺口和低置信度，不编造结论。

由于当前会话未获授权使用子代理，场景验证采用静态断言和主代理逐例检查，不启动并行代理。

## 11. 验收标准

- 用户可以直接调用 `bbxm-three-factor-analysis`；
- 三要素分析有唯一规则来源和统一模板；
- `risk-identification` 只判断风险增强或减弱，不再承担三要素与交易动作；
- 两个技能互不调用；
- `bbxm-expert` 能正确处理单一请求和综合请求；
- 相邻技能不再维护重复三要素规则；
- 自动验证与场景测试通过；
- 只提交本任务相关文件。
