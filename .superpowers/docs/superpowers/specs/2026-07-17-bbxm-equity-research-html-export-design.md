# BBXM 个股研报 HTML 导出设计

## 目标

统一 `bbxm-equity-research` 的 Markdown 与 HTML 交付契约：正文必须严格采用当前 `template.md` 的完整 16 模块，HTML 必须由同一份 Markdown 自动生成，避免手工维护两套结构。

## 范围

1. 将航天电子机构级决策研报重排为当前模板的完整 16 模块，保留已有研究事实、估值数据、产业位置和来源边界。
2. 以电科蓝天 HTML 的版式作为唯一视觉基准：渐变封面、判断徽章、四张关键指标卡、左侧目录、右侧正文。
3. 为 `bbxm-equity-research` 增加必需的 HTML 导出步骤和可复用渲染脚本。
4. 重新生成航天电子 HTML，并用同一渲染器重新生成电科蓝天 HTML，消除 `??`、`12?24` 等编码缺陷。

## 内容契约

- Markdown 的一级标题、16 个二级模块名称及顺序以 `.agents/skills/bbxm-equity-research/template.md` 为唯一来源。
- 不为填满模板而编造数据；缺失信息进入“未获取到的数据与研究局限”。
- 估值、冰冰小美框架判断、最终结论和监控指标必须相互一致。
- 航天电子的产业位置保留为：核心配套平台，覆盖上游关键电子部件、中游分系统和地面测控；商业航天纯度中低、产业受益确定性中高、利润与现金流弹性待验证。

## HTML 契约

- HTML 只从 Markdown 生成，不维护独立研究正文。
- 页面结构固定为：封面区、判断徽章、四张指标卡、左侧吸顶目录、右侧正文、打印样式。
- 渲染器读取 UTF-8 文件，不通过 PowerShell 管道传递含中文的脚本文本。
- 支持 Markdown 表格、列表、引用、链接和 Obsidian 双链转换。
- 关键字段优先从“决策摘要”表提取；字段缺失时显示“未获取”，不得猜测。

## 技能工作流

在现有研究与质量检查步骤之后增加“生成并验证 HTML”：

1. 完成并校验 16 模块 Markdown。
2. 调用技能内渲染脚本生成同名 HTML。
3. 检查 16 个目录锚点、中文编码、关键指标、内部链接和打印布局。
4. Markdown 或 HTML 任一未通过质量门，不得声称交付完成。

## 验收标准

- 航天电子 Markdown 恰好包含模板规定的 16 个一级模块，名称和顺序完全一致。
- 航天电子与电科蓝天 HTML 使用同一渲染器和同一视觉模板。
- HTML 中不存在 `??`、`12?24`、替换字符 `�` 或未转换的 `[[...]]`。
- HTML 目录可跳转到全部 16 个模块。
- 技能说明明确将 HTML 作为默认必交付物，并给出可执行命令和质量门。
- 仅修改本任务相关文件，并在 `log.md` 记录维护动作。
