# 兜底

## 1. 一句话定义

兜底，是[[people/碧树西风|碧树西风]]风控语言中用于回答“止损失效后系统如何不死”的概念。它处理的是价格穿透、流动性枯竭、无法成交、平台兑付失败、对手盘消失、杠杆强平和生活现金流被抽干等极端情形。

兜底不是“有人替我保本”，而是执行者自己提前设计的隔离层和备用路径。

## 2. 核心内涵

兜底至少包含四层：

- 资金兜底：不拿全部身家进场，保留生活现金流和下一轮行动资本；
- 执行兜底：预设滑点、无法成交、分批退出或对冲路径；
- 结构兜底：审计底层资产、偿付来源、对手盘和平台机制；
- 心理兜底：单次失败不击穿家庭、生活和继续复盘的能力。

它和[[concepts/碧树西风-rule-风控穿透|风控穿透规则]]直接相连：风控不能停在止损线，而要继续穿透到成交、流动性、底层资产、偿付来源、对手盘、出金和平账。

## 3. 解决的问题

兜底主要防止“纸面风控”。

常见纸面风控包括：

1. 写了止损价格，但真实市场中无法成交；
2. 看到低收益率，就误以为风险低；
3. 看到保本、实物、平台信用，就误以为有安全背书；
4. 只计算账面亏损，不计算生活现金流和继续行动能力。

在碧树西风的体系中，真正的风控必须问到底：谁买单、谁偿付、谁接盘、能否退出，失败后还剩什么。

## 4. 使用检查表

| 问题 | 判断含义 |
|---|---|
| 止损失效时怎么办？ | 检查极端预案。 |
| 流动性消失时能否退出？ | 检查成交和对手盘。 |
| 平台或产品靠什么偿付？ | 检查底层资产和偿付来源。 |
| 单次失败是否会击穿生活？ | 检查资金隔离和生活现金流。 |
| 是否还有下一轮行动资本？ | 检查长期系统是否仍能运行。 |

## 5. 概念边界

兜底不是替代风险承担，也不是把所有投资都变成无风险。它只说明：当判断错误、止损失效或极端情形发生时，系统是否仍有剩余资本、退出路径和恢复能力。

如果所谓兜底只是外部承诺，没有底层资产、偿付来源、对手盘和退出机制支撑，就不是真正的兜底。
