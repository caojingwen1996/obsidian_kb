# 止损

## 1. 一句话定义

止损，是[[people/碧树西风|碧树西风]]风控语言中用于承认“判断可能错误”的事前机制：在进入一笔交易或配置前，先写清什么条件说明原假设失效，以及触发后如何退出、减仓、对冲或停止行动。

止损的重点不是亏了以后安慰自己，而是不给临场情绪改写系统的机会。

## 2. 核心内涵

止损至少包含三层：

- 条件层：到什么价格、发生什么事件、出现什么变量变化，说明原假设失效；
- 行为层：触发后如何卖出、减仓、暂停、对冲或进入复盘；
- 纪律层：触发后不再用“灵活”“长期看好”“再等等”临时改写规则。

所以，止损不是单独一个价格点，而是一套认错流程。它和[[concepts/碧树西风-rule-策略启动前执行|策略启动前执行规则]]直接相连：止损、止盈、加仓、减仓和极端预案都应该在系统启动前写清。

## 3. 解决的问题

止损主要解决的是“世界不按我想象运行时怎么办”。

如果没有止损，交易容易退化为：

1. 看对方向时归功于能力；
2. 看错方向时拒绝承认错误；
3. 浮亏扩大后再用长期叙事安慰自己；
4. 最后让一次错误击穿长期系统。

碧树西风反复强调，成熟策略不是相信某件事一定发生，而是先承认自己可能错，并提前写清错了怎么办。

## 4. 使用检查表

| 问题 | 判断含义 |
|---|---|
| 哪个条件说明原假设失效？ | 防止把亏损解释成正常波动。 |
| 触发后执行什么动作？ | 把认错落成退出或减仓。 |
| 止损能否真实成交？ | 检查流动性和滑点。 |
| 是否允许例外？ | 例外必须事前写入，而不是临场生成。 |
| 止损后如何复盘？ | 避免把退出变成情绪动作。 |

## 5. 概念边界

止损不是完整风控。它只能处理“规则触发后可以执行退出”的情形；如果价格跳空、流动性消失、对手盘消失或平台无法兑付，止损会失效。

因此，止损必须继续连接到[[concepts/碧树西风-concept-兜底|兜底]]和[[concepts/碧树西风-rule-风控穿透|风控穿透规则]]：止损线只是第一层，真正的风控还要问止损触发后是否有人接、能否成交、资金是否能退出。
