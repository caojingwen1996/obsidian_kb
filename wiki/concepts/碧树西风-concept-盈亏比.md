# 盈亏比

## 1. 一句话定义

盈亏比，是[[people/碧树西风|碧树西风]]风险控制语言中用于回答“这笔机会值不值得做”的概念：如果判断正确，可以得到多少；如果判断错误，需要损失多少；这两个量是否在同一口径下值得交换。

它不是“我想赚多少”，而是“赢了得到什么，输了失去什么”。如果只看方向正确率，不看损失边界和执行者承受力，方向看对也可能不是一笔好交易。

## 2. 核心内涵

盈亏比可以抽象为：

```text
盈亏比 = 潜在收益 T / 初始风险 R
```

这里的 `T` 不是幻想收益，而是基于资产空间、市场结构、退出路径和时间成本后仍可能兑现的收益；`R` 也不是纸面止损距离，而是仓位、滑点、成交、心理承受力和生活现金流共同决定的真实风险单位。

因此，盈亏比必须和[[concepts/碧树西风-rule-仓位承受力|仓位承受力规则]]一起看：如果 `R` 超过执行者真实承受力，再漂亮的 `T/R` 也可能在波动中失效。

## 3. 解决的问题

盈亏比主要防止三类错误：

1. 只问会不会涨，不问错了亏多少；
2. 只看目标收益，不看风险、成本、回撤和时间占用；
3. 把“我有把握”当成可以放大仓位或取消止损的理由。

在碧树西风的体系中，收益不能脱离风险和成本单独评价。[[sources/articles/2025-11-22-碧树西风：脱离风险，脱离成本谈收益，都是耍流氓|2025-11-22 文章]]强调的就是同一层意思：收益必须回到单位风险和单位成本下核算。

## 4. 使用检查表

| 问题 | 判断含义 |
|---|---|
| 上方收益空间在哪里？ | 确认 `T` 是否真实存在。 |
| 判断错时损失边界在哪里？ | 确认 `R` 是否可定义。 |
| 收益是否覆盖风险、成本和时间占用？ | 判断收益质量。 |
| 仓位亏掉后还能否继续执行系统？ | 检查风险单位是否超出承受力。 |
| 退出路径是否真实存在？ | 防止纸面收益无法兑现。 |

## 5. 概念边界

盈亏比不是保证盈利的公式，也不是所有交易都必须机械满足某个固定倍数。它只能说明一笔行动的收益和风险是否值得交换，不能替代[[concepts/碧树西风-concept-止损|止损]]、[[concepts/碧树西风-concept-兜底|兜底]]和[[concepts/碧树西风-rule-风控穿透|风控穿透规则]]。

如果一个资产没有真实退出路径，或损失边界无法定义，盈亏比很容易变成纸面安慰。
