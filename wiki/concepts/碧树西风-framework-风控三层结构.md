# 风控三层结构

## 0. 拆分说明

本页原名为“盈亏比、止损与兜底”，现按 Concept Page 命名规则拆分为三个独立概念页：

- [[concepts/碧树西风-concept-盈亏比|盈亏比]]：判断收益空间与风险单位是否值得交换。
- [[concepts/碧树西风-concept-止损|止损]]：把判断错误后的认错退出写成事前规则。
- [[concepts/碧树西风-concept-兜底|兜底]]：处理止损失效、流动性恶化和极端情形下的生存问题。

本页保留为 framework 页，负责说明三者如何组成一套连续风控结构。

## 1. 一句话定义

风控三层结构，是[[people/碧树西风|碧树西风]]风险控制语言中的一套基础框架：用[[concepts/碧树西风-concept-盈亏比|盈亏比]]判断这笔交易或配置是否值得做，用[[concepts/碧树西风-concept-止损|止损]]承认判断可能错误并预设退出，用[[concepts/碧树西风-concept-兜底|兜底]]确保止损失效或极端情形出现时，长期系统仍能活下来。

它回答的不是“这次会不会对”，而是：

- 如果判断对了，最多能得到什么；
- 如果判断错了，最多会损失什么；
- 判断错时是否真的能退出；
- 退出失败时是否还有对冲、隔离和生存资本；
- 这套规则是否能长期重复，而不是靠一次运气。

整理者归纳：本页把“盈亏比”“止损”“兜底”整理为一个概念组合，因为在碧树西风的体系中，它们并不是三个分散技巧，而是同一套长期生存逻辑的三个层次。

## 3. 概念要解决的问题

这个概念要解决的是：投资者很容易只讨论“会不会涨、能赚多少、方向有没有把握”，却跳过“错了怎么办、止损能否执行、单次错误会不会击穿生活和长期系统”。

它主要帮助判断：

- 一次交易或配置是否具备值得承担风险的收益空间；
- 止损是否是入场前写好的规则，而不是浮亏后的情绪动作；
- 仓位规模是否匹配真实承受力，而不是匹配贪婪；
- 纸面止损能否在真实市场里成交；
- 如果止损失效，是否有资金隔离、对冲、退出路径和生活兜底。

没有这个概念时，容易出现三类错误：

1. 只看方向正确率，不看收益和亏损是否对称；
2. 只写止损价格，不检查执行、流动性和对手盘；
3. 只追求单次高收益，不保留长期复利所需的生存资本。

## 4. 核心内涵

### 4.1 盈亏比：先问值不值得做

盈亏比衡量的是“判断正确时的可得收益”与“判断错误时的可承受损失”之间的比例。它不是主观愿望，而是交易结构。

可抽象为：

```text
盈亏比 = 潜在收益 T / 初始风险 R
```

如果 `T / R` 不够，方向看对也不一定值得做；如果 `R` 超过执行者真实承受力，盈亏比再好也可能在心理和资金层面失效。

在碧树西风的表达里，盈亏比的核心不是“想赚多少”，而是“赢了得到什么，输了损失什么”。这与[[concepts/碧树西风-策略案例-黄金策略|黄金策略案例]]中的 `T` 与 `R` 是同一组语言。

### 4.2 止损：把认错写成事前规则

止损不是亏损后临时安慰自己的动作，而是入场前写入策略的认错机制。

它至少包含三层意思：

- 价格或条件层：到什么位置、发生什么条件，说明原假设失效；
- 行为层：触发后如何退出、减仓、对冲或停止交易；
- 纪律层：触发后不再临场讲“灵活”，不把希望当成风控。

止损的价值在于承认世界不可测。即使方向判断很强，也必须预设“不涨怎么办”“判断错怎么办”“市场不按我想象运行怎么办”。

### 4.3 兜底：防止止损失效后击穿系统

兜底处理的是止损之外的失败情形：价格穿透、流动性枯竭、无法成交、平台兑付失败、对手盘消失、杠杆强平、生活现金流被抽干。

因此，兜底不是“还有人帮我保本”，而是执行者自己提前设计的隔离层：

- 不拿全部身家进场；
- 不让单次亏损破坏家庭和生活；
- 为无法成交或滑点过大预留资金和对冲方案；
- 不把“保本”“实物”“低收益”直接当成安全背书；
- 对底层资产、偿付来源和对手盘做穿透审计。

这也是[[concepts/碧树西风-rule-风控穿透|风控穿透规则]]要补上的部分：止损线只是第一层，真正的风控要穿透到成交、流动性、底层资产和最终买单者。

### 4.4 仓位：让数学规则落到真实承受力

盈亏比和止损只有在仓位合适时才有意义。

如果亏损金额远超执行者真实承受力，交易者会在策略内正常波动中提前变形；如果盈利金额诱惑太大，也可能诱发加杠杆、取消止损或临场干预。

因此，仓位不是附属变量，而是把风险单位 `R` 翻译成真实生活冲击的工具。[[concepts/碧树西风-rule-仓位承受力|仓位承受力规则]]负责回答：这笔交易即使亏掉，执行者是否还能工作、生活、复盘和继续下一局。

### 4.5 成本：收益必须回到单位风险和单位成本

收益不能脱离风险和成本独立评价。

同样的收益，如果承担了更大回撤、更长资金占用、更高精力消耗或更差流动性，就不是同一种收益。碧树西风在“脱离风险，脱离成本谈收益”一文中强调，真正要比较的是单位风险、单位成本下的收益质量。

这使本概念连接到[[concepts/碧树西风-rule-收益质量核算|收益质量核算规则]]：账面收益只是结果的一部分，风险、成本、回撤和可重复性才决定收益质量。

### 4.6 长期系统：追求每一次，不追求某一次

[[sources/articles/2023-07-13-碧树西风：很多赌徒都曾经赢过赌场，为什么还是输光光？|2023-07-13 文章]]把赌徒和赌场的差异压缩成目标函数差异：赌徒想赢某一次，赌场和成熟系统追求每一次都站在长期占优的位置。

放到本概念里，这意味着盈亏比、止损和兜底不是为了让某一次结果更漂亮，而是为了让反复行动后的分布仍然偏向可承受、可复盘、可迭代的长期正期望。如果一套系统会因为单次盈利而过度兴奋，或因为单次机会而取消仓位、止损、退出和兜底约束，它仍然更接近赌徒结构，而不是成熟系统结构。

因此，本概念的底层问题不是“这一次能不能赢”，而是：

- 这一次即使赢了，规则是否仍可复制；
- 这一次即使输了，系统是否还能继续运行；
- 每一次出手是否都处在收益、风险、仓位和退出已经被预先约束的位置；
- 单次结果是否被纳入复盘，而不是被用来证明纪律可以临时取消。

## 5. 使用检查表

在审计一笔交易、投资配置或金融产品时，可以按以下顺序检查：

| 问题 | 对应概念 |
|---|---|
| 这笔机会上方收益空间在哪里？ | 盈亏比 |
| 如果判断错，损失边界在哪里？ | 止损 |
| 目标收益是否至少覆盖风险、成本和时间占用？ | 盈亏比 / 成本 |
| 止损触发后能否真实成交？ | 风控穿透 |
| 仓位亏掉后是否仍能正常生活和继续复盘？ | 仓位承受力 |
| 如果流动性消失、价格跳空或平台失效怎么办？ | 兜底 |
| 是否有退出路径、对冲方案或资金隔离？ | 兜底 / 成熟策略风控结构 |
| 这套规则能否长期重复，而不是只靠一次判断？ | 长期系统 / 复盘修系统 |
| 这次动作是在改善长期分布，还是只是在追求某一次漂亮结果？ | 长期系统 |

## 6. 概念边界

这个概念不是：

- 保证盈利的公式；
- 鼓励所有机会都机械套 3:1；
- 只要设置止损线就完成风控；
- 用低仓位替代底层资产审计；
- 用“保本承诺”替代真实兜底；
- 只适用于短线交易、不适用于配置和产品审计。

它主要适用于：

- 有明确收益空间和损失边界的交易；
- 需要仓位、止损、退出和对冲规则的策略；
- 需要审计底层资产、偿付来源和对手盘的金融产品；
- 需要把个人投资行为控制在生活承受力之内的普通投资者。

它不适合被误用为：

- 单纯用来证明自己方向判断正确；
- 在没有退出路径的资产里强行计算纸面盈亏比；
- 在无法承担波动的仓位上安慰自己“长期看会赢”。

## 7. 常见误区

- 把“我很有把握”当成可以不设止损的理由。
- 把“盈亏比大于 3”理解为可以取消仓位管理。
- 把止损理解成临场情绪动作，而不是入场前就写好的规则。
- 只看单次收益空间，不看长期能否重复执行。
- 只看赚了多少钱，不看承担了多少风险、时间成本和精力成本。
- 以为策略数学上成立，就自动等于自己能坚持执行。
- 以为自己“这次看得很准”，就可以取消退出、仓位和对冲约束。
- 把“写下止损位”误当成已经解决了流动性与成交问题。
- 把“收益率很低”误当成风险天然很低，忽略真正的偿付来源。
- 把“保本”“实物”“随时成交”这些词直接当成安全背书，而不审计产品结构和对手盘机制。

## 8. 在当前知识库中的作用

本页是碧树西风风险控制体系里的基础概念页，主要承担四个作用：

1. 作为[[topics/碧树西风-概率化决策|概率化决策]]的风控语言：概率判断必须落成收益、亏损、仓位和退出。
2. 作为[[topics/碧树西风-投资系统建模|投资系统建模]]的策略端检查项：任何系统都要回答入场、止损、退出和兜底。
3. 支撑[[concepts/碧树西风-rule-成熟策略风控结构|成熟策略风控结构规则]]：成熟策略不是押注相信，而是预设错误处理。
4. 连接[[concepts/碧树西风-rule-风控穿透|风控穿透规则]]和[[concepts/碧树西风-rule-仓位承受力|仓位承受力规则]]：一个负责外部执行可行性，一个负责内部承受力。
