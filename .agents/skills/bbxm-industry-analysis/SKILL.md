# 产业分析模型 Skill

## 1. 目标

本 Skill 用于将政策、规划、投资、产能、产量、订单、库存、价格、利润和资本市场信息，组织为一套完整的产业分析结论。

核心任务：

1. 判断产业是否符合长期时代主线；
2. 识别产业链价值分布、利润池和关键瓶颈；
3. 判断产业所处的发展阶段与景气周期；
4. 区分真实产业增量、阶段性题材和资本市场泡沫；
5. 判断资源、订单和利润将向哪些产业环节集中；
6. 建立阶段转换条件、反证条件和持续监测指标；
7. 为后续企业分析提供候选环节与候选公司范围。

本 Skill 聚焦产业层面。单家公司战略、组织、财务、估值和投资决策，应进入独立的企业分析流程。

## 3. 适用场景

出现以下需求时使用本 Skill：

- 分析某个产业、赛道、行业或产业主题；
- 解读行业增加值、固定资产投资、产量、价格、库存或利润数据；
- 判断产业处于技术验证、扩产、量产、泡沫、出清或再成长阶段；
- 拆解产业链上下游、利润池、瓶颈和增量来源；
- 判断产业规划如何传导到投资、产能、订单和利润；
- 识别真正承载政策资源、资本开支和产业订单的环节；
- 建立产业日报、周报、专题报告或动态监测体系；
- 对多个产业进行横向比较；
- 将宏观政策或宏观数据映射到具体产业。

以下需求应转入其他流程：

- 单家公司完整基本面研究；
- 管理层、组织能力和治理结构分析；
- 单家公司财务建模和估值；
- 个股买卖建议；
- 纯宏观分析；
- 只查询单一数据且不需要形成产业判断。

## 4. 输入要求

### 4.1 必要输入

尽量明确：

- 产业名称；
- 分析目的；
- 时间范围；
- 地域范围；
- 用户最关注的问题；
- 是否需要资本市场视角；
- 是否需要筛选候选产业环节。

### 4.2 可选输入

可使用：

- 政策与规划文件；
- 行业增加值和固定资产投资；
- 产量、销量、订单、库存、价格；
- 行业收入、利润、现金流；
- 产业链资料；
- 主要参与者和重大项目；
- 用户已有判断；
- 知识库中的相关概念文件。

输入不完整时，使用可验证的公开资料补全，并明确列出仍待验证的信息。

## 5. 研究原则

### 5.1 先定义统计口径

任何数据进入分析前，先确认：

- 指标定义；
- 统计范围；
- 包含的子行业；
- 指标反映产量、收入、利润、价格还是投资；
- 是否存在低基数；
- 是否存在项目集中交付；
- 是否能够代表整个产业。

不得把宽口径行业数据直接归因于单一子行业。

### 5.2 先产业、后环节、再公司

固定分析顺序：

```text
时代主线
↓
产业规划
↓
产业链结构
↓
约束条件
↓
周期与节奏
↓
竞争格局
↓
筛选高价值环节
↓
形成候选公司范围
```

企业数据用于验证产业结构、供需、扩产和竞争格局，不在本 Skill 中展开完整公司估值。

### 5.3 每个结论必须形成证据链

关键判断至少应包含：

- 当前判断；
- 核心证据；
- 传导机制；
- 下一阶段条件；
- 主要风险；
- 反证指标；
- 置信度。

### 5.4 区分信息性质

明确标识：

- **事实**：有官方数据、正式公告或权威一手资料；
- **推断**：由多项事实推导；
- **假设**：情景分析所使用的前提；
- **待验证**：公开证据不足。

### 5.5 证据优先级

依次优先使用：

1. 国务院、国家统计部门、部委、监管机构；
2. 行业主管部门和行业协会；
3. 公司公告、年报、招股书、交易所问询；
4. 招标采购、项目审批、环评和建设信息；
5. 权威研究机构；
6. 主流媒体；
7. 市场观点和二手资料。

涉及最新政策、规划、订单、产能、价格和当前主体时，必须核验时效性。

### 5.6 防止概念映射过度

不得直接建立以下等式：

- 国家支持产业 = 所有相关公司受益；
- 行业产量增长 = 行业利润增长；
- 集团承担项目 = 订单进入旗下某家上市公司；
- 企业具备产品 = 已获得批量订单；
- 获得订单 = 已确认收入；
- 收入增长 = 利润和现金流同步改善；
- 产业空间大 = 当前估值合理。

## 6. 标准分析流程

# 模块一：战略方向

## 6.1 时代主线

分析：

- 国家战略；
- 全球竞争与地缘格局；
- 技术革命；
- 资源、能源和供应链安全；
- 人口与社会需求；
- 国产替代；
- 出口与全球化；
- 新质生产力。

回答：

- 国家和时代为什么需要该产业；
- 资源是否可能长期流入；
- 产业属于战略资源、支柱产业或新兴产业中的哪一类；
- 当前机会是长期方向还是阶段性政策主题。

## 6.2 产业规划与发展目标

分析：

- 国家级、部委和地方规划；
- 重大工程与产业集群；
- 建设目标和量化指标；
- 时间表；
- 财政、信贷、产业基金、税收、土地、牌照、标准和政府采购工具。

判断规划所处位置：

```text
方向提出
→ 规划制定
→ 项目审批
→ 资金落实
→ 项目开工
→ 设备采购
→ 产能投放
→ 运营与应用
```

回答：

- 规划目标是什么；
- 目前完成到哪一步；
- 是否已有资金和实际项目支撑；
- 规划与实际投资是否存在偏差。

## 6.3 创造性破坏

分析：

- 新技术替代的旧产品和旧模式；
- 新增需求来源；
- 成本曲线变化；
- 产业边界重构；
- 利润池迁移；
- 受益与受损环节。

回答：

- 产业长期增长由什么创造；
- 哪些旧环节会被压缩；
- 哪些新环节将成为核心。

---

# 模块二：产业结构

## 6.4 产业链全景

按以下结构拆解：

```text
上游资源
→ 核心材料
→ 关键设备
→ 核心零部件
→ 中游制造
→ 基础设施与渠道
→ 终端产品
→ 运营与服务
→ 应用场景
```

同时补充：

- 标准；
- 牌照；
- 认证；
- 数据；
- 软件；
- 维修和服务；
- 生态平台；
- 关键客户。

## 6.5 价值量与利润池

判断：

- 单位价值量分布；
- 毛利率与资本强度；
- 规模效应；
- 议价能力；
- 定价权；
- 价格战风险；
- 未来利润池迁移方向。

## 6.6 技术路线与标准

分析：

- 主流与替代技术路线；
- 成熟度、性能、成本、可靠性；
- 标准制定和认证周期；
- 路线切换风险；
- 技术变化对现有产能的影响。

## 6.7 商业模式

分析：

- 付费主体；
- 收费方式；
- 收入重复性；
- 毛利率和现金流特征；
- 商业模式成熟度；
- 下游持续支付能力。

---

# 模块三：约束条件

## 6.8 约束识别

系统识别：

- 技术；
- 资源；
- 产能；
- 资金；
- 成本；
- 能源；
- 基础设施；
- 牌照和标准；
- 应用场景；
- 人才与组织；
- 地缘与供应链。

## 6.9 约束排序与迁移

至少确定前三项核心约束，并回答：

- 为什么构成瓶颈；
- 将持续多久；
- 谁具备解决能力；
- 需要投入何种资源；
- 哪些环节将优先获得资本和订单；
- 瓶颈缓解后，约束将向哪里迁移。

约束传导：

```text
需求增长
→ 瓶颈供给不足
→ 价格、订单和资本集中
→ 技术突破或扩产
→ 瓶颈缓解
→ 约束向下一环节迁移
```

---

# 模块四：周期与节奏

## 6.10 产业发展阶段

固定阶段：

1. 技术探索期；
2. 示范验证期；
3. 产能建设期；
4. 批量生产期；
5. 渗透率提升期；
6. 利润兑现期；
7. 成熟整合期；
8. 出清与再成长。

主要依据：

- 技术和产品成熟度；
- 商业模式；
- 产能建设；
- 订单规模；
- 渗透率；
- 竞争格局；
- 盈利能力。

## 6.11 供需与资本开支周期

核心问题：

> 当前需求增速和供给增速谁更快，未来新产能将如何改变供需关系？

分析五个层面：

### 需求

政府采购、企业资本开支、居民消费、出口、国产替代、新应用、更新换代、补贴、渗透率和补库存需求。

### 供给与产能

原有产能、新增产能、开工率、产能利用率、良率、交付周期、新进入者、进口供给和低效产能退出。

### 库存

原材料、在产品、产成品和渠道库存，识别主动补库存、被动补库存、主动去库存和被动去库存。

### 价格

产品价格、招标价格、原材料价格、单位产品收入、长协价格、折扣和价格战，区分良性降价与竞争性降价。

### 资本开支

固定资产投资、企业资本开支、在建工程、设备采购、新增产线、基础设施、项目投产和融资规模。

固定判断周期位置：

1. 景气启动；
2. 景气扩张；
3. 供给释放；
4. 出清修复。

## 6.12 行业利润周期

分析传导：

```text
需求改善
→ 订单增加
→ 排产与交付
→ 行业收入增长
→ 产能利用率变化
→ 单位成本变化
→ 毛利率变化
→ 净利润变化
→ 经营现金流变化
→ 自由现金流和资本回报变化
```

固定阶段：

1. 订单先行期；
2. 收入兑现期；
3. 利润释放期；
4. 现金流兑现期；
5. 利润承压期。

## 6.13 产业投资生命周期

固定阶段：

1. 产业早期：高赔率、低确定性；
2. 产业中期：产业资本、流动性与业绩共振；
3. 泡沫阶段：收益更多依赖流动性；
4. 出清阶段：产业延续，环节与参与者分化；
5. 再成长阶段：瓶颈缓解，优质环节和龙头重新定价。

分析：

- 市场关注度；
- 产业资本投入；
- 金融资本流入；
- ETF和指数承接；
- 行业估值；
- 融资与并购；
- 业绩兑现程度；
- 产业与市场错位。

---

# 模块五：竞争格局与资本作用

## 6.14 竞争格局

分析：

- 龙头、追赶者、新进入者和退出者；
- 海外参与者、国有主体和科研院所；
- 市场集中度和份额变化；
- 成本曲线；
- 技术、客户、牌照和规模优势；
- 并购整合和产能出清。

回答：

- 资源和订单是否向头部集中；
- 谁位于成本曲线左侧；
- 谁能穿越价格战和行业出清；
- 行业是否进入集中度提升阶段。

## 6.15 资本市场作用

分析资本通过以下方式影响产业：

- IPO和再融资；
- 产业基金和风险投资；
- 并购重组；
- ETF和指数产品；
- 估值溢价；
- 股权激励。

回答：

- 资本流向哪个环节；
- 是否解决真实产业瓶颈；
- 是否形成有效研发、产能或基础设施；
- 是否推动重复建设和产能过剩；
- 是否放大阶段性估值泡沫；
- 是否促进产业整合和效率提升。

## 7. 执行步骤

每次执行按以下顺序：

1. 明确产业、区域、时间范围和研究问题；
2. 读取核心知识源；
3. 核验统计口径；
4. 收集战略、规划、产业链、供需、投资、利润和资本市场证据；
5. 分别完成五大模块判断；
6. 识别各模块之间的传导关系；
7. 给出当前阶段、下一阶段条件和反证指标；
8. 按 `template.md` 选择对应输出模式；
9. 完成质量检查；
10. 对证据不足的结论标记“待验证”。

## 8. 输出规则

所有输出结构均从 `template.md` 选择：

- 完整产业分析报告；
- 简版产业分析；
- 周期与节奏专项；
- 动态监测报告；
- 多产业横向比较。

选择原则：

- 用户要求“详细、全面、专题报告”时使用完整模板；
- 用户要求“快速判断、简要解读”时使用简版模板；
- 用户只问产业阶段、投资增速、供需或利润时使用周期专项模板；
- 用户要求日报、周报、月报时使用动态监测模板；
- 用户要求比较多个行业时使用横向比较模板。

不得在本文件中复制或修改输出表格。模板结构只在 `template.md` 中维护。

## 9. 质量检查

### 9.1 口径

- 是否解释数据定义和统计范围；
- 是否区分产业与子行业；
- 是否识别低基数和集中交付；
- 是否区分投资、产量、收入和利润。

### 9.2 证据

- 每个核心结论是否有证据；
- 是否优先使用官方和一手资料；
- 是否标记事实、推断、假设和待验证；
- 是否给出置信度。

### 9.3 逻辑

- 是否说明规划如何传导到投资、产能和供给；
- 是否判断需求与供给谁增长更快；
- 是否说明资本开支何时形成新增供给；
- 是否判断产业利润传导到哪个阶段；
- 是否说明下一阶段条件和反证指标。

### 9.4 边界

- 是否保持产业层面分析；
- 是否只用企业数据验证产业判断；
- 是否避免将集团任务直接映射到上市公司；
- 是否避免将产品能力当作真实订单。

### 9.5 表达

- 是否先结论、后证据；
- 是否避免只贴阶段标签；
- 是否说明传导机制；
- 是否避免堆砌公司名单；
- 是否明确不确定性。

---

## 10. 统一收束逻辑

完成分析后，结论应体现：

> 时代主线决定产业方向，产业结构决定价值分布，约束条件决定稀缺环节，周期与节奏决定增量释放顺序，竞争格局决定利润归属，资本作用决定产业演化速度和市场定价。
