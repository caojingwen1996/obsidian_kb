## 定位

本技能用于把公开信息转化为可审计、可证伪、可直接支持真实决策的个股研报。完整公司研究由本技能主导；三要素结论使用 `bbxm-three-factor-analysis`，风险方向使用 `bbxm-risk-identification`，但“方向正确”不能直接推出“可以买”。

## 路由边界

以下请求使用本技能：

- 研究一家上市公司或产出机构级研报；
- 估算公允价值、目标价、DCF 或安全边际；
- 比较多种估值方法并判断低估、合理或高估；
- 回答“如果今天这是一笔现金，我会不会买”。

以下请求使用相邻技能：

- 单条新闻、公告、政策、传闻是否有用：`bbxm-information-filter-flow`；
- 交割单、成交记录、仓位行为复盘：`bbxm-trade-ticket-review`；
- 只判断风险增强、持平、减弱或重新增强：`bbxm-risk-identification`；
- 只分析竞争格局、流动性、情绪位置和三要素共振：`bbxm-three-factor-analysis`。

如果完整研报同时要求三要素和风险方向，以本技能为主，分别调用两个平行技能并导入各自结论；两个技能互不调用，也不互相充当前置门槛。

## 开始前必须确认

1. 公司全称、证券代码、交易所和币种；
2. 研究截止时间，以及“实时”数据的实际抓取时间；
3. 报告准则和法定披露体系；
4. 用户是否指定数据源、持仓成本、预期持有期或风险预算；
5. 当前项目的落盘规则。若处于 llmwiki，先读取根 `AGENTS.md`、`SCHEMA.md`、`page-types.md`、`index.md`，再读取 `workbench/AGENTS.md` 和 `workbench/index.md`。

标的身份存在歧义时，停止估值，先核对身份。不得把同名证券、母子公司或不同上市地数据混用。

## 证据纪律

### 来源优先级

1. 监管机构、交易所和公司法定文件；
2. 公司投资者关系材料、业绩会记录和正式公告；
3. 用户指定的行情、估值或评级平台；
4. 券商研报和权威媒体；
5. 聚合器、搜索摘要和社交信息仅用于定位线索。

财务历史必须按市场选择法定来源：

- 美国 SEC 申报公司：SEC 10-K、10-Q、8-K；
- A 股：上交所、深交所、北交所或巨潮资讯法定披露；
- 港股：港交所披露易及公司法定报告；
- 其他市场：当地监管机构或交易所正式披露。

用户要求 SEC 但目标不是 SEC 申报公司时，明确写“SEC 文件不适用”，再使用当地法定披露；不得伪造 SEC 口径。

### 写作约束

- 每个关键数据标注来源、发布日期或抓取时间和口径。
- 事实与判断分开。推断必须以 **“我的判断”** 开头，并紧接依据。
- 来源冲突时列出各自数字、口径和时间，再说明采信理由。
- 无法取得的数据统一写 **“未获取到”**，并写已检查的来源；不得使用记忆补全。
- 数据源失败不等于数据不存在，不得据此写“没有”。
- 多空论据必须并陈，并分别写出验证指标和失效条件。
- 搜索摘要不能成为关键财务数字或结论的唯一来源。
- 实时市场判断必须当日重新核验，不复用旧价格、旧估值或旧新闻结论。

## 执行流程

### Step 1：建立来源矩阵

为下列项目逐项记录“目标来源、实际来源、数据时间、状态”：

- 实时价格、52 周高低、今年以来表现；
- 第三方公允价值、护城河评级、星级；
- 最新季报、分部收入、管理层指引；
- 卖方评级、目标价分布；
- 近三个月新闻与催化剂；
- 五年营收、毛利率、自由现金流和 ROIC。
- 三要素所需的连续量价、换手率、相对强弱、融资、ETF / 机构代理和资金数据。
- 企业全部已识别产业位置的业务暴露与景气证据，包括需求、供给、价格 / 价差、库存 / 订单、产能 / 资本开支和政策信号中适用的字段。

状态只能为：`已获取`、`未获取到`、`不适用`、`口径冲突待解释`。

### Step 2：采集行情与外部预期

按用户指定来源获取实时价格、52 周高低和年初至今表现；未指定时优先使用交易所或可靠行情平台。记录是否复权、盘中或收盘、当地时区。

为后续三要素映射同步收集可取得的连续交易证据：至少两个可比窗口的价格、成交额、换手率、相对行业强弱，以及融资、ETF / 机构代理或大单资金。无法取得的字段登记到来源矩阵并说明缺口，不得等到 Step 7 才无记录地另起一套数据口径。

第三方公允价值、护城河和星级只能在实际取得页面字段后填入。付费墙或登录限制下写“未获取到”，不得从搜索摘要猜测。

卖方观点至少记录：机构、评级、目标价、发布日期、预测期和关键假设。目标价分布缺少样本明细时必须降低可信度。

### Step 3：提取法定财务历史

至少整理最近五个完整财年：

- 营业收入；
- 毛利率；
- 经营现金流；
- 资本性支出；
- 自由现金流；
- ROIC 或明确标注的 ROIC 代理值。

默认计算口径：

```text
自由现金流 = 经营现金流 - 购建长期资产支付的现金
NOPAT = 经营利润 × (1 - 标准化税率)
ROIC 代理值 = NOPAT ÷ 平均投入资本
```

若公司披露口径不同，可并列展示，但不得混为同一序列。一次性损益、会计政策变化、重大重组和币种变化必须说明。

### Step 4：拆解最新经营变化

提取最新季度或半年度的收入、利润率、净利润、调整后利润、经营现金流、订单/储备及资产负债表异常项。结合分部收入和利润率判断增长来自数量、价格、并购、会计重分类还是真实需求。

管理层指引必须区分：正式量化指引、定性展望、卖方预测和“我的判断”。

同时完成商业模式事实底稿：收入来源、客户与定价权、成本结构、资本强度、主要竞争者、替代方案和监管约束，并在第 4 章形成独立、可引用的基本面结论。

产业位置不得默认只有一个，也不得套用预设分类。根据公司的实际业务、资源禀赋、供需关系、客户用途、政策环境和技术变化，识别所有与收入、利润、资产价值或估值逻辑有实质关联的产业位置，并分别写明识别依据。某种属性、主题或需求驱动只有能写出“外部变化 → 公司产品量价或成本 → 分部利润 / 现金流”的传导链时才可列入；仅有概念关联时标为“弱关联 / 待验证”，不得当作已兑现产业位置。

第 4 章必须按以下规则形成“多产业位置景气度排序”：

1. 每个位置分别写明位置名称、识别依据、公司业务 / 资产暴露、当前景气度、景气趋势、核心证据、财务传导、验证指标和失效条件；不得把多个位置合并成一句标签，也不得为了套用分类而拆分或合并实际位置。
2. 当前景气度只允许写 `高 / 较高 / 中 / 较低 / 低 / 证据不足`。至少核对需求、供给、价格或利润指标三类证据；按行业特性补充库存、订单、产能、资本开支或政策信号。
3. 排序首先按当前景气度从高到低，其次按趋势 `上行 / 持平 / 下行`，再次按公司可验证的利润或资产暴露强弱。`证据不足` 单列在已评级位置之后，不得伪装成低景气。
4. 同一利润来源被多个产业位置或叙事覆盖时明确交叉关系，不重复计算收入、利润或估值贡献。
5. 排名靠前不等于公司必然受益。结论必须同时说明受益弹性、兑现时滞、约束条件，以及景气能否进入公司利润和现金流。

### Step 5：整理新闻、催化剂与风险

新闻窗口默认最近三个月，按发生日而非搜索结果发布日期排序。每条催化剂写：

```text
事件 → 影响变量 → 财务传导 → 当前价格是否已反映 → 验证节点 → 失效条件
```

同时列出负面催化剂、公司治理风险、融资/减持/解禁、订单兑现、政策变化和估值锚失效风险。

### Step 6：执行三类估值

至少使用以下三种方法，不得用三个相近倍数冒充三种方法：

1. **相对估值**：选择可比公司，解释业务、成长、利润率、资本结构和会计口径的可比性；
2. **DCF**：明确预测期、收入增长、利润率、税率、折旧摊销、资本开支、营运资本、WACC、永续增长率和净债务；
3. **行业特定估值**：按行业选择 P/B-ROE、EV/产能、EV/用户、订单/储备价值、NAV 或分部加总，并解释适用性。

DCF 必须包含：

- 基准、悲观、乐观三种情景；
- WACC × 永续增长率或 WACC × 终值利润率的二维**敏感性分析**；
- 股权价值到每股价值的桥接；
- 反向 DCF 或反向估值，说明现价隐含的增长与利润率；
- 对长期负自由现金流公司的终值风险提示。

三种方法不得机械平均。按数据质量和商业模式适用性赋权，给出最终估值区间及相对现价的上行/下行空间。

### Step 7：映射冰冰小美框架

Step 7 是估值之后的独立决策流程，不是重新搜索并生成另一份独立三要素报告，也不是对前文风险的四句摘要。它以 Step 1—6 已收集、已登记来源的证据和数据为主要输入，按照冰冰小美体系三要素重新组织，辅助最终决策判断：

1. 读取并执行 `bbxm-three-factor-analysis/SKILL.md` 的分析方法，把 Step 1—6 的证据映射为竞争格局、流动性辩证分析、情绪位置变化三类。第 11 章引用前文章节与第 2 章来源矩阵，不重复抄写商业模式、财务、新闻和估值正文。
2. 报告需要风险方向时，读取并完整执行 `bbxm-risk-identification/SKILL.md`，只导入“风险增强 / 风险持平 / 风险减弱 / 风险重新增强 / 证据不足”之一，以及传导链、验证指标、失效条件和置信度。
3. 三要素结论与风险方向并列展示，互不替代。完整研报的估值、现金决策和最终动作仍由本技能综合财务、估值、三要素、风险方向与用户约束后形成。
4. 对每个要素先执行证据充分性闸门，再判断状态。证据不足时，状态必须写“证据不足”，不得判断为“有利”“中性”或“不利”；同时列出缺失字段、已检查来源、对决策的影响和下一验证点，不得用产业叙事、单日涨跌或主观感觉补空。
5. Step 7 默认不补采，也不为完成表格而另起搜索。若用户明确要求补齐最新数据或重跑完整研报，只补采已识别的关键缺口；补采结果必须先回填第 2 章来源矩阵和对应事实章节，再进入第 11 章，不能只在第 11 章出现孤立数据。

映射口径：

- 竞争格局：主要引用第 4—7 章的商业模式、多产业位置景气度排序、分部盈利、资本回报、订单和最新经营证据；
- 流动性辩证分析：主要引用 Step 2 形成的量价、换手、相对强弱、融资、ETF / 机构代理和资金数据，并结合第 8—9 章的供给与事件影响；
- 情绪位置变化：主要引用行情相对强弱、催化兑现后的价格反馈、卖方预期、多空论据和交易扩散证据；
- Step 6 的估值结果不强行归入任一要素，作为三要素行动映射的价格约束，防止把“跌得多”直接等同于“低估”或买点。

`template.md` 第 11.1—11.7 节是三要素导入的硬契约：不得删节、合并成一张摘要表，也不得因数据不足而省略小节。状态总表只能作为索引，不能替代逐项证据分析。第 11.1 节必须显示“前六步证据映射”，使每个要素可以追溯到前文章节和第 2 章来源矩阵。数据不足时仍保留对应小节，并明确写出“未获取到”、已检查来源和对结论的影响。

验收必须核对两个子技能的实际证据与结论；章节存在、状态总表存在或出现 `wait`、`observe` 等关键词，不能证明 Step 7 已执行。

### Step 8：形成决策并落盘

使用同目录 `template.md`。结论必须回答：

1. 当前属于低估、合理还是高估；
2. 动作为买入、观望还是规避；
3. 估值区间与相对现价空间；
4. **如果今天这是一笔现金，我会买它吗**；
5. 哪些事实会改变结论；
6. 后续 3–5 个监控指标。

默认文件名：`YYYY-MM-DD-HHmm-<标的>-机构级决策研报.md`。

在 llmwiki 中，库外原始材料必须先保存到 `sources/`。正式研报保存为 `workbench/targets/YYYY-MM-DD-HHmm-<标的>-机构级决策研报.md`，frontmatter 使用 `artifact_type: equity_research`；`workbench/targets/` 保持扁平，不创建证券代码子目录或单标的索引。更新 `workbench/index.md` 与根 `log.md`，不更新根 `index.md`，也不得把研报保存到 `wiki/queries/`。其他项目保存到用户指定位置；未指定时使用当前项目合理的研究输出目录。

### Step 9：生成并验证 HTML

Markdown 通过完整性检查后，默认必须再生成一份同源 HTML；只有用户明确要求仅交付 Markdown 时才可跳过。HTML 不维护独立研究正文，必须由最终 Markdown 通过技能内统一渲染器生成。

使用命令：

```text
node .agents/skills/bbxm-equity-research/scripts/render-report-html.cjs --input <研报.md> --output <研报.html> --vault-root <知识库根目录>
```

若系统 PATH 中没有 `node`，先调用工作区依赖定位能力取得 Node.js 可执行文件，再用其绝对路径执行同一命令。禁止通过 PowerShell 管道或内联脚本传递中文 HTML 文本。

输出规则：

1. 在 llmwiki 中，HTML 不进入 `workbench/`；写回原自动化分类 `sources/automations/<分类>/`，尚未分类时写入 `sources/automations/temp/`，文件名与 Markdown 主文件名一致；
2. 在其他项目中，用户指定 HTML 目录时写入该目录；未指定时与 Markdown 同目录、同名，仅扩展名改为 `.html`；
3. 页面固定使用技能内 `assets/report.css` 的版式：渐变封面、判断徽章、四张关键指标卡、左侧目录、右侧正文和打印样式；
4. 封面字段从“决策摘要”和报告元数据自动提取；缺失字段显示“未获取”，不得猜测；
5. 生成后必须检查 16 个目录锚点、UTF-8 中文、关键指标、Obsidian 双链转换和 Markdown / HTML 结论一致性；
6. HTML 生成或验证失败时，整份研报仍视为未完成。

### Step 10：生成每日跟踪面板

HTML 必须在16章正文之前生成一个可重复更新的“每日跟踪面板”，用于把静态研报转成持续决策入口。面板不是第二套研究正文，只消费第1章决策摘要和已有资金证据。

第1章决策摘要必须额外填写：`基本面状态`、`资金状态`、`风险方向`、`每日跟踪时间`、`资金面分析链接`。统一渲染器据此生成六张卡片：基本面状态、动态价值区间、股价偏离度、每日同步价、盘中实时、动作与置信度。

执行规则：

1. 面板使用 `<!-- DAILY_TRACKING_START -->` 和 `<!-- DAILY_TRACKING_END -->` 作为唯一稳定边界；重新渲染后必须各出现一次。
2. 每日同步价来自报告的正式行情快照；直接打开 `file://` 时仍可读取，不能依赖联网脚本。
3. 盘中实时只在本地大盘代理已为该证券配置白名单时启用；失败时显示降级状态，不得覆盖每日同步价。
4. 实时价格只能更新盘中卡和股价偏离度，不能自动改写动态价值区间、基本面状态或动作。
5. 资金面分析链接只允许同目录 HTML 文件名；文件不存在或未生成时写“未获取到”，不得伪造链接。
6. 头部不再重复显示静态 verdict/KPI；估值、价格、动作和置信度统一进入每日面板。

### Step 11：检查产业研报并建立 HTML 反向链接

在 llmwiki 中生成个股 HTML 后，检查该 HTML 所在的 `sources/automations/<分类>/` 目录是否已有产业研报。个股研报与产业研报位于同一分类目录时，产业研报应提供通往个股 HTML 的入口，使产业链阅读能够继续下钻到公司研究。

使用命令：

```text
node .agents/skills/bbxm-equity-research/scripts/link-report-to-industry.cjs --equity-html <个股研报.html>
```

执行规则：

1. 脚本自动查找同目录名称包含“产业完整分析报告”“产业分析报告”或“产业研报”的 HTML；找到唯一文件时，在其“个股研报链接”区追加当前个股 HTML；
2. 链接使用同目录相对路径；重复执行必须保持幂等，不得生成重复条目；
3. 产业研报不存在时，保留个股 HTML，并在完成说明中明确记录“产业研报不存在，未建立反向链接”，不得凭空创建产业研报；
4. 同目录存在多个产业研报候选时停止自动写入，核对主产业研报后通过 `--industry-report <产业研报.html>` 明确指定；
5. 写入后检查产业研报已包含个股 HTML 链接，且链接目标文件真实存在；
6. 非 llmwiki 项目或 HTML 不在产业分类目录时，此步骤标记为“不适用”。
