# 路由决定

**不应由 `fund-flow-analysis` 作为主技能执行。** 该请求应路由到 `bbxm-risk-identification`。

## 理由

用户明确限定“只判断风险方向和释放节点”，要求在风险增强、持平、减弱、重新增强之间作判断。请求的主动作是风险识别与风险方向跟踪，不是分析资金来源、资金共振或资金驱动的仓位动作。按照 `fund-flow-analysis` 的路由边界，纯风险方向请求应交给 `bbxm-risk-identification`，不应扩写成完整资金面报告。

## 资金面 Skill 的边界角色

若风险识别需要补充证据，`fund-flow-analysis` 可提供杠杆撤离、ETF 承接、订单流分歧、价格响应和筹码压力等资金证据，帮助验证风险是否仍在传导或开始释放。它只能作为证据输入，不能取代风险方向分类、风险传导链和释放节点判断。
