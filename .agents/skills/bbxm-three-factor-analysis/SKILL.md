# 冰冰小美体系三要素分析

## 定位

这是三要素分析的唯一项目级入口。独立分析竞争格局、流动性辩证分析和情绪位置变化，再映射为综合状态与行动语言。

本技能不负责风险增强或减弱判断；`bbxm-risk-identification` 独立处理风险方向。两者平行工作，互不调用。

## 路由边界

- 用户直接要求三要素分析、三要素共振或三要素行动映射：使用本技能。
- 用户要求风险增强、风险持平、风险减弱或风险重新增强：使用 `bbxm-risk-identification`。
- 用户要求过滤单条信息并判断其金融意义：使用 `bbxm-information-filter-flow`。
- 用户提供交割单或交易记录：使用 `bbxm-trade-ticket-review`；它可以消费本技能的分析结果。
- 用户要求完整公司研究、估值或 DCF：使用 `bbxm-equity-research`；它可以消费本技能的分析结果。

## 输入

开始前明确：

- 分析对象及所属市场；
- 对象层级：市场 / 指数 / 板块 / 个股；
- 分析截止时间；
- 需要回答的决策问题；
- 用户已提供的数据，以及仍缺失的数据。

必须先读取：

- `wiki/topics/冰冰小美-情绪体系理论篇.md`
- `wiki/concepts/冰冰小美-concept-体系三要素之竞争格局的比较优势.md`

继续搜索与对象有关的行业、事件、观点和推导页面。涉及当前市场的分析必须核验截至分析时点的最新可得数据。无法取得的数据必须逐项说明缺口、已检查范围及其对置信度的影响，不得用猜测补齐。

## 分析步骤

### 1. 竞争格局与比较优势

检查安全与发展方向、利润流向、议价权、国内与全球位置、供给扩张或出清、优势持续性。分别列出证据和反证，并给出失效条件。

### 2. 流动性辩证分析

核心边界：成交量只是结果，不能仅凭放量或缩量判断流动性有利或不利。必须同时解释钱从哪里来、能否持续、流向哪里、需要承接多少筹码，以及撤退时谁来承接。

按三层分别检查，不得只写微观成交和资金流：

- 宏观流动性：检查全球利率、美元、商品和外部风险偏好是否改变风险资产估值上限。
- 中观流动性：检查央行操作、政策利率、M2/M1、信用扩张及货币向实体和市场的传导，区分“货币存量充足”和“资金实际流通”。
- 微观流动性：检查成交与换手、上涨广度、ETF、机构、杠杆资金、做空回补、主线承接、报团溢价和资金分化，判断资金是扩散、切换、掩护退出还是整体撤退。

资金来源必须至少分为三类，并说明渠道、持续性与退出条件：

1. 场外真实增量：新资金进入市场或对象；
2. 存量迁移：卖出其他方向后转入对象；
3. 减量报团：市场总体资金下降，但剩余资金集中到少数方向。

不得混用两类“融资”：

- 杠杆资金：融资余额、融资买入、保证金或其他杠杆接力；
- 股票融资供给：IPO、增发、配股、可转债等新增股票或潜在筹码供给。

最低数据检查必须覆盖适用于该对象的项目，并给出数据时点、观察窗口和来源；无法取得或不适用时逐项说明：

- 宏观：全球利率、美元或汇率、商品与外部风险偏好；
- 中观：央行公开市场操作、政策利率、M2/M1、社融或信用传导；
- 微观：成交额与换手率、上涨/下跌广度、ETF 份额或净申购、机构或主力承接、融资余额、做空回补迹象；
- 供给与抛压：IPO/再融资、减持、解禁、套牢盘和高位筹码松动；
- 时间窗口：至少比较当日、短周期和中周期；不能取得三档窗口时说明缺口及其对结论的影响。

最后检查潜在退出路径。分别列出支持证据、反向证据、下一验证点和失效条件；资金流向数据不得脱离价格、广度、筹码供给与连续性单独使用。

### 3. 情绪位置变化

判断处于冰点、试错、扩散、拥挤、退潮或修复阶段；检查广度、接力、赚钱与亏钱效应、消息反应、情绪标与套利行为。分别列出证据和反证，并给出失效条件。

### 4. 形成状态判断

- 每个要素只能使用：`有利 / 中性 / 不利 / 证据不足`。
- 综合状态只能使用：`同步有利 / 部分有利 / 尚未形成有利共振 / 同步不利 / 证据不足`。
- 不得用一个强要素机械抵消两个不利要素。必须指出主导要素、拖累要素、下一验证点和失效条件。
- 不得因单条利好、单日行情或单一叙事给出确定性结论。

### 5. 映射行动

行动词只使用：`buy / sell / reduce / wait / observe / review`。

- 新增现金和已有持仓必须分开判断，并为每个行动写明触发条件。
- 证据不足时，新资金优先 `observe`，已有持仓优先 `review`；若持仓事实也不足，可用 `observe`。
- 行动结论必须能由三要素状态、主导矛盾和失效条件反推，不能只凭故事或单一信号。

## 输出要求

严格使用 [template.md](template.md) 的章节顺序。事实、判断和数据缺口必须分开表达，并明确来源、分析截止时间、核验范围、缺口、置信度与失效条件。
