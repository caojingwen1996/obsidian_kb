# 信息过滤流程 Skill

> 备注：本 skill 参考冰冰小美的《信息的金融意义》全系列。

## 0. 触发边界

本 Skill 是**显式调用型技能**。只有用户明确要求使用本技能时才启用。

明确调用包括：

- 点名 `bbxm-information-filter-flow`、`信息过滤流程`、`信息过滤技能`；
- 明确说“用 / 按信息金融意义框架过滤这条信息”；
- 明确要求输出“信息过滤报告”。

以下情况**不得自动触发**本 Skill：

- 用户只是提供新闻、公告、研报、文章、帖子、产业、指数或个股材料；
- 用户只是说“分析一下”“怎么看”“对比一下”“总结一下”“买入逻辑是什么”；
- 用户要求风险识别、三要素分析、产业分析、公司研究、交易复盘或知识库查询，但没有明确点名本技能。

这些情况应按用户的实际问题选择其他技能或直接回答，不要把普通分析改写成信息过滤报告。

## 1. Skill 目标

本 Skill 用于过滤、归纳、评级和处理金融市场中的各类信息，帮助判断一条信息是否具备金融意义，是否已经被市场反应，是否仍有使用价值，以及最终应进入交易、配置、跟踪、归档或回避。

核心目标：

> 先识别信息属性，再判断风险状态；先确认来源与动机，再判断价格反应；先等待风险减弱，再决定是否交易。

## 输出格式要求

执行本 Skill 时，最终回复必须引用并严格套用同目录的 `template.md`。

规则：

1. `template.md` 是唯一的最终输出模板。
2. 本文件后续 Step 1-14 是分析检查清单，用于保证判断完整，不是最终回复格式。
3. 不得自行压缩为“结论版”、聊天版或自由格式，除非用户明确要求“简版”或“只要结论”。
4. 如果某一项信息不足，仍保留模板字段，并填写“待核验”“暂无证据”或“不适用”，不得删除字段。
5. 最终输出必须包含 `template.md` 中的全部一级/二级标题顺序。

## 2. 适用场景

仅当用户已经明确调用本 Skill 后，以下输入类型适用：

1. 一条新闻、政策、公告、研报、帖子或市场传闻。
2. 一篇文章中的核心观点或投资线索。
3. 某个板块、指数、产业、个股相关信息。
4. 央行、财政、五年规划、中央会议等官方信息。
5. 自媒体、大V、投顾、媒体、券商报告等二级信息。
6. 用户希望判断某条信息是否有用、是否过时、是否能交易。
7. 用户要求“用冰冰小美的信息金融意义框架分析”。

## 3. 核心原则

离“钱”最近的信息：央行、利率、汇率、信用、流动性、国债收益率、信贷结构。这类信息能解释直接影响钱的数量、价格、流向、风险偏好和资源配置，是她最高权重的信息来源。

第一手、官方、可验证的信息：官方原文、央行报告、统计数据、中央经济工作会议、五年规划、财政预算等。她不喜欢只看新闻、自媒体或别人的解读，认为二手三手信息会混入立场、动机和误差。

能解释国情、国策、产业承接的信息：非金融信息里，她偏好能回答“国家要解决什么问题、政策如何推动、产业如何承接、企业是否真实兑现”的信息，而不是简单的“政策利好”。

公开、真实、可长期跟踪的显学信息：她更信真实事件、政策文件、历史节点、产业演变，而不是内幕、神秘逻辑、翻墙信息、小作文或所谓信息差。

能被评级、能落到风控和交易窗口的信息：她关心信息是否仍有时效、是否已被市场充分反应、是否还有风险减弱节点。信息不是直接导向买入，而是先用于识别风险、判断仓位和决定是否回避。

---


## 4. 工作流程

## Step 1：信息的归纳

判断信息为金融信息，非金融信息，或者其他信息。

### 金融信息的判断原则：

调用金融信息类型字典 `financial-information-types.md`进行判断，输出结果：

```markdown
## 信息的归纳
- 金融信息类型代码：
- 类型名称：
- 作者参考级别：
- 匹配到的代表变量：
- 识别理由：
- 识别边界：
```
### 非金融信息的判断原则：

如果属于非金融信息，调用 `non-financial-information-types.md` 识别该信息属于一国、二策、三业、四界、五略中的哪一类，并提取作者参考级别、离有形之手远近、显学 / 隐学属性、代表变量与识别边界。

本步骤只回答“这条信息属于什么类型的非金融信息”，不判断交易、配置、跟踪、归档或回避。
输出格式：

```markdown
## 信息的归纳

- 非金融信息类型代码：N2.1 + N2.3
- 类型名称：国策信息 + 规划执行信息
- 所属层级：二策
- 作者参考级别：P0-P1
- 作者态度：
- 识别理由：五年规划直接体现国家阶段性目标和资源配置方向，属于国策与规划执行信息。
- 识别边界：不能直接推出某个板块或个股机会，需要继续验证政策、产业和企业承接。
```
---



## Step 2：判断信息来源与可信度

信息来源按可信度分层。

| 评级 | 来源                   | 权重 |
| -- | -------------------- | -- |
| S  | 央行、中央级会议、官方原文、统计数据   | 最高 |
| A  | 国务院、财政、发改、行业主管部门正式文件 | 高  |
| B  | 上市公司公告、财报、交易所文件      | 中高 |
| C  | 券商研报、机构调研、专业分析       | 中  |
| D  | 媒体、自媒体、大V、投顾         | 低  |
| E  | 小道消息、传闻、群聊、无法验证信息    | 极低 |

输出格式：

```markdown
## 来源可信度

- 信息来源：
- 是否一手信息：
- 是否官方原文：
- 是否经过二次解读：
- 是否存在商业动机：
- 可信度评级：S / A / B / C / D / E
- 是否需要回到原文核验：
```

---

## Step 3：传播对象与发布动机判断

判断信息是说给谁听的。

传播对象类型：

| 类型     | 特征               | 风险     |
| ------ | ---------------- | ------ |
| 普遍对象   | 五年规划、重大政策、官方发布   | 需要长期跟踪 |
| 特定对象   | 投顾、收费社群、荐股、产业链信息 | 可能引导行动 |
| 情绪对象   | 新闻标题、自媒体热点、吃瓜信息  | 放大情绪   |
| 后手对象   | 热点尾声、利好刷屏、群体追涨   | 接盘风险   |
| 体系建设对象 | 深度文章、方法论、长期框架    | 适合归纳   |

输出格式：

```markdown
## 传播对象与动机

- 发布者是谁：
- 发布者动机：
- 信息说给谁听：
- 我是否属于目标对象：
- 是否引导交易行为：
- 是否制造紧迫感：
- 是否存在收费、引流、荐股、知识变现：
- 结论：
```

## Step 4：信息评级

| 评级  | 信息类型           | 代表内容                 | 用途        |
| --- | -------------- | -------------------- | --------- |
| S 级 | 央行、货币、信贷、利率、汇率 | 央行报告、美联储政策、货币周期      | 判断大势。     |
| A 级 | 国家战略、五年规划、财政预算 | 五年规划、中央经济工作会议、政府工作报告 | 判断中长期方向。  |
| B 级 | 产业真实发展         | 审厂、订单、资本开支、技术突破、产能验证 | 判断投资机会。   |
| C 级 | 企业经营变化         | 财报、公告、并购、业务转型        | 判断个体承接能力。 |
| D 级 | 题材概念情绪         | 热点、涨停、概念叠加、自媒体扩散     | 判断投机波动。   |
| E 级 | 未验证来源          | 小作文、传闻、情绪化分析、马后炮     | 默认降权或忽略。  |
输出格式：

```markdown
## 9. 信息评级

- 信息属性：
- 评级理由：
- 是否适合配置：
- 是否适合交易：
- 是否只适合跟踪：
- 是否需要过滤：
```

## Step 5：时效性判断

判断：

- 这条信息处于哪个时间阶段？
- 市场已经反应了多少？
- 大众是否已经形成共识？
- 媒体是否开始集中报道？
- 价格是否已经完成表达？
- 短期利空是否改变长期趋势？
- 当前还能参与，还是只适合复盘？

| 阶段   | 特征         | 处理   |
| ---- | ---------- | ---- |
| 预热期  | 信息出现但市场未重视 | 跟踪   |
| 启动期  | 资金开始反应     | 观察窗口 |
| 扩散期  | 媒体和大众开始关注  | 谨慎参与 |
| 高潮期  | 利好刷屏，价格大涨  | 防止后手 |
| 兑现期  | 事件落地或预期兑现  | 防止回落 |
| 失效期  | 信息无法再推动价格  | 过滤   |
| 再激活期 | 旧信息遇到新催化   | 重新评级 |

输出格式：

```markdown
## 时效性判断

- 当前处于：预热 / 启动 / 扩散 / 高潮 / 兑现 / 失效 / 再激活
- 大众认知程度：
- 媒体报道强度：
- 是否接近事件落地：
- 是否存在催化节点：
- 是否还有交易窗口：
```

---

## Step 6：信息使用价值判断

判断：

- 市场现在用不用它？
- 资金现在认不认它？
- 它是否处于主线中？
- 它是否符合宏观背景？
- 它是否已经被炒过？
- 它是否已经透支未来？
- 它再次出现时，还能否推动股价？
- 它现在是机会，还是后手陷阱？

| 状态     | 含义         | 处理   |
| ------ | ---------- | ---- |
| 尚未被使用  | 有潜在价值但缺少催化 | 长期跟踪 |
| 正在被使用  | 资金和情绪正在反应  | 观察风险 |
| 过度使用   | 已经透支远期预期   | 回避   |
| 暂时无人使用 | 有价值但无资金关注  | 归档等待 |
| 已经失效   | 再次出现也无推动力  | 过滤   |

输出格式：

```markdown
##  信息使用价值

- 市场是否正在使用：
- 是否有资金配合：
- 是否有情绪配合：
- 是否有宏观背景支持：
- 是否已经被上一轮行情使用过：
- 是否已经过度使用：
- 当前处理方式：
```

## 9. 禁止事项

执行本 Skill 时，禁止以下行为：

1. 单凭利好直接给出买入结论。
2. 单凭官方政策判断某个个股必然受益。
3. 把产业长期正确直接等同为当前可交易。
4. 忽略价格是否已经充分反应。
5. 忽略信息是否已经被上一轮行情使用过。
6. 忽略传播对象和发布动机。
7. 把媒体标题、自媒体热文、群聊传闻当成高权重信息。
8. 中观信息未经宏观验证直接进入交易评估。
9. 风险仍在增强时输出积极交易判断。
10. 把投机信息包装成投资信息。
11. 把噪音信息强行解释成机会。
12. 忽略归档、跟踪、回避等非交易处理方式。
13. 跳过同目录 `template.md`，直接输出自由格式结论。
