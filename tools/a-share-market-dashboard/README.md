# A 股大盘估值分位监控面板

这是一个可直接双击打开的单文件 HTML 工具，用透明、固定的规则回答：以历史为坐标，当前 A 股整体更偏便宜还是偏贵。

## 使用方法

双击打开 `a-share-market-dashboard.html`。页面先以明确标记的内置示例数据完成首屏渲染，再并行刷新公开数据；如果某个来源失败，则使用该数据域最近一次成功快照，若没有快照就将对应指标退出评分。

工具不需要安装依赖或启动本地服务器。`src/`、`tests/` 和 `scripts/` 仅用于维护、验证和重新构建，不影响成品 HTML 使用。

## 评分规则

- 位置层 40%：上证指数 10%、沪深 300 15%、中证全指 15%。计算 `当前收盘价 / MA250 - 1`，再求所选 1/3/5/10 年窗口内的历史分位；买点得分为 `100 - 偏离分位`。
- 估值层 35%：沪深 300 TTM PE 12%、前瞻盈利收益率 10%、ERP 13%。TTM PE 越低得分越高；前瞻盈利收益率和 ERP 越高得分越高。
- 情绪层 25%：上涨家数占比 7%、涨停/跌停强弱 6%、全市场成交额分位 6%、融资余额变化 6%。先计算贪婪度，再以 `100 - 贪婪度` 计入买点评分。
- 单项缺失时，不填 0、不估算，剩余有效权重自动归一化。
- 原始有效权重低于 70% 时，仅显示不完整试算，顶部结论固定为“数据不足，暂不判断”。

结论阈值、每项公式、原始权重、有效权重、分位、贡献、数据时间、抓取时间、来源和状态都显示在面板中，可解释、可回溯。

## 数据来源与边界

| 数据域 | 当前公开来源 | 刷新与失败处理 |
|---|---|---|
| 指数历史行情 | 东方财富；沪深 300、中证全指另以中证指数官网备援 | 约 60 秒轮询；失败后保留快照 |
| 沪深 300 TTM PE | 中证指数官网 | 日频；已兼容当前 `peg` 字段和亿元成交额单位 |
| 中国 10 年期国债收益率 | 东方财富数据中心 | 日频；JSONP 失败后尝试 Fetch |
| 市场宽度、涨跌停、成交额 | 东方财富沪深 A 股行情 | 约 5 分钟轮询 |
| 融资余额 | 金十数据公开汇总 | 日频 |
| 沪深 300 前瞻 PE | 暂无满足单文件公开访问和可核验要求的数据源 | 明确缺失；前瞻盈利收益率与 ERP 退出评分 |

公开接口可能改变字段、限制跨域或触发访问频率控制。工具会在“数据审计”中记录每个失败边界，不会把请求失败解释为“市场没有数据”。在本次验证环境中，中证指数、国债收益率和融资余额接口可以返回数据；东方财富历史行情接口出现代理连接中断，因此仍保留 JSONP、缓存、备援和缺失降级机制。

## 缓存与离线行为

每个数据域独立写入浏览器 `localStorage`。直接打开本地文件时，如果浏览器策略禁止持久化存储，工具会自动退回当前页面生命周期内的内存缓存，不会因此白屏。清理浏览器对本地文件的站点数据即可删除持久快照。

示例数据不会写入真实缓存。页面中的“示例数据”标记表示当前数值只用于演示交互，不能当作最新市场结论。

## 开发验证

需要 Node.js 20 或更高版本：

```powershell
node --test tests/*.test.mjs
node scripts/build.mjs
```

构建脚本会把 CSS 和 JavaScript 全部内联到 `a-share-market-dashboard.html`，并在写入前做 JavaScript 语法检查。
