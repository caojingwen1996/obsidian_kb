# 温度计

这是一个规则透明、可回溯的 A 股大盘监控工具，用历史分位回答：当前整体更偏便宜还是偏贵。评分不预测明日涨跌，也不构成投资建议。

## 推荐启动方式

双击 `启动大盘面板.cmd`。启动器会：

1. 从系统 `PATH` 或 Codex 工作区运行时定位 Node.js，并重新构建 `a-share-market-dashboard.html`；
2. 使用系统中的 Python 3 启动仅监听 `127.0.0.1` 的本地服务；
3. 自动选择一个可用端口并打开默认浏览器；
4. 让面板通过同源 `/api` 获取公开数据，绕开浏览器对本地 HTML 的 CORS 和 JSONP 限制。

不需要安装 pip 包，但系统需要 Python 3。系统 Node 与 Codex 内置 Node 均不可用，或看板构建失败时，启动器会停止而不会继续打开旧看板。命令窗口必须保持打开；关闭窗口后实时刷新停止，浏览器中的最近成功快照仍可展示。

`a-share-market-dashboard.html` 仍是完整的单文件 artifact，可以直接双击查看示例数据或已有缓存。要稳定获取联网数据，请使用启动器。

左侧使用树形导航，一级为“温度计 / 产业 / 我的 / 更新日志”。“温度计”下保留市场总览、位置分位、估值与 ERP、情绪温度、评分规则和数据审计；“产业”下按冰冰小美三大产业方向分为“战略 / 新兴 / 支柱”，对应战略资源、新兴产业和支柱产业；“我的”下提供仓位管理与持仓跟踪。展开一个分组时会收起另一个分组，并在本次页面打开期间记住各分组最后访问的子页。“更新日志”直接打开看板自身的更新记录。

仓位管理支持新增、编辑和删除持仓，并按记录的数量、成本价和现价计算持仓成本、当前市值、浮动盈亏与仓位占比；持仓跟踪可维护“持有 / 观察 / 计划加仓 / 计划减仓”状态与关注要点。通过启动器打开时，个人数据保存在本机 `data/portfolio.json`；直接双击单文件时降级保存在当前浏览器。仓位数据不会提交到公开数据接口。

产业页采用信息流布局，包含细分方向标签、产业研报看板、日期时间线和研报卡片，并支持当前页内搜索与标签筛选。

## 产业研报自动加载

构建时只扫描以下三个固定根目录：

- `sources/automations/战略资源/`
- `sources/automations/新兴产业/`
- `sources/automations/支柱产业/`

每个根目录的一级子目录会直接成为筛选标签；其下更深层级的 HTML 仍继承该一级标签。根目录直属 HTML 只在“全部”下显示。文件名包含“完整分析报告”的 HTML 显示在顶部“产业研报”区域，其他 HTML 进入日期信息流；Markdown 等非 HTML 文件不会加载。新增目录或移动研报后，重新双击启动器即可自动构建并同步页面；也可以手动运行构建脚本，无需修改代码中的产业关键词。

通过“启动大盘面板.cmd”打开看板时，本地服务器仅额外提供 `sources/automations/` 目录内的 HTML 研报；其他仓库文件、非 HTML 文件以及目录外路径仍拒绝访问。

航天电子机构级研报采用“每日同步 + 盘中增强”方式：直接双击 HTML 时显示每日任务写入的价格与判断；从大盘本地服务打开时，盘中行情卡通过固定白名单接口获取航天电子价格，每 60 秒刷新。接口失败只影响盘中卡，不会覆盖每日同步值。

## 更新日志维护

看板更新记录维护在 `src/changelog.json`，与知识库根目录的 `log.md` 分开。每条记录包含日期、星期、类型、标题、摘要和可选明细；构建时按日期倒序生成时间线。修改该文件后重新运行构建脚本，独立 HTML 产物中的更新日志会同步刷新。

## 评分规则

- 位置层 40%：上证指数 10%、沪深 300 15%、中证全指 15%。计算 `当前收盘价 / MA250 - 1`，再求所选 1/3/5/10 年窗口内的历史分位；买点得分为 `100 - 偏离分位`。
- 估值层 35%：沪深 300 TTM PE 12%、前瞻盈利收益率 10%、ERP 13%。TTM PE 越低得分越高；前瞻盈利收益率和 ERP 越高得分越高。
- 情绪层 25%：上涨家数占比 7%、涨停/跌停强弱 6%、全市场成交额分位 6%、融资余额变化 6%。先计算贪婪度，再以 `100 - 贪婪度` 计入买点评分。
- 单项缺失时不填 0、不估算，剩余有效权重自动归一化。
- 原始有效权重低于 70% 时，仅显示不完整试算，顶部结论固定为“数据不足，暂不判断”。

结论阈值、公式、原始权重、有效权重、分位、贡献、数据时间、抓取时间、来源和状态都显示在面板中。

## 数据来源与边界

| 数据域 | 本地代理来源 | 失败处理 |
|---|---|---|
| 指数历史行情 | 东方财富；不可用时切换腾讯行情；沪深 300、中证全指另有中证指数官网备援 | 独立快照或缺失 |
| 沪深 300 TTM PE | 中证指数官网 | 独立快照或缺失 |
| 中国 10 年期国债收益率 | 东方财富数据中心 | 独立快照或缺失 |
| 市场宽度、涨跌停、成交额 | 东方财富全量分页；不可用或响应不完整时切换新浪财经全量分页 | 独立快照或缺失 |
| 融资余额 | 金十数据公开汇总 | 独立快照或缺失 |
| 航天电子盘中行情 | 东方财富固定个股行情接口；不可用时切换腾讯行情；仅允许 `600879.SH` | 两个行情源均失败时，研报保留每日同步值并明确显示实时连接失败 |
| 沪深 300 前瞻 PE | 暂无满足公开访问和可核验要求的数据源 | 明确缺失；相关两项退出评分 |

本地代理只允许固定数据路由，不支持任意 URL 转发，也不监听局域网地址。本地文件只开放当前知识库的 `sources/` 与 `workbench/` 两棵白名单目录；解析后的路径必须仍位于对应目录内，任何试图通过 `../` 越界到知识库根目录或其他位置的请求都会返回 404。为适应全市场分页请求，前端在本地模式下使用 60 秒超时并按数据域顺序刷新，避免免费上游的连接池拥塞。

2026-07-17 的实际链路验证中，腾讯返回 3000 条指数日线，新浪返回 5527/5527 条 A 股行情；最终有效覆盖率为 77%，评分能够形成结论。未达到 100% 的原因是前瞻 PE 按规则保持缺失，不是网络错误。

第三方免费接口仍可能变更字段、限流或暂时不可达。工具会在“数据审计”中记录真实失败边界，不会把请求失败解释为“市场没有数据”。

## 缓存与离线行为

每个数据域独立写入浏览器 `localStorage`。如果浏览器禁止本地文件持久化，工具会退回当前页面生命周期内的内存缓存。示例数据不会写入真实缓存；“示例数据”标记下的数值不能当作最新市场结论。

## 开发验证

需要 Node.js 20 或更高版本；代理测试只需要 Python 3：

```powershell
python -m unittest discover -s tests -p test_local_proxy.py -v
node --test tests/*.test.mjs
node scripts/build.mjs
```

构建脚本把 CSS 和 JavaScript 全部内联到 `a-share-market-dashboard.html`，并在写入前检查 JavaScript 语法。
