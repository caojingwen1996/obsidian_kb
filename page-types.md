# 页面类型规范

本文件定义 `llmwiki/wiki/` 下的页面类型。  
Agent 在整理资料、创建页面或重构页面时，必须根据资料性质选择合适的页面类型。

各页面类型的可复制模板已拆分到 `templates/` 目录下。本文保留页面类型语义、适用场景和选择规则；创建新页面时必须以对应模板作为输出结构，并按资料内容补齐。

---

## 1. 页面类型总览

当前知识库使用以下七类页面：


| 页面类型           | 中文名   | 说明                  | 推荐目录              |
| -------------- | ----- | ------------------- | ----------------- |
| Concept Page   | 概念页   | 解释“是什么”             | `wiki/concepts/`  |
| Event Page     | 事件页   | 解释“发生了什么”           | `wiki/events/`    |
| View Page      | 观点页   | 解释“谁怎么看”            | `wiki/views/`     |
| Timeline Page  | 时间线页  | 解释“先后顺序是什么”         | `wiki/timelines/` |
| Reasoning Page | 推导链页  | 解释“为什么会这样推导”        | `wiki/reasoning/` |
| Topic Page     | 主题聚合页 | 组织一组相关页面, 承载“问题域入口” | `wiki/topics/`    |
| People Page    | 人物页   | 承载“作者入口”            | `wiki/people/`    |


核心理解：

```text
Concept Page   = 知识原子
People Page    = 观点和行动主体
Event Page     = 事实节点
View Page      = 主体观点
Timeline Page  = 事件序列
Reasoning Page = 因果推导
Topic Page     = 主题聚合页 / Hub 页
```

---

## 2. Concept Page：概念页

Concept Page 是稳定知识单元，用于解释概念、术语、模型、方法、框架。

它回答的问题是：

```text
这是什么？
它解决什么问题？
它由什么组成？
它和其他概念有什么关系？
```

适用于：

- 概念；
- 术语；
- 方法；
- 模型；
- 框架；
- 工具；
- 架构知识；
- 需求规格知识；
- 长期可复用的稳定知识。

 Concept 页面写作风格要求:

Concept 页面用于解释一个概念、框架或作者体系中的专有说法。写作时要保持“定义清楚、来源明确、边界克制、推导不过度”。

推荐路径：

```text
wiki/concepts/
```

必须参考的输出模板：

- `templates/concept-page-template.md`

---

## 3. People Page：人物页

本页面用于记录一个观点主体、投资者、作者、博主、研究者或关键人物的“人物画像”。

本页面重点关注：

- 这个人是谁；
- 这个人为什么值得进入知识库；
- 这个人的投资理念、价值观、性格、判断风格是什么；
- 这个人的观点适合如何使用；
- 这个人的观点有哪些边界和盲区。

本页面不负责承载全部文章、全部观点或全部推导。

推荐路径：

```text
wiki/people/
```

必须参考的输出模板：

- `templates/people-page-template.md`

---

## 4. Event Page：事件页

Event Page 用于记录单个事件、政策、会议、市场节点、公司动作、产业节点。

它回答的问题是：

```text
发生了什么？
什么时候发生？
谁参与？
背景是什么？
影响是什么？
它和哪些观点、推导、时间线有关？
```

适用于：

- 单个政策事件；
- 单个市场事件；
- 单个公司事件；
- 单个会议；
- 单个交易动作；
- 单个风险节点；
- 单个产业节点。

示例：

```text
2026-05-14-风险节点
2026-05-15-特朗普访华
大基金减持长电科技
美债收益率突破5%
```

推荐路径：

```text
wiki/events/
```

推荐命名：

```text
YYYY-MM-DD-事件标题.md
```

如果日期不确定，可以使用：

```text
事件标题.md
```

必须参考的输出模板：

- `templates/event-page-template.md`

---

## 5. View Page：观点页


View Page 用于记录一个阶段性观点、判断框架或投资结论。

它的核心职责不是记录事实本身，也不是展开完整推导链，而是回答：

> 在当前信息条件下，我怎么看这个问题？

View Page 关注的是“判断结果”和“观点演化”，尤其适合记录产业判断、投资观点、宏观判断、人物观点、市场风格判断等内容。

适用于：

- 某个作者对某个市场的看法；
- 某个博主对某个政策的判断；
- 某个机构对某个产业的观点；
- 某个人物对某个方法的主张；
- 用户自己对某个主题的阶段性判断。



推荐路径：

```text
wiki/views/
```

推荐命名：

```text
人物名对主题的看法.md
机构名对主题的判断.md
作者名关于主题的观点.md
```

必须参考的输出模板：

- `templates/view-page-template.md`

---

## 6. Timeline Page：时间线页

Timeline Page 用于记录多个事件的时间顺序。

注意：

```text
Timeline Page 只记录时间先后，不承担完整因果推导。
```

如果某个时间点背后有复杂因果逻辑，应链接到对应的 Reasoning Page。  
如果某个时间点来自某个作者观点，应链接到对应的 View Page。

适用于：

- 政策发布时间线；
- 市场事件时间线；
- 技术演进时间线；
- 项目阶段时间线；
- 产业发展时间线；
- 某个主题的发展脉络。

示例：

```text
市场事件时间线
AI Agent 发展时间线
半导体产业链事件时间线
财政税收模式转型时间线
```

推荐路径：

```text
wiki/timelines/
```

必须参考的输出模板：

- `templates/timeline-page-template.md`

---

## 7. Reasoning Page：推导链页

Reasoning Page 用于记录因果链、逻辑推导、观点拆解和风险演绎。

这类页面不是普通摘要，必须体现：

```text
前提 → 中间变量 → 传导机制 → 结论 → 风险点
```

适用于：

- 作者如何得出某个判断；
- 风险节点如何推导出来；
- 产业链逻辑；
- 宏观政策传导；
- 市场情绪与流动性变化；
- 事件之间的因果关系；
- 财政、产业、资产价格之间的传导。

生成规则引用

Reasoning Page 的具体生成要求，见：rules/reasoning-page-rules.md

推荐路径：

```text
wiki/reasoning/
```

必须参考的输出模板：

- `templates/reasoning-page-template.md`

---

## 8. Topic Page：主题聚合页 / Hub 页

Topic Page 用于组织一组相关页面。

它不是普通概念页，也不是文章摘要页。  
它的作用是做导航和聚合，组织一个问题域。

它回答的问题是：

```text
围绕这个主题，有哪些概念、人物、事件、观点、时间线、推导链和资料？
```

适用于：

- 市场风险推导；
- 半导体产业链；
- 财政税收模式转型；
- 儿童眼科就医；
- 任何需要长期跟踪的主题。

推荐路径：

```text
wiki/topics/
```

必须参考的输出模板：

- `templates/topic-page-template.md`

## 9. 页面类型选择规则

Agent 在整理资料时，应按以下规则判断页面类型。

---

### 9.1 默认不要直接创建 Topic Page

Topic Page 是聚合页，不是默认内容页。

默认优先判断资料是否属于：

1. Concept
2. People
3. Event
4. View
5. Timeline
6. Reasoning

只有当资料是在组织一个主题地图时，才创建或更新 Topic Page。

---

### 9.2 优先选择 Concept Page 的情况

当资料重点是解释一个稳定概念时，优先使用 Concept Page。

典型特征：

- 解释一个概念是什么；
- 说明方法、框架、模型；
- 可被多个页面复用；
- 不依赖单一事件或单篇文章。

---

### 9.3 优先选择 People Page 的情况

当资料重点是某个人或某个观点来源时，优先使用 People Page。

典型特征：

- 人物身份重要；
- 人物观点被多次引用；
- 人物参与了多个事件；
- 人物的行为影响观点或推导链；
- 需要追踪“谁说过什么”。

---

### 9.4 优先选择 Event Page 的情况

当资料重点是某个单独事件时，优先使用 Event Page。

典型特征：

- 有明确发生时间；
- 有明确参与方；
- 有事件结果或影响；
- 可被时间线引用；
- 可被观点或推导链引用。

---

### 9.5 优先选择 View Page 的情况

当资料重点是某个主体对某个问题的看法时，优先使用 View Page。

典型特征：

- 有明确观点来源；
- 有明确判断或立场；
- 重点是“谁怎么看”；
- 观点可能被后续推导引用；
- 不一定包含完整推导链。

---

### 9.6 优先选择 Timeline Page 的情况

当资料重点是多个事件的先后顺序时，优先使用 Timeline Page。

典型特征：

- 多个日期；
- 多个事件；
- 需要按时间排列；
- 不一定需要完整解释因果链。

---

### 9.7 优先选择 Reasoning Page 的情况

当资料重点是“作者如何得出某个判断”时，优先使用 Reasoning Page。

典型特征：

- 有明确结论；
- 有多个前提；
- 有因果传导；
- 有时间节点；
- 有风险触发条件；
- 有不确定性；
- 适合拆成“前提 → 变量 → 传导 → 结论”。

---

### 9.8 优先选择 Topic Page 的情况

当资料重点是组织某个领域的知识地图时，优先使用 Topic Page。

典型特征：

- 聚合多个概念；
- 聚合多个人物；
- 聚合多个事件；
- 聚合多个观点；
- 聚合多条推导；
- 作为导航入口；
- 不是单篇资料的摘要。

---

## 10. 页面类型组合规则

有些资料可能同时包含多种内容。此时应拆分处理。

---

### 10.1 同时有概念和观点

处理方式：

- 稳定概念写入 Concept Page；
- 某个主体对该概念的看法写入 View Page；
- View Page 链接 Concept Page。

示例：

```markdown
相关概念：[[LLM Wiki]]
相关观点：[[Karpathy对LLM Wiki的观点]]
```

---

### 10.2 同时有概念和推导

处理方式：

- 稳定概念写入 Concept Page；
- 当前文章中的因果分析写入 Reasoning Page；
- Reasoning Page 链接 Concept Page。

示例：

```markdown
相关概念：[[流动性]]、[[市场情绪]]
相关推导：[[冰冰小美-5月14日风险节点推导]]
```

---

### 10.3 同时有人物和观点

处理方式：

- 人物身份和稳定信息写入 People Page；
- 具体观点写入 View Page；
- People Page 与 View Page 互相链接。

示例：

```markdown
观点来源：[[Michael Burry]]
相关观点：[[Michael Burry对半导体做空的判断]]
```

---

### 10.4 同时有观点和推导

处理方式：

- 观点本身写入 View Page；
- 观点背后的推导链写入 Reasoning Page；
- 两个页面互相链接。

示例：

```markdown
相关观点：[[蓝庭集对有色金属的看法]]
相关推导：[[冰冰小美-有色金属拐点推导]]
```

---

### 10.5 同时有事件和时间线

处理方式：

- 单个事件写入 Event Page；
- 多个事件顺序写入 Timeline Page；
- Event Page 与 Timeline Page 互相链接。

示例：

```markdown
所属时间线：[[市场事件时间线]]
相关事件：[[2026-05-14-风险节点]]
```

---

### 10.6 同时有时间线和推导链

处理方式：

- 时间顺序写入 Timeline Page；
- 因果分析写入 Reasoning Page；
- 两个页面互相链接。

示例：

```markdown
相关时间线：[[市场事件时间线]]
相关推导：[[冰冰小美-5月14日风险节点推导]]
```

---

### 10.7 同时有主题和具体页面

处理方式：

- 具体知识写入 Concept / People / Event / View / Timeline / Reasoning；
- Topic Page 只负责聚合链接；
- 不要把所有内容都堆进 Topic Page。

示例：

```markdown
所属主题：[[市场风险推导]]
相关概念：[[流动性]]
相关观点：[[某作者对市场风险的判断]]
相关事件：[[2026-05-14-风险节点]]
相关推导：[[冰冰小美-5月14日风险节点推导]]
```

---

## 11. 页面创建原则

Agent 创建新页面前必须检查：

1. 是否已有同义页面；
2. 是否可以更新已有页面；
3. 新页面是否属于稳定知识单元；
4. 新页面是否能被未来复用；
5. 新页面属于哪种页面类型；
6. 新页面是否需要加入 `index.md`；
7. 新页面是否需要与现有页面建立双链。

如果只是临时摘录，不应创建新的 wiki 页面，应放入 `sources/`。

---

## 12. 页面质量要求

每个 `wiki/` 页面至少应满足：

1. 有清晰标题；
2. 有明确页面类型；
3. 有来源；
4. 有相关页面；
5. 有必要的双链；
6. 没有把观点误写成事实；
7. 没有把推测误写成结论；
8. 没有与其他页面重复表达同一概念、人物、事件或观点；
9. 没有把 Topic Page 当成普通笔记页；
10. 没有把 View Page 和 Reasoning Page 混用；
11. 没有把 Timeline Page 和 Reasoning Page 混用。
