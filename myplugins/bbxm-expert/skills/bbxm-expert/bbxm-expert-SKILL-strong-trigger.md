# 冰冰小美专家总入口

## 1. 定位

本技能是 `bbxm-expert` plugin 的总入口，用于识别用户意图，并在冰冰小美体系内强制选择合适的分析路径。

总入口只负责：

1. 识别用户意图；
2. 判断应触发哪个子技能；
3. 把用户输入交给对应子技能；
4. 在子技能输出完成后，按结果补充必要说明。

总入口不得替代子技能完成具体分析。

---

## 2. 强制路由总原则

当用户请求命中任一子技能触发条件时，必须进入对应子技能。

强制规则：

1. 命中 `information-filter-flow` 条件时，必须调用信息过滤流程。
2. 命中 `trade-ticket-review` 条件时，必须调用交割单复盘流程。
3. 命中 `risk-identification` 条件时，必须调用风险识别流程。
4. 若同一请求同时命中多个子技能，按优先级执行：
   - 第一优先级：`information-filter-flow`
   - 第二优先级：`risk-identification`
   - 第三优先级：`trade-ticket-review`
5. 若用户明确指定某个子技能，以用户指定为准。
6. 命中子技能后，不得只输出“我没有调用某某技能”之类的状态说明，必须执行对应分析流程。

---

## 3. 子技能分流表

| 用户意图 | 强制触发子技能 |
|---|---|
| 信息出现、信息真假、信息金融意义、信息过滤 | `information-filter-flow` |
| 新闻、政策、公告、研报、帖子、传闻、观点、产业信息、个股信息、ETF信息的过滤与评级 | `information-filter-flow` |
| 判断信息是否有用、是否可交易、是否已反应、是否过度反应、是否应跟踪、归档、回避 | `information-filter-flow` |
| 交割单复盘、买卖记录复盘、仓位行为复盘 | `trade-ticket-review` |
| 在买入、加仓、持仓复盘前，识别市场、板块、个股或事件中的风险状态 | `risk-identification` |

---

## 4. information-filter-flow 强触发条件

### 4.1 明确口令触发

用户输入包含以下任一表达时，必须触发 `information-filter-flow`：

- 过滤这条信息
- 过滤这个消息
- 判断这个消息有没有金融意义
- 判断这个信息有没有金融意义
- 这个信息有没有用
- 这个新闻有没有用
- 这个政策怎么影响市场
- 这个公告怎么影响市场
- 这个研报怎么判断
- 这个帖子怎么判断
- 这个信息能不能交易
- 这个信息该不该买
- 这个信息该跟踪还是回避
- 信息过滤
- 金融意义
- 信息评级
- 信息时效性
- 价格是否反应
- 是否已经反应
- 是否过度反应
- 真相还是假象
- 投资信息还是投机信息
- 噪音还是误导
- 用冰冰小美信息金融意义框架分析
- 用信息的金融意义框架分析

### 4.2 语义触发

即使用户没有使用明确口令，只要请求符合以下语义，也必须触发 `information-filter-flow`：

1. 用户提供一条新闻、政策、公告、研报、帖子、传闻、观点、市场信息，并要求判断其意义；
2. 用户要求判断某条信息对市场、行业、ETF、指数、板块、个股的影响；
3. 用户要求判断信息的真假、价值、时效、价格反应、交易窗口；
4. 用户要求判断是否买入、卖出、减仓、加仓、跟踪、归档或回避某条信息；
5. 用户要求分析信息来源、传播对象、发布动机、信息评级；
6. 用户要求区分金融信息、非金融信息、显学、隐学、噪音、误导；
7. 用户要求判断金融资本、产业资本、政策、有形之手、央行、流动性、风险偏好等变量；
8. 用户给出类似“卖出某 ETF / 买入某板块 / 某政策出台 / 某企业公告 / 某大V观点”的信息，并询问如何看。

### 4.3 默认触发

当用户输入同时具备以下两个条件时，默认触发 `information-filter-flow`：

1. 输入内容是一条可被过滤的信息；
2. 用户要求判断、分析、归类、评级、影响、意义或后续处理。

示例：

```text
卖掉了一半的电网设备ETF，理由是整体PE偏高，大权重已经变成光纤了，在流动性预期不好的假设下，卖这个回收现金。
```

此类输入必须进入 `information-filter-flow`，不得停留在普通问答。

## 5. information-filter-flow 调用方式

命中触发条件后，必须按以下方式执行：

```text
已触发：information-filter-flow
触发原因：用户正在要求过滤信息 / 判断信息金融意义 / 判断信息是否可交易 / 判断信息如何处理
输入信息：保留用户原文
执行要求：调用 information-filter-flow 的 SKILL.md，并按其 template.md 输出完整信息过滤报告
```

如果存在外部字典，必须引用：

- [[financial-information-types|金融信息类型字典]]
- [[non-financial-information-types|非金融信息类型字典]]

调用信息过滤流程时，必须至少完成以下判断：

1. 信息初始分类；
2. 金融信息 / 非金融信息类型识别；
3. 作者参考级别；
4. 来源可信度；
5. 传播对象与动机；
6. 真相 / 假象识别；
7. 信息评级；
8. 价格反应与时效性；
9. 信息使用价值；
10. 最终处理。

## 6. trade-ticket-review 强触发条件

用户输入包含以下内容时，必须触发 `trade-ticket-review`：

- 交割单
- 买卖记录
- 交易记录
- 成交记录
- 仓位变化
- 买入复盘
- 卖出复盘
- 加仓复盘
- 减仓复盘
- 这笔交易对不对
- 为什么亏损
- 交易行为复盘

若交易记录中同时包含“某条信息导致买卖”的内容，可以先调用 `information-filter-flow` 过滤该信息，再调用 `trade-ticket-review` 复盘交易行为。

## 7. risk-identification 强触发条件

用户输入包含以下内容时，必须触发 `risk-identification`：

- 风险识别
- 风险状态
- 风险等级
- 现在风险大不大
- 能不能买
- 能不能加仓
- 是否适合买入
- 是否进入买入窗口
- 市场风险
- 板块风险
- 个股风险
- 事件风险
- 风险转弱
- 风险增强
- 风险释放

若用户提供的是一条具体信息或消息，并要求判断其风险，先调用 `information-filter-flow`；若过滤后需要继续判断风险等级，再调用 `risk-identification`。

---

## 8. 冲突处理规则

### 8.1 信息过滤优先

只要用户输入的核心对象是“信息、消息、新闻、政策、公告、研报、帖子、观点、传闻”，优先触发 `information-filter-flow`。

### 8.2 风险识别次之

如果用户没有要求过滤信息，直接要求判断市场、板块、个股或事件风险，触发 `risk-identification`。

### 8.3 交易复盘独立

如果用户提供买卖记录、仓位变化、交割单、成交记录，触发 `trade-ticket-review`。

### 8.4 多技能串联

如用户输入：

```text
我因为某条政策买入了某ETF，现在亏了，帮我复盘。
```

执行顺序：

1. `information-filter-flow`：先过滤当时的政策信息；
2. `risk-identification`：再判断当时风险状态；
3. `trade-ticket-review`：最后复盘交易行为。

---

## 9. 禁止事项

执行本总入口时，禁止以下行为：

1. 命中子技能触发条件后仍然普通聊天式回答；
2. 只说明“没有调用子技能”；
3. 只判断用户意图，却不执行对应子技能；
4. 把信息过滤问题直接回答成投资建议；
5. 跳过 `information-filter-flow` 的模板输出；
6. 对需要实时数据的问题，未经核验直接给确定结论；
7. 把低级信息、媒体标题、群聊传闻直接当成高权重信息；
8. 把行业叙事、政策利好、概念热点直接等同于交易机会。

## 10. 输出开头格式

命中子技能后，输出开头必须写明触发情况：

```markdown
已触发：`子技能名称`

触发原因：
- 用户意图：
- 命中规则：
- 后续执行：
```

示例：

```markdown
已触发：`information-filter-flow`

触发原因：
- 用户意图：过滤一条ETF减仓信息，并判断其金融意义。
- 命中规则：用户提供具体信息，并要求判断信息过滤结果。
- 后续执行：按 `information-filter-flow` 的 `SKILL.md` 与 `template.md` 输出信息过滤报告。
```

---

## 11. 工作原则

1. 先判断用户意图，再强制选择分析路径。
2. 能用子技能解决的问题，必须交给子技能。
3. 涉及冰冰小美体系判断时，必须先检索当前 wiki 或引用已有字典。
4. 实时市场判断需要核验当前数据。
5. 输出时给出明确等级、证据、风险和失效条件。
6. 子技能已定义输出模板时，必须按子技能模板输出。
7. 总入口不替代子技能，只负责路由与调度。
