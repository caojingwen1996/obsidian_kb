# 冰冰小美专家总入口

## 定位

本技能是 `bbxm-expert` plugin 的总入口，用于识别用户意图，并在冰冰小美体系内选择合适的分析路径。

## 入口强制规则

当用户点名 `bbxm-expert`、`冰冰小美专家` 或 `冰冰小美体系` 时，必须先读取本总入口，再按下面的分流规则读取对应子技能；不得凭印象直接使用相邻 skill。

如果用户的主动作是“过滤一条信息”，必须读取并执行 `information-filter-flow/SKILL.md`。这里的“过滤”包括但不限于：

- “过滤这条信息”
- “帮我过滤这条新闻 / 公告 / 政策 / 研报 / 传闻”
- “这条信息有没有金融意义”
- “这个新闻有用吗”
- “这个信息能不能交易”
- “这个政策怎么影响市场”
- “判断投资 / 投机 / 噪音 / 误导”
- “该跟踪、交易、归档、回避还是过滤”

过滤类请求的优先级高于风险识别。即使信息中出现“风险”“买入”“加仓”“交易”“利好”“利空”等词，只要用户是在问这条信息本身如何处理，仍然先走 `information-filter-flow`，不要直接跳到 `risk-identification`。

## 子技能分流

| 用户意图 | 优先使用 |
|---|---|
| 信息出现、信息真假、信息金融意义、信息过滤、新闻/公告/政策/研报有没有用、能不能交易、该不该跟踪/归档/回避 | `information-filter-flow` |
| 交割单复盘、买卖记录复盘、仓位行为复盘 | `trade-ticket-review` |
| 在买入、加仓、持仓复盘前，识别市场、板块、个股或事件中的风险状态 | `risk-identification` |

## 冲突消解

按以下顺序判断，不得跳步：

1. 用户是否要求“过滤/判断一条信息如何处理”？是则使用 `information-filter-flow`。
2. 用户是否提供交割单、成交记录、买卖动作、仓位变化并要求复盘？是则使用 `trade-ticket-review`。
3. 用户是否要求识别市场、板块、个股或事件的风险状态，且不是在过滤单条信息？是则使用 `risk-identification`。
4. 如果一句话同时满足多个条件，优先选择用户的动词；“过滤/判断信息有没有用”优先于“风险/交易”这些名词。

反例：

- 用户说“过滤这条信息：某公司公告拟投资某项目”，不能只用 `risk-identification`；必须使用 `information-filter-flow`。
- 用户说“这个新闻能不能交易”，不能只给风险节点评级；必须使用 `information-filter-flow` 判断信息类型、来源可信度、价格反应、时效和最终处理方式。
- 用户说“复盘这笔买入依据是不是被消息误导”，先用 `trade-ticket-review` 复盘交易，再按需要引用 `information-filter-flow` 判断消息。

## 工作原则

1. 先判断用户意图，再选择分析路径。
2. 能用子技能解决的问题，优先交给子技能。
3. 分流到子技能后，必须读取该子技能的 `SKILL.md`，并遵守其输出格式；例如 `information-filter-flow` 要使用同目录 `template.md`。
4. 涉及冰冰小美体系判断时，必须先检索当前 wiki。
5. 实时市场判断需要核验当前数据。
6. 输出时给出明确等级、证据、风险和失效条件。
