# 机构级个股研究

## 定位

本技能用于把公开信息转化为可审计、可证伪、可直接支持真实决策的个股研报。完整公司研究由本技能主导；涉及风险阶段时可引用 `risk-identification`，但“方向正确”不能直接推出“可以买”。

## 路由边界

以下请求使用本技能：

- 研究一家上市公司或产出机构级研报；
- 估算公允价值、目标价、DCF 或安全边际；
- 比较多种估值方法并判断低估、合理或高估；
- 回答“如果今天这是一笔现金，我会不会买”。

以下请求使用相邻技能：

- 单条新闻、公告、政策、传闻是否有用：`information-filter-flow`；
- 交割单、成交记录、仓位行为复盘：`trade-ticket-review`；
- 只判断风险阶段或买入窗口：`risk-identification`。

如果完整研报同时要求风险判断，以本技能为主，并在最后调用或引用 `risk-identification` 的风险语言。

## 开始前必须确认

1. 公司全称、证券代码、交易所和币种；
2. 研究截止时间，以及“实时”数据的实际抓取时间；
3. 报告准则和法定披露体系；
4. 用户是否指定数据源、持仓成本、预期持有期或风险预算；
5. 当前项目的落盘规则。若处于 llmwiki，先读取 `AGENTS.md`、`schema.md`、`page-types.md` 和 `index.md`。

标的身份存在歧义时，停止估值，先核对身份。不得把同名证券、母子公司或不同上市地数据混用。

## 证据纪律

### 来源优先级

1. 监管机构、交易所和公司法定文件；
2. 公司投资者关系材料、业绩会记录和正式公告；
3. 用户指定的行情、估值或评级平台；
4. 券商研报和权威媒体；
5. 聚合器、搜索摘要和社交信息仅用于定位线索。

财务历史必须按市场选择法定来源：

- 美国 SEC 申报公司：SEC 10-K、10-Q、8-K；
- A 股：上交所、深交所、北交所或巨潮资讯法定披露；
- 港股：港交所披露易及公司法定报告；
- 其他市场：当地监管机构或交易所正式披露。

用户要求 SEC 但目标不是 SEC 申报公司时，明确写“SEC 文件不适用”，再使用当地法定披露；不得伪造 SEC 口径。

### 写作约束

- 每个关键数据标注来源、发布日期或抓取时间和口径。
- 事实与判断分开。推断必须以 **“我的判断”** 开头，并紧接依据。
- 来源冲突时列出各自数字、口径和时间，再说明采信理由。
- 无法取得的数据统一写 **“未获取到”**，并写已检查的来源；不得使用记忆补全。
- 数据源失败不等于数据不存在，不得据此写“没有”。
- 多空论据必须并陈，并分别写出验证指标和失效条件。
- 搜索摘要不能成为关键财务数字或结论的唯一来源。
- 实时市场判断必须当日重新核验，不复用旧价格、旧估值或旧新闻结论。

## 执行流程

### Step 1：建立来源矩阵

为下列项目逐项记录“目标来源、实际来源、数据时间、状态”：

- 实时价格、52 周高低、今年以来表现；
- 第三方公允价值、护城河评级、星级；
- 最新季报、分部收入、管理层指引；
- 卖方评级、目标价分布；
- 近三个月新闻与催化剂；
- 五年营收、毛利率、自由现金流和 ROIC。

状态只能为：`已获取`、`未获取到`、`不适用`、`口径冲突待解释`。

### Step 2：采集行情与外部预期

按用户指定来源获取实时价格、52 周高低和年初至今表现；未指定时优先使用交易所或可靠行情平台。记录是否复权、盘中或收盘、当地时区。

第三方公允价值、护城河和星级只能在实际取得页面字段后填入。付费墙或登录限制下写“未获取到”，不得从搜索摘要猜测。

卖方观点至少记录：机构、评级、目标价、发布日期、预测期和关键假设。目标价分布缺少样本明细时必须降低可信度。

### Step 3：提取法定财务历史

至少整理最近五个完整财年：

- 营业收入；
- 毛利率；
- 经营现金流；
- 资本性支出；
- 自由现金流；
- ROIC 或明确标注的 ROIC 代理值。

默认计算口径：

```text
自由现金流 = 经营现金流 - 购建长期资产支付的现金
NOPAT = 经营利润 × (1 - 标准化税率)
ROIC 代理值 = NOPAT ÷ 平均投入资本
```

若公司披露口径不同，可并列展示，但不得混为同一序列。一次性损益、会计政策变化、重大重组和币种变化必须说明。

### Step 4：拆解最新经营变化

提取最新季度或半年度的收入、利润率、净利润、调整后利润、经营现金流、订单/储备及资产负债表异常项。结合分部收入和利润率判断增长来自数量、价格、并购、会计重分类还是真实需求。

管理层指引必须区分：正式量化指引、定性展望、卖方预测和“我的判断”。

### Step 5：整理新闻、催化剂与风险

新闻窗口默认最近三个月，按发生日而非搜索结果发布日期排序。每条催化剂写：

```text
事件 → 影响变量 → 财务传导 → 当前价格是否已反映 → 验证节点 → 失效条件
```

同时列出负面催化剂、公司治理风险、融资/减持/解禁、订单兑现、政策变化和估值锚失效风险。

### Step 6：执行三类估值

至少使用以下三种方法，不得用三个相近倍数冒充三种方法：

1. **相对估值**：选择可比公司，解释业务、成长、利润率、资本结构和会计口径的可比性；
2. **DCF**：明确预测期、收入增长、利润率、税率、折旧摊销、资本开支、营运资本、WACC、永续增长率和净债务；
3. **行业特定估值**：按行业选择 P/B-ROE、EV/产能、EV/用户、订单/储备价值、NAV 或分部加总，并解释适用性。

DCF 必须包含：

- 基准、悲观、乐观三种情景；
- WACC × 永续增长率或 WACC × 终值利润率的二维**敏感性分析**；
- 股权价值到每股价值的桥接；
- 反向 DCF 或反向估值，说明现价隐含的增长与利润率；
- 对长期负自由现金流公司的终值风险提示。

三种方法不得机械平均。按数据质量和商业模式适用性赋权，给出最终估值区间及相对现价的上行/下行空间。

### Step 7：映射冰冰小美框架

在估值之后检查：

- 产业方向和竞争格局；
- 流动性来自真实增量还是短期接力；
- 情绪处于扩散、拥挤、释放还是转弱；
- 基本面、估值、情绪和流动性的风险传导；
- 风险是否转弱，以及承接和退出条件；
- 当前动作：`buy / sell / reduce / wait / observe / review`。

若需要完整风险阶段判定，读取 `risk-identification/SKILL.md`。不得仅凭产业方向或长期叙事给出买入结论。

### Step 8：形成决策并落盘

使用同目录 `template.md`。结论必须回答：

1. 当前属于低估、合理还是高估；
2. 动作为买入、观望还是规避；
3. 估值区间与相对现价空间；
4. **如果今天这是一笔现金，我会买它吗**；
5. 哪些事实会改变结论；
6. 后续 3–5 个监控指标。

默认文件名：`YYYY-MM-DD-HHmm-<标的>-机构级决策研报.md`。

在 llmwiki 中，库外原始材料必须先保存到 `sources/`，正式研报放入合适的 `wiki/` 页面类型，并按项目规则更新 `index.md` 与 `log.md`。其他项目保存到用户指定位置；未指定时使用当前项目合理的研究输出目录。

## 完成前质量门

逐项检查，任一失败都不能声称报告完成：

- [ ] 标的身份、市场、币种和截止时间明确；
- [ ] 关键数字均有来源、时间和口径；
- [ ] 五年历史与最新季度来自适用的法定披露；
- [ ] “事实”“我的判断”“未获取到”使用正确；
- [ ] 来源冲突已解释；
- [ ] 多头与空头论据均有证据和失效条件；
- [ ] 相对估值、DCF、行业特定估值均已完成；
- [ ] DCF 假设、敏感性分析和反向估值完整；
- [ ] 冰冰小美风险、流动性、情绪和动作语言完整；
- [ ] 结论回答现金决策并列出监控指标；
- [ ] Markdown 已按时间戳保存；
- [ ] 链接、计算、编码和文件路径已验证。
