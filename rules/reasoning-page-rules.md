## Reasoning 页面生成规则

当整理 reasoning 页面时，不能直接总结原文，也不能只做线性流程图。必须按以下步骤处理：

### 1. 识别核心命题

先用一句话说明作者真正想证明什么。

格式：

> 核心命题：作者试图证明「……」。

### 2. 拆解推导链

必须将原文拆成以下结构：

- 背景事实：作者用来作为起点的现实条件、政策、事件、数据或市场状态。
- 关键变量：推导中真正发生变化的变量，例如流动性、情绪、估值、供需、产业趋势、政策预期。
- 作用机制：变量之间如何发生影响，不能只写“导致”，必须说明传导路径。
- 中介环节：从原因到结果之间经过的市场行为或产业行为。
- 推导结论：作者最终得出的判断。
- 验证信号：后续可以观察什么来确认推导是否成立。
- 失效条件：什么情况出现时，该推导不成立。

### 3. 强制生成推导链表

每条推导必须使用以下格式：

| 层级 | 内容 | 推导关系 | 可信度 | 观察指标 |
|---|---|---|---|---|
| 背景事实 |  | 作为推导起点 | 高/中/低 |  |
| 关键变量 |  | 受到背景事实影响 | 高/中/低 |  |
| 作用机制 |  | 解释变量如何传导 | 高/中/低 |  |
| 中介环节 |  | 连接机制与结果 | 高/中/低 |  |
| 结论 |  | 推导结果 | 高/中/低 |  |

### 4. 强制绘制 Mermaid 推导图

图不能只画线性流程，必须至少包含以下分区：

- 背景事实
- 关键变量
- 作用机制
- 市场行为 / 产业行为
- 结论
- 验证信号
- 失效条件

推荐 Mermaid 模板：

```mermaid
flowchart TD

subgraph A[背景事实层]
A1[事实1]
A2[事实2]
A3[事实3]
end

subgraph B[关键变量层]
B1[变量1]
B2[变量2]
end

subgraph C[作用机制层]
C1[机制1：说明传导逻辑]
C2[机制2：说明传导逻辑]
end

subgraph D[行为层]
D1[市场行为]
D2[产业行为]
D3[政策行为]
end

subgraph E[结论层]
E1[阶段性结论]
E2[最终判断]
end

subgraph F[验证与失效层]
F1[验证信号]
F2[失效条件]
end

A1 --> B1
A2 --> B2
B1 --> C1
B2 --> C2
C1 --> D1
C2 --> D2
D1 --> E1
D2 --> E2
E2 --> F1
F2 -. 反证 .-> E2
