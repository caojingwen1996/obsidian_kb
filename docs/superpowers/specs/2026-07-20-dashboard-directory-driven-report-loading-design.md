# 看板目录驱动研报加载设计

## 背景

`sources/automations/` 已按冰冰小美三类产业重新整理：

```text
sources/automations/
  战略资源/
    电解铝/
    铜/
  新兴产业/
    商业航天/
    机器人/
    算力/
  支柱产业/
    电网/
```

现有 `tools/a-share-market-dashboard/scripts/build.mjs` 仍分别硬编码旧的 `商业航天`、`电网产业` 等路径，并通过公司名称推断标签。目录迁移后，这套逻辑会遗漏嵌套文件、生成旧链接，并使目录分类与看板标签发生漂移。

## 目标

让产业看板以当前目录结构作为研报发现和分类的唯一来源：

1. 固定识别 `战略资源`、`新兴产业`、`支柱产业` 三个产业根目录。
2. 各产业根目录的第一级子目录名直接成为筛选标签。
3. 递归加载子目录内的 HTML 文件。
4. 产业根目录中的 HTML 只在“全部”筛选下展示。
5. 文件移动或新增子目录后，重新构建即可自动更新看板，无需修改关键词规则。

## 非目标

- 不迁移、删除或重命名用户已调整的自动化文件。
- 不改变温度计、估值、数据代理或市场数据逻辑。
- 不从 Markdown、Excel、TXT 或其他非 HTML 文件生成研报卡片。
- 不根据公司名称、标题关键词或证券代码猜测产业分类。
- 不为二级以下目录创建额外筛选层级。

## 目录发现规则

### 产业根目录

构建脚本只扫描：

| 看板页面 | 根目录 |
|---|---|
| 战略 | `sources/automations/战略资源/` |
| 新兴 | `sources/automations/新兴产业/` |
| 支柱 | `sources/automations/支柱产业/` |

### 筛选标签

- 每个根目录的直接子目录都生成一个筛选标签，即使该子目录当前没有 HTML。
- 标签显示文本和过滤值都使用目录原名，例如 `商业航天`、`算力`、`电网`。
- 标签顺序使用中文目录名排序，`全部` 始终排在第一位。
- 二级及更深目录中的 HTML 继承其第一级子目录标签。例如：

```text
新兴产业/商业航天/产业进度动态监测/example.html
```

仍归入 `商业航天`。

### 根目录文件

- 产业根目录直属的 HTML 没有细分标签。
- 这类文件只在“全部”筛选下可见。
- 它们不自动归入“其他”“高端制造”或任何旧标签。

## 文件展示规则

### 产业研报区域

- 文件名包含 `完整分析报告` 的 HTML 作为产业级报告，渲染到顶部“产业研报”区域。
- 一个产业页面可以有多个产业级报告。
- 产业级报告继承所在第一级子目录标签；根目录产业级报告只在“全部”下显示。
- 产业级报告不重复出现在下方日期信息流中。

### 日期信息流

- 其他 HTML 作为普通研报或资金面卡片进入日期信息流。
- 文件名包含 `资金面` 时类型为“资金面”，否则为“研报”。
- 标题继续从文件名去除日期、时间和 `.html` 后缀得到。
- 卡片链接和来源说明使用相对于仓库根目录的完整新路径。

### 计数

- 页面日期行计数只统计日期信息流中的 HTML 卡片。
- 顶部产业级完整分析报告不计入该数量，保持现有界面语义。

## 构建结构

`build.mjs` 使用一个通用扫描流程替代四套硬编码加载函数：

1. 读取产业根目录的直接子目录，生成标签。
2. 递归枚举根目录内所有 HTML。
3. 为每个文件记录相对路径、第一层目录标签、标题、类型和日期。
4. 将 `完整分析报告` 分配到产业研报集合，其余文件分配到信息流集合。
5. 分别渲染动态筛选标签、产业研报、信息流和计数。

模板为三类产业各保留一组占位符：

- 筛选标签；
- 产业研报区域；
- 普通报告信息流；
- 普通报告数量。

旧的 `COMMERCIAL_SPACE_REPORTS`、`ELECTRIC_GRID_REPORTS` 等细分产业占位符和对应加载函数删除。

## 异常处理

- 三个产业根目录缺失时构建失败，并指出缺失路径，避免静默生成空看板。
- 空子目录仍生成标签，但选择后显示现有“没有匹配的研报项”提示。
- 无法读取文件或目录时构建失败，不吞掉错误。
- HTML 文件名和目录名在输出前继续进行 HTML 转义。

## 测试方案

采用测试驱动开发：

1. 先修改构建测试，使其要求新目录链接和动态标签，并确认在旧实现上失败。
2. 失败测试覆盖：
   - `电解铝`、`铜`、`商业航天`、`机器人`、`算力`、`电网` 标签；
   - 新嵌套路径生成正确链接；
   - 算力完整分析报告出现在新兴产业研报区域；
   - 商业航天和电网完整分析报告使用新路径；
   - 支柱产业根目录 HTML 只带空过滤值；
   - 旧的根级 `商业航天`、`电网产业` 链接不再出现；
   - 完整分析报告不重复进入日期信息流；
   - 三个页面的计数与实际普通 HTML 数一致。
3. 编写最小实现使新测试通过。
4. 运行完整看板测试套件并重新构建独立 HTML。
5. 检查生成物无未替换占位符、无旧目录链接、无中文乱码。

## 验收标准

- 新增一个产业一级子目录和 HTML 后，不修改代码即可在重新构建后出现相应标签与卡片。
- 移动现有 HTML 后，生成链接完全对应新路径。
- 截图中的“战略 / 新兴 / 支柱”导航保持不变。
- 细分筛选标签由目录生成，不再依赖标题关键词。
- `算力产业完整分析报告` 能在“新兴 → 算力”筛选下显示。
- 当前看板其他功能和全部既有测试保持通过。
