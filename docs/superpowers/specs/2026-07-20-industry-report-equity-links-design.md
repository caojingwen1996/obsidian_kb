# 产业分析报告链接个股研报设计

## 目标

在产业分析报告的“产业链公司映射”中，将已经存在个股研报的公司名称变为可点击链接，使读者能从产业判断直接进入对应标的研究。

本次先更新商业航天产业完整分析报告，并将同一规则固化到产业分析 Skill、模板和渲染验证中，供以后所有产业报告复用。

## 当前文件关系

商业航天产业报告的可维护源文件是：

```text
wiki/queries/2026-07-17-商业航天产业完整分析报告.md
```

由该 Markdown 生成并保持同步的两个阅读版是：

```text
wiki/queries/2026-07-17-商业航天产业完整分析报告.html
sources/automations/商业航天每日跟踪/商业航天产业完整分析报告.html
```

个股 Markdown 权威研报位于扁平目录：

```text
workbench/targets/
```

个股 HTML 阅读版保留在所属自动化来源目录，例如：

```text
sources/automations/商业航天每日跟踪/
```

## 链接展示

- 不增加新表格列，直接把“公司 / 主体”或“公司”单元格中的公司名称变为链接。
- 同一单元格包含多家公司时，每家公司分别链接。例如“航天电器、国博电子、新雷能等”中的三个公司分别可点击，“等”保持普通文本。
- 未上市主体、科研院所和没有研报的公司保持普通文本，不制造空链接。
- 链接只提供研究入口，不改变产业报告中的业务纯度、证据状态、产业判断或投资边界。

## 链接选择规则

对每个公司按以下顺序选择目标：

1. 如果存在对应 HTML 研报，链接 HTML 阅读版；
2. 如果没有 HTML，但存在 `workbench/targets/` 下的权威 Markdown，链接 Markdown；
3. 两者都不存在时，保持普通文本。

匹配时以证券代码为第一标识，公司全称和简称只用于复核。不得仅凭模糊公司名猜测目标文件。

## Markdown 与 HTML 处理

### Markdown 源文件

有 HTML 时，公司名称使用指向具体 HTML 文件的 Obsidian 链接，例如：

```markdown
[[sources/automations/商业航天每日跟踪/2026-07-17-1616-中国卫星-机构级决策研报.html|中国卫星]]
```

只有 Markdown 时，使用指向权威研报的 Obsidian 链接，例如：

```markdown
[[workbench/targets/YYYY-MM-DD-HHmm-标的-机构级决策研报|标的]]
```

### HTML 渲染

产业报告渲染器继续把显式 `.html` 目标转换为相对 HTML 链接。

对于 `workbench/targets/` 下的 Markdown 回退链接，渲染器不得假设同目录存在同名 HTML；应保留 `.md` 目标，避免生成不存在的 `workbench/targets/*.html`。

其他 Wiki Markdown 双链仍维持当前转换为同路径 HTML 的行为，不扩大本次修改范围。

## 未来生成契约

`bbxm-industry-analysis` 和产业报告模板增加以下规则：

- 生成“产业链公司映射”时，先按证券代码检查是否已有对应标的研报；
- 按“HTML 优先、Markdown 回退、无研报不链接”的顺序写入公司名称；
- 多公司单元格逐家公司处理；
- 不得因为存在个股研报而把产业分析改写成个股推荐；
- HTML 导出前验证所有显式研报链接的目标文件真实存在。

## 验证

### 当前报告验证

1. Markdown 中存在 25 处公司研报链接，覆盖 18 个唯一标的；
2. 所有链接目标文件均存在；
3. 两份 HTML 均由同一 Markdown 重新生成，正文区域一致；
4. HTML 中 25 处链接均为相对路径并可解析；
5. 报告的 0—7 章、产业链公司映射、目录和中文编码保持正常。

### 渲染器回归验证

产业报告渲染测试增加两类用例：

- 显式 HTML 研报链接保持指向 HTML；
- `workbench/targets/` Markdown 回退链接保持指向 Markdown，不产生不存在的同名 HTML。

### 合同验证

项目技能验证检查 Skill 和模板都包含：

```text
HTML 优先
Markdown 回退
无研报不链接
```

## 失败处理

- 证券代码与公司名无法唯一对应时，不添加链接并标记待核验；
- 目标文件不存在时，渲染或验证失败，不输出带失效链接的 HTML；
- 两份 HTML 生成结果不一致时，以 Markdown 为唯一来源重新生成，不手工分别修补；
- 链接加入后若破坏 Markdown 表格结构，停止提交并修复单元格写法。

## 非目标

- 不移动商业航天产业报告现有 Markdown 或 HTML 路径；
- 不修改产业报告结论、证据、业务纯度或公司名单；
- 不为缺少研报的公司生成新研报；
- 不让渲染器对普通公司名称执行自动模糊匹配；
- 不批量改写其他既有产业报告；新规则只约束后续生成和更新。
