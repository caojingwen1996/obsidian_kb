# 冰冰小美“实战操作”219—250 篇批量整理设计

## 目标

将 `sources/manual/体系及案例分析/三、实战操作/1、套利机会与风险提示/` 中第 219—250 篇原始文章蒸馏进现有知识库。整理结果应保留来源边界，使用冰冰小美体系原有术语，并把文章结论转化为可复用的概念、观点、复盘方法、风险规则和操作边界。

## 范围

- 来源范围：第 219—250 篇，共 32 篇。
- 原始资料保持不变，不在 `sources/` 中写入深度分析。
- 优先更新现有正式页面；只有出现边界清晰、可长期复用且现有页面无法承载的新知识单元时，才创建新页面。
- 不处理工作区中与本批次无关的现有修改。

## 页面路由

1. 先检查 `.manifest.json`、`index.md`、`hot.md`、`log.md` 和相关正式页面。
2. 主题级结构优先进入 `wiki/topics/冰冰小美-情绪体系操作篇.md`。
3. 复盘方法优先进入 `wiki/concepts/冰冰小美-framework-复盘.md`。
4. 稳定术语和方法进入相应 Concept Page；带有明确作者立场且不宜抽象成通用定义的内容进入 View Page；完整因果链进入 Reasoning Page。
5. 已存在的套利、情绪一致性、围魏救赵、轮动亏钱效应等页面优先合并新来源，不重复建页。
6. 案例中的个股和市场节点只用于解释当时机制，不写成当前行情结论。

## 写作与来源边界

- 明确区分作者原文、整理者归纳和待验证判断。
- 跨文章归纳或抽象出的结论使用 `^[inferred]`；来源表达冲突或不完整时使用 `^[ambiguous]`。
- 所有新增断言必须能追踪到本批次具体来源。
- 正文和“相关页面”统一使用 Obsidian 双链。
- 操作含义尽量落到 `买入 / 卖出 / 减仓 / 等待 / 观察 / 复盘 / 回避`。

## 特殊文件维护

- `.manifest.json`：记录每篇来源的哈希、时间、更新页面和新增页面。
- `index.md`：仅在新增页面或导航结构发生变化时更新。
- `hot.md`：记录每批次形成的概念性变化，保留最近活动结构。
- `log.md`：每批追加 ingest 记录，并在最终记录总体覆盖情况。

1. 每篇来源至少映射到一张正式页面，或明确记录为何不产生独立更新。

4. 中文文件保持 UTF-8，修改文件不存在典型乱码字符。
5. `.manifest.json` 可解析，内部链接与目标路径一致。
6. 最终生成 219—250 的“来源 → 正式页面”覆盖清单。
