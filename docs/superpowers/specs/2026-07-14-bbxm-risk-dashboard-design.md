# 冰冰小美风险提示工具设计

## 目标

在当前项目中新增一个本地运行的风险提示网页工具。工具固定读取项目内 Excel 风险记录，自动联网获取对应区间的上证指数日线收盘数据，并用接近用户参考图的黑底界面展示指数曲线和风险提示节点。

工具用于回看“冰冰小美风险提示出现时，上证指数处于什么位置”，不自动生成新的风险判断，也不把历史提示解释为买卖信号。

## 交付范围

- 一个可在本机启动的网页工具。
- 一份固定路径的 Excel 数据模板。
- 上证指数行情自动更新与本地缓存。
- 风险节点悬浮详情和明确的数据错误提示。
- 启动与 Excel 维护说明。

初版不包含账户、数据库、在线部署、自动编辑 Excel、风险原因分类、买卖建议、右侧统计卡片或其他指数切换。

## 技术方案

采用轻量的 Python 本地服务承担读取文件、获取行情和整理数据，普通 HTML、CSS、JavaScript 页面承载图表和交互。

选择这一方案的原因：

1. 浏览器不能在无授权的情况下直接读取固定本地 Excel，Python 服务可以稳定完成读取。
2. 行情请求由本地服务发起，可以避开浏览器跨域限制，并统一处理缓存和失败状态。
3. 普通网页更容易精确还原参考图，不需要引入完整前端工程。
4. 相比 Streamlit，布局和样式控制更精确；相比 React 加独立后端，初版更简单。

## 文件边界

工具使用独立目录，避免与 `wiki/` 正式知识页和现有自动化混合。建议结构：

```text
tools/bbxm-risk-dashboard/
  app.py
  requirements.txt
  README.md
  data/
    冰冰小美风险提示.xlsx
    shanghai-index-cache.json
  static/
    index.html
    styles.css
    app.js
  tests/
```

缓存文件属于运行产物，不作为人工维护的数据源。

## Excel 数据契约

固定读取：

```text
tools/bbxm-risk-dashboard/data/冰冰小美风险提示.xlsx
```

工作表第一行必须包含以下三列，列名完全一致：

| 列名 | 类型 | 规则 |
|---|---|---|
| 日期 | 日期 | 支持 Excel 日期或可解析的 `YYYY-MM-DD` 文本 |
| 当日累计风险提示次数 | 非负整数 | 表示该自然日累计提示次数 |
| 风险原因 | 文本 | 允许换行；页面悬浮详情保留原始文本 |

初版按“一行对应一个自然日”处理。若日期重复，启动时将同日次数相加、风险原因按原行顺序合并，并在页面状态区提示存在合并记录。空日期、负数、非整数次数等错误应指出具体 Excel 行号，不静默修正。

Excel 最早和最晚有效日期决定图表默认时间区间。为了让末端风险节点有行情背景，行情请求结束日期至少延伸到当前可获得的最新交易日；图表首次显示完整风险记录区间，并允许用户缩放查看后续行情。

## 上证指数数据

- 标的固定为上证指数。
- 使用日线实际收盘点位，不做首日归一化，不使用累计涨跌幅替代点位。
- 启动时自动联网刷新行情；成功后写入本地缓存，并记录行情最后交易日和刷新时间。
- 网络或数据源失败时，如有缓存则继续展示缓存，并清楚标明“缓存数据”和最后更新时间；没有缓存时显示可操作的错误信息，不绘制伪造曲线。
- 后续实施阶段选择无需用户密钥、可稳定获取上证指数历史日线的数据源，并通过独立适配函数隔离，便于数据源变化时替换。

## 日期对齐规则

风险提示以自然日保存，指数只有交易日数据：

- 交易日风险提示：节点落在当日收盘点位。
- 非交易日风险提示：节点落在此前最近一个交易日的收盘点位。
- 悬浮信息始终显示 Excel 中的原始提示日期，并注明用于定位的行情日期，避免把周末提示误写成交易日提示。
- Excel 中早于可用行情范围的记录作为数据错误展示，不擅自丢弃。

## 界面设计

整体风格直接沿用参考图，不额外引入新的视觉方向：

- 黑色页面背景。
- 主图区域使用细白色边框。
- 上证指数曲线使用低饱和灰色细线。
- 网格线使用低对比度灰色虚线。
- 风险提示使用红色实心圆点，并增加细描边，确保节点在曲线上清晰可辨。
- 横轴显示日期，纵轴显示上证指数实际收盘点位。
- 标题为“冰冰小美风险提示记录”，副标题说明数据来自固定 Excel，上证指数为联网数据。

风险圆点大小随“当日累计风险提示次数”分级变化，但设置最小和最大尺寸，避免低次数看不见或高次数遮挡曲线。颜色不承担次数差异，避免用户把颜色深浅误解为风险等级。

鼠标悬浮或键盘聚焦风险节点时显示：

- 风险提示日期；
- 当日累计风险提示次数；
- 风险原因；
- 对齐的行情日期；
- 对齐日上证指数收盘点位。

图表支持时间轴缩放和横向拖动。移动端以查看为主，详情通过点击节点打开。

## 右侧区域

保留与参考图比例接近的右侧竖向区域和边框，但初版内部保持空白，不展示统计指标、说明卡片或占位文字。该区域只作为未来扩展空间，不影响主图读取和交互。

窄屏下优先保证主图可读性：右侧空白区域隐藏，不强制压缩主图。

## 数据流

1. 用户启动本地服务。
2. 服务读取固定 Excel，验证列名和每一行数据。
3. 服务根据 Excel 日期范围读取行情缓存并尝试联网补全最新数据。
4. 服务聚合重复日期并将风险日期对齐到指数交易日。
5. 浏览器请求整理后的统一数据结构。
6. 页面绘制指数曲线和风险节点，并展示数据来源状态。

页面不直接解析 Excel，也不直接访问行情源；这样可以保持前端简单，并让验证、缓存和失败处理只有一个实现位置。

## 状态与错误处理

- Excel 缺失：显示固定路径和放置说明。
- 缺少必需列：列出缺失列和实际检测到的列。
- 单元格格式错误：列出 Excel 行号、列名和原始值。
- Excel 暂时被占用：提示用户关闭占用文件后重试。
- 行情联网失败：优先回退缓存，并显示缓存时间；无缓存则停止绘图。
- 行情区间不完整：显示缺失范围，不把缺失解释为停牌或无风险。
- 页面请求失败：显示重试按钮，不保留会误导用户的旧状态。

错误信息使用正常中文，不显示不必要的底层调用栈；详细异常保留在本地服务输出中供排查。

## 测试与验证

自动测试至少覆盖：

1. 正常三列 Excel 的读取。
2. Excel 日期值和文本日期的统一解析。
3. 缺列、空日期、负数和非整数次数的报错定位。
4. 重复日期的次数和原因合并。
5. 交易日与非交易日风险提示的对齐。
6. 行情刷新成功、缓存回退和无缓存失败三种路径。
7. 数据接口返回值包含图表所需字段。

人工验收至少覆盖：

1. 桌面宽屏下整体布局与参考图风格一致。
2. 右侧区域存在且为空。
3. 纵轴显示实际收盘点位。
4. 风险点大小能反映次数且不遮挡主要曲线。
5. 悬浮和键盘聚焦均能读取完整风险原因。
6. 图表缩放、拖动和窄屏查看可用。
7. 网络断开后缓存状态表达真实、明确。

## 完成标准

- 按说明启动后，工具无需用户手动上传文件即可读取固定 Excel。
- 能自动获取并缓存上证指数数据，失败时按设计回退或报错。
- 能在实际收盘点位曲线上正确定位 Excel 中的每个风险提示日。
- 风险次数和原因可通过节点交互查看。
- 页面视觉风格与参考图一致，右侧预留区保持空白。
- 测试通过，README 包含安装、启动、Excel 更新和常见错误说明。
- 不修改或提交工作区内与本工具无关的既有改动。
