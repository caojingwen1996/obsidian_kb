# 资金面分析 Skill 设计

## 背景

现有 `bbxm-three-factor-analysis` 已包含流动性检查，但没有独立的资金面分析入口。用户希望参考华泰证券 2021 年《A股择时之资金面指标测试》的资金来源分类，以及文章中的 F3/F5/F7 日频监测方法，形成可独立调用、也可被其他分析流程消费的资金面 Skill。

## 目标

- 分析市场、指数、板块或个股的资金来源、方向、力度、持续性和分歧。
- 以华泰研报的多资金来源视角作为全景框架，以 F3/F5/F7 作为日频代理面板。
- 用价格、广度、筹码供给和时间连续性验证资金信号。
- 分别为新增资金和已有持仓输出条件化的 `buy / sell / reduce / wait / observe / review`。
- 对数据缺口、重复统计和身份代理作显式披露。

## 非目标

- 不复刻华泰研报的完整回测模型。
- 不把 F3/F5/F7 宣称为论文原生三因子。
- 不内置依赖令牌的 Tushare 或 AKShare 采集脚本。
- 不替代公司基本面、产业、三要素或风险方向分析。

## 文件结构

```text
.agents/skills/fund-flow-analysis/
├── SKILL.md
├── template.md
├── references/
│   ├── fund-source-framework.md
│   └── indicator-boundaries.md
├── evals/
│   └── evals.json
└── agents/
    └── openai.yaml
```

`SKILL.md` 定义触发、边界、数据契约、分析流程、状态词和行动映射。`template.md` 是唯一输出结构。两个 reference 文件分别保存资金来源分类和指标口径边界。`evals.json` 覆盖典型与近似误触发场景。`openai.yaml` 提供项目内展示元数据。

## 输入契约

每次分析先明确：对象、层级、市场、分析截止时间、决策问题、已有持仓事实、可用数据和缺失数据。默认比较窗口为当日、近 5 个交易日和近 20 个交易日；对象或数据不支持时，必须解释替代窗口。

## 分析架构

### 1. 数据审计

记录来源、时点、频率、口径、覆盖范围和缺口。区分真实账户数据、统计代理和经验推断。

### 2. 资金来源全景

检查外资与跨境资金、杠杆资金、公募与 ETF、产业资本、保险与长期资金、私募与信托、券商资管、个人资金，以及股票融资、减持、解禁等筹码供给。并把观察到的流入归类为场外真实增量、存量迁移或减量报团。

### 3. F3/F5/F7 日频面板

- F3：融资净买入，代理杠杆资金的边际变化。
- F5：特大单净买入，代理大额主动交易，不确认账户身份。
- F7：ETF 估算净申赎，反映 ETF 份额变化，不确认申购者身份。

三项分别评价方向、强度、持续性、稳定性和数据质量，不直接相加为全市场净流入。

### 4. 交叉验证

资金流必须与价格响应、成交与换手、上涨广度、相对强弱、筹码供给和连续性共同解释。发现信号重叠、价格背离或数据范围不一致时，降低置信度。

### 5. 状态与行动

资金状态只使用：`共振流入 / 结构性流入 / 多空分歧 / 结构性流出 / 共振流出 / 证据不足`。

行动只使用：`buy / sell / reduce / wait / observe / review`。新增资金和已有持仓分开判断。`buy` 必须同时满足资金持续、价格确认、非极端拥挤和风险可承受；缺少关键数据时不得给 `buy`，默认降级为 `observe`，已有持仓事实不足时为 `review`。

仓位只给条件化区间，不给无依据的精确比例。每个行动必须包含触发条件、维持条件、取消条件、下一验证点和失效条件。

## 输出契约

最终输出严格使用 `template.md`，保留全部章节。字段缺失时填写“待核验”“证据不足”或“不适用”，不得删除章节或缩写成自由格式。

## 验证

测试覆盖：市场大跌中 ETF 托底但杠杆撤离、板块资金分歧、个股特大单流入但价格走弱、关键数据缺失，以及只问风险方向或完整公司估值等不应触发场景。校验重点是模板完整、代理边界、禁止简单求和、三窗口比较、状态限定和条件化行动。
