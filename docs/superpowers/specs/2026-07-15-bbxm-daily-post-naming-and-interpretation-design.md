# 冰冰小美每日任务：原帖命名与逐帖解读设计

## 目标

优化 `.agents/automations/bbxm_daliy_brief.md`，使每日抓取结果具备稳定、可读的一对一文件关系：原帖文件名体现实际发布时间和核心观点，每篇原帖旁生成一份只包含冰冰小美体系解读的文件。

## 文件契约

每篇帖子生成两个 Markdown 文件：

```text
HHMMSS_核心观点.md
HHMMSS_核心观点_解读.md
```

- `HHMMSS` 使用帖子元数据中的实际发布时间，不使用抓取时间。
- `核心观点` 必须在通读帖子完整正文后提炼，不得直接截取首句或沿用雪球页面自动标题。
- 核心观点应简洁、准确，优先写成“对象 + 判断”或“主题 + 结论”。
- 文件名必须清理 Windows 禁用字符，并避免尾部空格或句点。
- 同一作者同一秒产生相同核心观点时，后续文件追加 `_2`、`_3`；原帖与解读文件使用相同后缀。

## 解读内容契约

解读文件只允许包含以下结构：

```markdown
# HHMMSS 核心观点 解读

## 冰冰小美体系解读

<解读正文>
```

解读正文必须：

1. 基于对应原帖完整正文；
2. 基于任务要求读取的冰冰小美体系知识页；
3. 使用体系自身的概念和判断方式；
4. 对证据不足的部分明确标注“待验证”。

不得增加核心观点、观点依据、风险节点、行为翻译、相关页面或其他独立章节；不得联网补充外部行情、新闻或数据；不得把模型常识写成作者观点。

## 同日重跑

- 不覆盖或重命名已保存的原帖和解读文件。
- 新增帖子按新命名规则生成原帖与解读文件。
- 已有原帖缺少对应解读时，允许补生成 `_解读.md`。
- 判断是否为同一帖子继续使用 `state.json`、稳定来源标识或 URL，不依赖文件名。

## 任务文件修改范围

只修改 `.agents/automations/bbxm_daliy_brief.md`：

1. 在抓取与重跑规则后增加“原帖命名与逐帖解读”强制契约；
2. 在完整性门禁中要求每个已保存帖子都有对应解读文件；
3. 在收尾报告中增加逐帖解读生成数量和缺失项。

不修改抓取技能脚本，不批量重命名历史文件，不改动 `summary.md` 既有结构。

## 验证标准

- 任务文件明确出现两个目标文件名模板。
- 任务文件明确禁止使用首句命名。
- 任务文件明确解读文件只有“冰冰小美体系解读”一个正文章节。
- 同日重跑规则覆盖新增帖子、缺失解读补生成和不覆盖已有文件。
- 差异只包含设计文档与目标任务文件。
