# 资金面分析 SVG 流程图设计

## 目标

把 `.agents/skills/fund-flow-analysis/SKILL.md` 中的完整分析流程绘制为一张可在 Obsidian、网页和打印材料中使用的 SVG，并同时保留可维护的 Mermaid 源文件。

## 范围

- 展示输入契约与完整八步分析流程。
- 明确资金来源全景、三种流入机制和 F3/F5/F7 只是不同证据切面。
- 展示供给与退出、价格与广度等交叉验证。
- 只使用技能限定的六类资金状态和六类行动词。
- 分开呈现新增资金与已有持仓的行动映射。
- 不加入当前市场数据、个股结论或未经验证的账户身份判断。

## 版式

采用纵向审计漏斗：

1. 顶部为输入契约；
2. 进入数据完整性审计；
3. 中段左侧呈现资金来源全景与三种流入机制；
4. 中段右侧呈现 F3、F5、F7 日频面板及代理边界；
5. 两路证据汇入供给与退出路径检查；
6. 再经过价格、成交、广度、相对强弱、连续性和筹码供给交叉验证；
7. 输出六类资金状态；
8. 末端分为新增资金和已有持仓两条行动路径，并共同回到下一验证点与失效条件。

## 视觉规则

- 使用浅色专业主题和竖向主阅读路径，适合 Obsidian 窄栏及打印。
- 蓝色表示数据与资金证据，橙色表示验证与风险，紫色表示状态，绿色表示行动。
- 每条关键连接带文字标签；颜色只辅助分层，不单独承载语义。
- 节点使用短句和换行控制密度，避免把报告模板全文塞入流程图。
- SVG 应包含中文字体回退，并保持文字可搜索、可缩放。

## 输出文件

- `.agents/skills/fund-flow-analysis/assets/资金面分析完整流程.mmd`
- `.agents/skills/fund-flow-analysis/assets/资金面分析完整流程.svg`
- `log.md` 追加本次可视化维护记录。

## 验证标准

- 八步编号齐全且顺序正确。
- F3/F5/F7 不被机械相加，也不被写成账户身份事实。
- 六类资金状态完整且没有新增自造状态。
- 新增资金与已有持仓动作分开。
- SVG 能正常解析，包含标题、中文文本和可用的 `viewBox`。
- 检查典型乱码字符与替换字符，不修改任何无关文件。
