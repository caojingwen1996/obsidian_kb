# 看板树形导航与更新日志设计

## 背景

当前 A 股看板左侧通过“温度计 / 产业”顶部切换器在两套导航之间切换。用户希望取消这层切换器，将“温度计”“产业”“更新日志”统一为左侧一级树节点，并新增一个参考 AIHOT 更新日志界面的独立日志页面。

## 目标

1. 将左侧导航改为单一树形结构。
2. “温度计”“产业”“更新日志”成为三个一级节点。
3. 当前一级分组展开时，其他一级分组自动收起。
4. 新增独立更新日志页面，按日期和时间展示看板自身的功能变化。
5. 保持现有温度计、产业筛选、研报打开和本地数据代理功能不变。

## 非目标

- 不读取或展示仓库根目录 `log.md`。
- 不展示知识库 ingest、个股研报生成、雪球抓取等非看板更新。
- 不新增日志搜索、筛选、分页、编辑器或后台管理。
- 不保存树节点的历史展开状态；刷新后仍从默认页面开始。
- 不改变产业报告目录扫描规则。

## 左侧树形导航

### 信息架构

```text
温度计
├─ 市场总览
├─ 位置分位
├─ 估值与 ERP
├─ 情绪温度
├─ 评分规则
└─ 数据审计

产业
├─ 战略
├─ 新兴
└─ 支柱

更新日志
```

### 节点行为

- “温度计”和“产业”是可展开的一级分组按钮。
- “更新日志”是没有子节点的一级叶子按钮，点击后直接打开日志页。
- 页面首次加载时展开“温度计”，选中“市场总览”。
- 点击另一个可展开一级分组时：展开目标分组、收起其他分组，并打开目标分组最近一次选中的子页面；若尚无记录，则打开该分组第一个子页面。
- 点击已经展开的一级分组时保持展开，不将当前导航折叠为空，避免用户失去当前页面位置。
- 点击“更新日志”时收起“温度计”和“产业”子树，并选中“更新日志”。
- 从更新日志返回某个一级分组时，恢复该分组本次页面生命周期内最近一次选择的子页面。

### 视觉与可访问性

- 取消现有顶部 `.shell-switcher`。
- 一级分组使用文本和方向箭头共同表达展开状态，不只依赖颜色。
- 一级按钮设置 `aria-expanded` 和 `aria-controls`；子树使用对应 `id`。
- 叶子页面继续使用现有蓝色选中背景，并设置 `aria-current="page"`。
- 一级节点、叶子节点均保持清晰的悬停、聚焦和按下状态，键盘可依次访问。
- 桌面端保留深色侧栏；移动端继续通过“打开导航”按钮展开侧栏内容。

## 更新日志页面

### 页面结构

日志页使用参考界面的信息层级，并适配当前看板配色：

1. 页头小标题：`CHANGELOG`。
2. 主标题：`更新日志`。
3. 简介：`最近发生了什么——新功能、调整与修复，都写在这里。`
4. 日志正文按日期从新到旧分组。
5. 每个日期组显示中文日期和星期。
6. 每条记录左侧显示时间与绿色类型标识，右侧显示标题、摘要和可选要点列表。

内容区采用较窄的可读宽度，日期组之间保留明显垂直间距，分隔线和时间线使用低对比度边框，避免与正文竞争。

### 首批记录

首版回填 2026-07-20 的三项看板更新：

1. 产业研报改为目录驱动自动加载。
2. 修复本地服务器无法打开产业研报。
3. 产业研报链接改为在新标签页打开。

记录时间使用看板专用日志数据中明确填写的时间；若无法确认精确发生时间，允许省略时间，不从文件修改时间或当前时间推测。

### 页面状态

- 进入更新日志页时隐藏行情刷新按钮、示例数据开关、市场数据提示和温度计状态页脚。
- 顶部页面标题显示“更新日志”，顶部英文眉题显示 `CHANGELOG`。
- 日志页不触发行情刷新，也不影响已加载的市场数据状态。

## 日志数据

新增看板专用静态数据文件，建议使用 `src/changelog.json`，由构建脚本读取并渲染到独立 HTML 产物。

每条记录包含：

```json
{
  "date": "2026-07-20",
  "weekday": "星期一",
  "time": "",
  "type": "更新",
  "title": "产业研报改为目录自动加载",
  "summary": "简要说明本次变化及用户影响。",
  "details": ["可选要点一", "可选要点二"]
}
```

约束：

- `date`、`weekday`、`type`、`title`、`summary` 必填。
- `time` 和 `details` 可选。
- 构建时按 `date` 降序排列；同日记录保持数据文件中的顺序。
- 所有文本在写入 HTML 前进行转义。
- 数据为空时显示明确的空状态，不生成空白页面。

## 构建与交互改造

### 模板

- `src/index.html` 将两套侧栏面板合并为一个树形导航。
- 新增 `changelog-view` 页面和日志内容占位符。
- 温度计与产业原页面结构保持不变。

### 构建脚本

- `scripts/build.mjs` 读取并校验 `src/changelog.json`。
- 将记录按日期分组，渲染日期标题、时间、类型、标题、摘要和要点。
- 继续生成单文件 `a-share-market-dashboard.html`，不产生运行时外部数据依赖。

### 前端状态

- `src/app.mjs` 将当前 `shell` 概念扩展为三个一级导航域：`thermometer`、`industry`、`changelog`。
- 保留温度计和产业各自最近一次选择的子页面。
- 一级树节点负责切换导航域和展开状态；叶子节点负责切换具体页面。
- 市场控件、提示和页脚只在温度计域显示。

## 响应式设计

- 桌面端维持固定深色侧栏和右侧主内容布局。
- 760px 以下继续使用移动导航开关；树节点和子节点占满可用宽度。
- 日志条目在桌面端使用“时间栏 + 内容栏”两列结构。
- 移动端改为单列：日期、时间和类型位于正文上方，不保留固定宽度时间栏。
- 页面不得出现横向滚动，正文行宽在大屏上保持可读。

## 异常处理

- 日志 JSON 不可读取、不是数组或必填字段缺失时，构建失败并指出具体问题。
- 可选字段缺失时不渲染对应空元素。
- 未知导航域回退到“温度计 → 市场总览”。
- 原有行情接口或产业目录异常处理逻辑不变。

## 测试方案

采用测试驱动开发：

1. 先新增失败测试，要求三个一级节点、树形 ARIA 属性和更新日志页面存在。
2. 验证原 `.shell-switcher` 已移除。
3. 验证默认展开温度计，切换产业后温度计收起，点击更新日志后两个子树都收起。
4. 验证更新日志三条首批记录按日期分组渲染。
5. 验证日志数据中的文本经过 HTML 转义。
6. 验证日志页隐藏市场控件，返回温度计后恢复。
7. 运行既有产业筛选、研报新标签页、本地代理和温度计计算测试。
8. 在桌面宽度与 760px 以下宽度检查导航和日志布局无横向溢出。

## 验收标准

- 左侧不再显示“温度计 / 产业”顶部切换器。
- 左侧一级节点严格为“温度计”“产业”“更新日志”。
- 任意时刻最多只有一个带子节点的一级分组展开。
- 默认进入“温度计 → 市场总览”。
- 更新日志页面视觉层级与参考图一致，并沿用当前看板颜色与字体体系。
- 首批三项更新完整展示，且内容只来自看板专用日志数据。
- 更新日志包含明确的日期、类型、标题和说明。
- 原有温度计、产业看板、研报链接和本地代理功能全部通过回归测试。
