# 冰冰小美风险提示工具启动脚本设计

## 目标

为 `tools/bbxm-risk-dashboard/` 增加一个可直接双击运行的 Windows 批处理脚本，降低首次安装和日常启动成本。

## 文件名与位置

```text
tools/bbxm-risk-dashboard/启动工具.bat
```

脚本必须使用自身所在目录作为工作目录，不能依赖用户从哪个目录双击或调用。

## 启动行为

1. 切换到脚本所在目录。
2. 检查 `.venv\Scripts\python.exe` 是否存在。
3. 如果虚拟环境不存在，使用系统 `python` 创建 `.venv`，并通过虚拟环境中的 Python 安装 `requirements.txt`。
4. 如果虚拟环境已经存在，直接复用，不在每次启动时重复安装依赖。
5. 启动一个延迟两秒的浏览器打开任务，目标为 `http://127.0.0.1:5179`。
6. 在当前批处理窗口中以前台方式运行 `run.py`。
7. 用户关闭窗口或按 `Ctrl+C` 时，Flask 服务随当前进程停止，不留下独立后台服务。

## 错误处理

- 系统找不到 `python`：显示中文提示，说明需要安装 Python 3.11 或更高版本，然后暂停窗口。
- 虚拟环境创建失败：显示失败步骤并暂停。
- 依赖安装失败：显示失败步骤并暂停。
- 服务异常退出：保留错误输出并暂停，避免窗口一闪而过。
- 正常按 `Ctrl+C` 停止后，窗口可以正常结束，不额外清理 Excel 或行情缓存。

## 范围边界

- 不修改现有 Python 服务、Excel 数据契约和页面界面。
- 不注册开机启动，不创建桌面快捷方式，不开放局域网访问。
- 不在后台常驻，不自动结束其他占用 5179 端口的进程。

## 验证

1. 从工具目录外调用脚本，仍能正确定位 `requirements.txt` 和 `run.py`。
2. `.venv` 不存在时能完成首次创建和依赖安装。
3. `.venv` 已存在时跳过安装并启动服务。
4. 服务启动后浏览器自动打开正确地址。
5. 关闭运行窗口后 5179 端口不再监听。
6. 批处理语法检查和现有 15 项 Python 测试全部通过。
