# 商业航天产业每日跟踪自动化设计

## 目标

建立一项与“机器人行业每日信息收集”结构相近、但更适合商业航天慢变量的自动任务。任务每天生成产业日报，同时维护一份滚动进度看板，用于持续观察中国星网组网速度、产业规划与投资增速、细分领域关键技术以及全球竞争份额变化。

本任务服务于冰冰小美体系三要素中的“竞争格局比较优势”判断。它只回答产业方向资格、产业进度和相对优势变化，不把单条产业信息直接转译为买入信号。

## 调度与统计窗口

- 自动任务名称：`商业航天产业每日跟踪`
- 执行时间：每天北京时间 `09:45`
- 目标日期：运行日的前一自然日
- 检索窗口：目标日期 `00:00` 至运行日 `09:30`
- 周末与节假日：照常运行；无 A 股交易时明确说明，并重点覆盖海外进展、政策规划、公告、发射任务和产业投资信息
- 与现有机器人任务错峰运行，避免同时写入仓库追踪文件

## 项目内任务契约

新增项目提示词：

`.agents/automations/commercial_space_daily_brief.md`

自动任务本身只负责：

1. 读取该项目提示词；
2. 计算目标日期和检索窗口；
3. 按提示词完成检索、核验、分析和文件写入；
4. 报告输出路径、覆盖情况、失败数据源和人工跟进项。

项目提示词是检索范围、评级口径和输出格式的唯一来源，避免自动任务配置与仓库内规则发生漂移。

## 输出文件

### 每日产业日报

路径：

`sources/automations/商业航天每日跟踪/YYYY-MM-DD.md`

每份日报包含：

1. 一句话产业结论；
2. 四维产业进度表；
3. 当日最重要的三至五条增量信息；
4. 竞争格局增强、减弱或未变的判断；
5. 最接近订单、收入和利润的环节；
6. 仍处于规划、试验或概念阶段的环节；
7. 后续验证指标与失效条件；
8. 来源覆盖、失败数据源及待人工核验项。

### 滚动产业进度看板

路径：

`sources/automations/商业航天每日跟踪/产业进度看板.md`

看板只更新相对上次发生变化的指标，并为每项指标保存：

- 当前值或当前阶段；
- 上次记录；
- 本次变化；
- 数据日期；
- 来源；
- 可信度与待验证边界。

每日任务先读取既有日报和看板，避免把旧信息重复写成当天新增。

## 四个固定跟踪维度

### 1. 中国星网与卫星组网

重点跟踪：

- 中国星网、卫星互联网国家队星座及千帆等国内大型星座；
- 规划卫星数量、累计入轨数量、阶段完成比例；
- 单次发射卫星数量、月度和季度组网速度；
- 执行发射的火箭型号、发射场、失败或延期情况；
- 批量造星能力、星箭匹配能力和高密度发射能力；
- 卫星实际运营、频率许可、终端和应用进展。

不得把规划数量等同于已形成订单，不得把单次发射成功等同于组网商业闭环完成。

### 2. 产业规划与投资增速

重点跟踪：

- 国家和地方商业航天规划、标准、许可与监管；
- 产业基金、社会融资、IPO、定增和并购；
- 企业资本开支、固定资产投资、扩产计划和项目终止；
- 发射场、火箭总装基地、卫星工厂、测运控和地面基础设施建设；
- 已披露投资额、同比增速、实际完成额和建设进度。

规划金额、签约金额、计划投资、实际投资和形成产能必须分开记录。

### 3. 细分领域核心关键技术

重点跟踪：

- 可重复使用火箭、发动机、回收、检修和复飞；
- 批量造星、柔性产线和组批测试；
- 星间激光通信、相控阵天线、卫星芯片和空间计算；
- 测运控、地面终端、频轨资源和空间安全；
- 关键材料、电子元器件、载荷、电源和热管理。

所有技术进展按以下阶段区分：

`方案或规划 -> 地面试验 -> 飞行试验 -> 成功入轨 -> 成功回收 -> 同一产品复飞 -> 批量交付 -> 规模商用`

不得用“试验成功”替代订单、收入、利润或规模商用验证。

### 4. 全球竞争份额变化

不使用无法核验的单一“全球市场份额”。分别跟踪以下代理指标：

- 全球及主要国家年度发射次数；
- 入轨卫星数量和有效载荷规模；
- 低轨卫星在轨存量与新增量；
- 发射运力、发射服务订单和商业发射收入；
- 卫星制造交付量；
- 卫星通信用户、终端和应用收入；
- 频率、轨道和大型星座部署进度。

不同机构口径不一致时并列呈现，不强行拼接为一个确定比例。

## 竞争格局判断

每日结论使用以下三档：

- `增强`：真实发射、回收复飞、批量交付、订单、资本开支、收入或全球相对份额出现可核验改善；
- `未变`：只有旧规划重复传播、一般性表态或没有改变产业变量的消息；
- `减弱`：延期失败、项目终止、投资放缓、关键技术受阻、需求不及预期或全球相对位置下降。

结论必须说明证据、风险和失效条件，并区分产业方向资格与具体公司交易资格。

## 来源与证据等级

优先顺序：

1. 中国政府部门、国家航天局、中国载人航天工程网、工信部、交易所公告、上市公司正式披露及海外对应监管机构；
2. 航天科技、航天科工及企业官网、发射服务商、卫星运营商和行业协会；
3. 权威媒体和具名访谈；
4. 券商或咨询机构报告；
5. 社交媒体、转述和市场传言。

关键数值至少记录统计日期、统计口径和原始来源。低等级来源只能作为待核实线索，不能单独改变滚动看板结论。

## 失败与不确定性处理

- 某一来源访问失败时，记录失败来源和已使用的替代来源；
- 无法完成有效核验时写“未确认新增”，不得写成“没有新增”；
- 当天某一维度没有新信息时，列出已检索关键词和来源范围；
- 同一数据出现冲突时并列记录，并说明可能的统计口径差异；
- 不自动补写缺失数值，不根据新闻措辞反推投资额、市场份额或订单规模。

## Wiki 边界

自动任务只写入 `sources/automations/商业航天每日跟踪/` 和必要的 `log.md` 记录，不直接改写正式 `wiki/` 页面、`index.md` 或 `.manifest.json`。

重大技术节点、组网里程碑或产业竞争格局变化经过人工复核后，再整理进入现有的：

- `wiki/timelines/冰冰小美-商业航天投资跟踪.md`
- `wiki/timelines/冰冰小美对太空经济产业的追踪.md`
- `wiki/reasoning/冰冰小美-商业航天如何复制碳中和产业投资路径.md`

## 验收标准

首次实施完成后应验证：

1. 项目提示词文件存在且完整；
2. 自动任务处于启用状态，执行时间为每天北京时间 `09:45`；
3. 自动任务读取项目提示词而不是内嵌完整规则；
4. 手动试运行能生成目标日期日报和滚动看板；
5. 四个跟踪维度均有结果或明确的“未确认新增”说明；
6. 事实、推导和不确定性已分离；
7. 未修改无关文件，也未覆盖用户已有变更。
