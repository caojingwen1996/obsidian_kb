# A 股大盘面板本地代理修复设计

## 1. 问题与目标

当前单文件 HTML 直接从浏览器访问东方财富、中证指数官网和金十公开接口。截图已经证明主要失败发生在浏览器网络边界：东方财富 JSONP 脚本加载失败，中证指数 Fetch 被跨域或网络策略拒绝，东方财富全市场响应未通过完整性校验。结果是位置、估值和情绪层大量指标退出评分，覆盖率低于 70%。

本次修复的目标是保留现有透明评分、单文件 artifact、缓存和示例模式，同时提供一个可双击启动的本地同源代理。页面通过 `http://127.0.0.1` 打开后，只向同源 `/api` 请求数据；本地代理再访问第三方公开接口，从根因上移除浏览器 CORS 和 JSONP 限制。

## 2. 交付方式

新增 `启动A股大盘面板.cmd`。用户双击后：

1. 启动器在系统 PATH 中寻找 `python`，找不到时尝试 Windows `py -3`；
2. Python 服务绑定到 `127.0.0.1`，不监听局域网地址；
3. 服务选择可用本地端口并用默认浏览器打开面板；
4. 页面识别本地代理环境，自动把行情、估值、国债、市场宽度和融资数据请求切换到同源 `/api`；
5. 关闭命令窗口后服务停止，页面不再刷新实时数据，但已写入的浏览器快照仍可用于展示。

原有 `a-share-market-dashboard.html` 继续作为独立 artifact，可直接双击查看示例或已有缓存。稳定联网刷新以启动器方式为准。

## 3. 本地代理边界

代理使用 Python 3 标准库，不增加 pip 依赖。只允许以下固定路由：

| 本地路由 | 上游数据 |
|---|---|
| `/api/eastmoney-kline` | 东方财富指数历史行情 |
| `/api/csindex-performance` | 中证指数点位、TTM PE、成交额历史 |
| `/api/treasury` | 中国 10 年期国债收益率 |
| `/api/market` | 沪深 A 股市场宽度、涨跌停和成交额 |
| `/api/margin` | 沪深融资余额汇总 |

安全约束：

- 不提供任意 URL 转发能力；上游主机、路径和参数由服务端固定构造。
- `secid`、`indexCode`、日期、历史条数和融资市场编号均使用白名单或格式校验。
- 上游请求设置超时、浏览器型 User-Agent、Referer 和响应大小上限。
- 上游非 2xx、超时、无效 JSON 或超限响应转换为结构化错误和相应 HTTP 状态，不填充虚假数据。
- 静态文件仅提供面板成品，不暴露仓库其他文件。

## 4. 前端数据流

适配器新增可测试的本地代理检测和 URL 构造：

```text
页面位于 127.0.0.1 / localhost
  → fetch 同源 /api
  → 复用现有 parseEastmoneyKlines / parseCsindexPerformance 等解析器

页面位于 file:// 或其他来源
  → 保留现有 JSONP / Fetch
  → 失败时继续使用快照、缺失项退出评分和示例模式
```

本次不修改评分公式、权重、70% 覆盖门槛或缺失处理逻辑。前瞻 PE 仍保持明确缺失，因为本地代理只能解决访问技术问题，不能创造未授权的数据源。

## 5. 错误与状态呈现

- 本地代理可用时，数据来源名称追加“本地代理”，便于审计。
- 页面启动于 `file://` 时，顶部提示建议使用启动器获得稳定联网数据。
- 本地代理失败时，现有数据域隔离继续生效：单个来源失败不能清空其他成功数据。
- 服务端错误正文只返回路由、上游类别和安全化错误信息，不返回本机路径或调用栈。

## 7. 测试与验收

1. Python 单元测试证明代理拒绝未知路由和非法参数，只构造允许的上游 URL。
2. Python 单元测试以固定上游响应验证 JSON、JSONP、超时和错误状态处理。
3. Node 测试先失败再实现，证明 `localhost` 页面使用 `/api`，`file://` 页面继续使用旧传输。
4. 启动本地服务后，健康检查和面板静态文件返回 200。
5. 在实际网络允许时，逐个本地 `/api` 路由进行烟雾检查；第三方不可达必须记录为外部失败，不能把测试写成固定依赖公网成功。
6. 重新构建单文件 HTML，运行现有 34 项 Node 测试和既有风险面板回归测试。
7. 页面通过启动器打开后，不再出现浏览器 CORS 或 JSONP 错误；主要数据源可用时覆盖率应达到至少 70%。

## 8. 明确限制

本地代理解决浏览器跨域、JSONP 和响应访问限制，但不能保证第三方免费接口永久可用。若上游主机在用户网络中完全不可达，或接口字段再次变更，面板仍会按原规则显示快照、缺失和审计错误。前瞻 PE 仍需要未来单独接入合法、稳定的数据源。
