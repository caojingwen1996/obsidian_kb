# 航天电子每日跟踪面板与盘中行情设计

## 已确认需求

- 将研报头部的静态结论、当前价格、估值区间、动作和置信度卡片移入“每日跟踪面板”。
- 每日跟踪面板继续以每日同步数据为正式决策快照。
- 面板新增独立“盘中实时”位置；通过大盘本地服务打开时定时刷新，通过 `file://` 直接打开时明确显示“需通过大盘启动器连接”，不伪装成实时行情。

## 方案比较

### 方案 A：每日快照 + 本地代理盘中增强（采用）

每日值直接写入 HTML，离线可用；本地代理只允许查询航天电子并返回行情 JSON，页面每 60 秒刷新盘中卡片。接口失败时保留每日快照并显示错误状态。该方案兼顾稳定性和盘中信息。

### 方案 B：HTML 直接访问第三方行情接口

代码较少，但 `file://` 容易受跨域、HTTPS 和第三方接口策略影响，不采用。

### 方案 C：只预留盘中位置，不接数据

最稳定，但不满足“增加盘中实时”的实际用途，不采用。

## 页面结构

- 研报头部只保留标题、报告日期和证券身份，不再显示四张静态 KPI 卡和静态 verdict 胶囊。
- 每日跟踪面板使用两行结构：
  1. 决策快照：基本面状态、动态价值区间、股价偏离度；
  2. 行情与动作：每日同步价、盘中实时、动作与置信度。
- 每日同步价明确标注快照时间；盘中实时卡显示价格、涨跌幅、行情时间和连接状态。

## 数据流与边界

1. 页面加载时先显示写入 HTML 的每日同步值。
2. 若协议为 `http:` 或 `https:`，请求 `/api/stock-quote?secid=1.600879`。
3. 本地代理仅放行 `1.600879`，向东方财富固定行情接口请求指定字段。
4. 返回值必须包含证券代码、名称、价格、昨收、涨跌幅和行情时间；字段缺失或非数值时返回上游错误。
5. 成功后立即更新盘中卡，之后每 60 秒刷新；失败时显示“实时行情暂不可用”，每日同步值不变。

## 测试与验收

- 测试先证明股票行情路由不存在，再实现固定白名单 URL。
- 服务测试证明接口返回标准化 JSON，并拒绝其他证券代码。
- 页面结构测试证明头部不再含静态 KPI，动态面板含每日价、盘中实时、动作和置信度。
- 浏览器分别验证 `file://` 降级提示和本地服务实时位置。
- 原研报 16 个章节、动态区替换标记和资金报告链接保持不变。

## 自检

- 范围只覆盖航天电子，不建设通用个股行情平台。
- 每日同步值与盘中实时值在界面上明确分开。
- 网络失败不覆盖每日数据，也不显示虚假的实时状态。
