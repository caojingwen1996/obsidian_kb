# 冰冰小美风险提示明细表设计

## 目标

在现有冰冰小美风险提示页面的曲线下方增加风险明细表，用文字方式完整展示 Excel 中的风险提示记录，同时保持现有黑底白框风格、上证指数曲线和右侧空白区域不变。

## 数据来源与排序

- 复用现有 `/api/dashboard` 返回的 `risk_points`，不新增接口，也不重复读取 Excel。
- 每条记录展示三个字段：风险日期、当日累计风险提示次数、风险原因。
- 按风险日期倒序排列，最新日期在最上方。
- 表格与曲线来自同一次接口响应，避免两处数据不一致。

## 页面布局

- 表格位于左侧 `chart-panel` 内，在曲线图例下方。
- 右侧 `reserved-sidebar` 继续保持空白，不放入表格或统计信息。
- 表格区域标题为“风险提示明细”。
- 表头依次为“日期”“当日累计风险提示次数”“风险原因”。
- 日期列和次数列保持紧凑，风险原因列占用剩余宽度并允许完整换行。

## 视觉样式

- 沿用页面现有颜色变量和无圆角、细边框风格。
- 表格背景为黑色，表头使用较深灰色层次，边框和分隔线清晰可见。
- 正文使用高对比度浅色文字；次数使用红色文字并配合列标题表达含义，不仅依赖颜色。
- 日期和次数使用等宽数字特性，便于纵向比较。
- 行悬停只改变背景亮度，不移动布局。

## 响应式与可访问性

- 使用语义化 `table`、`caption`、`thead`、`tbody` 和列标题单元格。
- 桌面端直接显示三列表格。
- 窄屏时仅表格容器允许横向滚动，页面本身不产生横向溢出。
- 风险原因保持换行，不使用省略号截断。
- 无数据时使用跨三列的一行显示“暂无风险提示记录”。

## 前端行为

- 数据加载成功后，图表和表格使用同一份 `payload.risk_points` 更新。
- 渲染前复制并按 `risk_date` 倒序排序，不修改接口原始数组。
- 单元格使用 DOM `textContent` 写入，避免风险原因中的特殊字符被当作 HTML。
- 重试成功时完整替换表格内容，不累积旧行。
- 数据加载失败时清空表格并显示空状态，避免残留上一次成功数据。

## 错误与边界

- 缺少风险原因时显示“未填写原因”。
- 次数为 `0` 时正常显示 `0`。
- 多行风险原因保留换行。
- 非交易日风险仍按原始风险日期显示；表格不额外展示对齐行情日和收盘点位。

## 验证

1. 页面结构测试确认存在风险明细标题、语义化表格和三列表头。
2. 前端行为测试确认按日期倒序、三字段渲染、缺失原因回退和空状态。
3. 错误处理测试确认加载失败时清空旧表格。
4. 全量 Python 测试保持通过。
5. 浏览器桌面宽度下确认表格与左侧图表等宽、右侧保持空白。
6. 窄屏下确认页面无整体横向溢出，表格内容可访问且原因不截断。

## 范围边界

- 不修改 Excel 列结构。
- 不新增筛选、搜索、分页、排序按钮或导出功能。
- 不修改风险节点、行情获取、缓存或日期对齐逻辑。
- 不向右侧空白区域增加内容。
