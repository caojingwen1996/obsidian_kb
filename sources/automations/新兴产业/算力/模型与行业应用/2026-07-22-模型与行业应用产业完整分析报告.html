  • 产业类型： 新兴产业 / 算力需求转化层
  • 分析日期： 2026-07-22
  • 数据截止日期： 2026-07-22；财务数据采用最近可得法定披露
  • 地域范围： 中国大陆，兼顾全球模型技术与商业模式竞争
  • 分析范围： MaaS / 模型 API、训练 / 微调服务、推理服务与 Token 计费、智能体开发运行平台、行业软件 / 解决方案，以及制造、金融、医疗、科研等可验证应用
  • 排除范围： 不把全部传统软件、全部云计算、消费互联网流量或只具“AI 概念”的项目纳入；只有能解释如何消耗算力并形成持续付费需求的产品和服务进入主线
  • 核心研究问题： 模型与行业应用能否把算力资本开支转化为可持续调用、复购、ROI 和现金流；产业利润池是在基础模型、推理平台、智能体平台、行业软件还是最终应用方？
  • 主要来源： 国务院、工信部、国家网信办、交易所 / 公司法定披露、企业官方业绩资料

关键口径：

  • 模型能力是技术供给，不能替代调用量；调用量 / Token是算力消耗，不能替代订单或收入；订单是合同承诺，不能替代交付与收入确认；收入不能替代复购、ROI 或现金流；ROI必须以客户节省成本、增收、缩短周期或降低风险的可核验结果衡量。
  • MaaS / 模型 API提供标准化模型调用；训练 / 微调服务按项目、算力、数据处理或交付收费；推理服务按 Token、请求、时长或保底资源收费；智能体平台提供编排、工具调用、评测、治理和运行；行业软件 / 解决方案把模型嵌入工作流并对业务结果负责。五者客户、交付责任和毛利机制不同。
  • Token 是模型输入输出的计量单位。全国日均词元调用量突破 140 万亿，证明推理需求快速增长，但公开口径未同时给出付费 Token 占比、平均单价、缓存 / 免费调用和毛利，不能直接换算产业收入。中国网信网转人民日报

1. 执行摘要#

1.1 一句话结论#

中国模型与行业应用已从“模型发布与试用”进入“推理调用扩张、行业工作流改造与商业验证并行”的产业中期：Token 使用量、备案服务和制造业普及率证明需求已启动，MaaS / AI 云和少数开放平台已进入收入兑现，但智能体、训练微调及多数行业方案仍以部署、项目和用户数为主；产业第一约束已经从模型是否可用转向高价值场景的准确性、数据与系统集成、责任治理和可持续 ROI，利润池将从同质化基础模型逐步迁移到高利用率推理平台、智能体运行层及拥有数据、渠道和工作流控制权的行业软件。

2025 年中国人工智能核心产业规模超过 1.2 万亿元、企业超过 6200 家；截至年末生成式 AI 用户 6.02 亿，累计 748 款生成式人工智能服务备案、435 款调用已备案模型的应用 / 功能完成登记。工信部、中国网信网 这些数据说明供给与使用广泛扩张，但不能证明每个模型或应用已有可持续商业回报。

1.3 核心结论#

  1. 长期方向： 算力需求从训练资本开支转向持续推理和智能体执行，应用决定算力的最终利用率。
  2. 当前阶段： 通用模型供给充分，MaaS 和推理调用快速扩张，行业应用进入从试点到复制的转换期。
  3. 供需矛盾： 模型多、应用也多，但能持续付费且可复购的高价值工作流仍少。
  4. 利润传导： AI 云 / MaaS 最先看到调用收入；行业软件需经过集成、交付、验收；最终由续费和现金流验证。
  5. 第一约束： 数据质量、准确性、系统集成、安全责任与业务 ROI 的共同约束。
  6. 主要增量： 企业 Token 调用、低成本微调、智能体执行、办公订阅、制造 / 金融 / 医疗 / 科研工作流改造。
  7. 最受益环节： 有高利用率推理平台、行业数据 / 客户渠道、标准产品和可度量 ROI 的主体。
  8. 关键反证： Token 增长但收入 / 毛利不增；项目数增长但复购 / 回款不增；效率案例无法跨客户复制。

4.2 核心约束排序#

  1. 可验证 ROI： 没有业务结果，调用只是成本而不是生产力。
  2. 数据、权限和集成： 模型必须接入真实知识与业务系统才能执行任务。
  3. 准确性、安全和责任： 高风险行业需确定人机边界、审计与失败恢复。
  4. 单位任务经济性： Token 便宜不等于任务便宜，Agent 可能反复调用。
  5. 标准化复制： 单一客户项目无法形成软件规模效应。

第一约束已经从“模型有没有能力”转为“能否在受治理的工作流中稳定产生高于总拥有成本的业务结果”。

5.5 周期综合结论#

供给侧已从“缺模型”转为“模型充足且持续降价”，需求侧从用户增长进入企业调用与工作流验证。景气已传到部分云与 MaaS 收入，但利润和现金流尚未普遍传到智能体及行业应用。下一阶段不是模型数量继续增加，而是付费 Token、AI 订阅、复购和客户 ROI 同步增长。

6.2 集中度与成本曲线#

  • 模型层： 训练与推理固定成本高，开源降低模型租金，分发和生态促使份额向头部集中。
  • 推理层： 利用率、批处理、缓存、模型压缩和异构调度决定单位 Token 成本；价格下降不必然降低利润，前提是成本下降更快。
  • 平台层： Agent 创建工具易同质化，长期壁垒在运行可靠性、工具生态、权限与评测数据。
  • 软件层： 存量客户、数据和工作流形成迁移成本；但大量定制会使“软件”退化为低毛利服务。
  • 应用方： 最大超额收益可能留在采用 AI 提升效率的制造、金融、医疗企业，而非全部支付给软件供应商。

7. 综合产业判断#

7.1 当前最重要的产业矛盾#

模型供给和 Token 调用已经快速扩张，但企业愿意长期付费的是可审计、可集成、可负责并能产生业务结果的完整工作流。产业核心矛盾是“低成本智能供给”与“高成本场景落地”之间的错位。

模型微调与私有化 数据 → 微调 → 部署 → 运维 平台、集成、数据服务 模板化复购 一次性定制
智能体执行 对话 → 工具调用 → 业务结果 Agent 平台、软件入口 成功任务、低失败、扩席 创建多、使用少
办公 AI 增购 高频用户 → 席位 / 用量付费 办公软件 ARPU、净留存提高 AI 变免费标配
制造 AI 质检 / 工艺 / 运维 / 排产 工业软件与解决方案 良率、人效、停机改善且复制 单一灯塔案例
金融 AI 研发、投研、客服、风控 金融 IT 合规、准确、续约 只能辅助、无新增预算
医疗 AI 临床辅助、病历、科研 医疗 AI / 数据平台 临床验证、付费方清晰 责任与采购周期
科学智能 文献、材料、药物、实验 科研平台 / 专业软件 研发周期和成功率改善 论文成果无商业收入

7.3 最值得跟踪的产业环节#

  1. MaaS / 推理平台： 最直接把 Token 转成算力收入，跟踪付费量、单价、单位成本和毛利。
  2. 智能体运行与治理： 不跟踪创建数，跟踪成功任务、工具调用、留存、SLA 和付费 ARR。
  3. 通用办公 AI： 最可能凭高频入口形成低交付订阅增购，需单列 AI 付费和续费。
  4. 制造行业软件： ROI 较易用良率、人效、能耗、停机时间量化，观察跨工厂复制。
  5. 金融行业软件： 客户预算和数据价值高，但强监管；观察从辅助到正式流程的权限变化。
  6. 医疗 AI： 社会价值与壁垒高，商业化受临床责任、支付方和长采购周期约束。
  7. 科学智能： 潜在价值大但验证周期长，应以研发效率和商业合同而非模型榜单判断。

7.4 情景分析#

情景 假设 产业结果 受益 承压
乐观 推理成本快速下降，Agent 成功率提升，企业形成经常性 AI 预算 Token、订阅、ROI 和现金流共振 高利用率 MaaS、办公 / 行业软件龙头 单一模型和纯项目公司
基准 调用高增、单价下降，少数场景实现复购 云 / MaaS 收入增长，应用层分化 有入口、数据和客户的软件 无独立收入披露的概念主体
悲观 模型同质化、Agent 失败、预算收缩、合规成本上升 Token 增长不盈利，项目回款恶化 低成本平台和刚需软件 高烧钱模型、非标集成、弱现金流公司

下修信号： API 单价快于成本下降、免费调用占比高、应收增长快于收入、案例长期停留试点、客户取消或不续费。

7.5 最终结论#

模型与行业应用是算力产业的最终需求发动机，但“消耗算力”与“创造经济价值”之间仍隔着数据、集成、治理、组织与 ROI。2026 年最确定的产业变化是推理调用进入规模扩张，最清晰的商业证据来自头部云 / MaaS 和少数开放平台；最具长期利润潜力的是占据客户工作流、数据和分发入口的智能体与行业软件；最需要警惕的是用模型榜单、Agent 创建数、备案量或用户数替代付费、复购和现金流。产业整体处中期验证，不是所有 AI 软件的无边界普涨阶段。

模型能力
→ 活跃调用 / Token
→ 付费订单
→ 交付与收入确认
→ 复购 / 扩席 / 留存
→ 客户 ROI
→ 毛利与经营现金流

只有这条链条至少推进到“复购 + ROI”，才可判断应用形成持续算力需求；推进到“毛利 + 现金流”，才可判断利润池真正形成。

通用风险与证据附录

风险与反证#

  1. 模型能力商品化、API 价格下降快于推理成本。
  2. Token 调用包含免费、缓存、内部或低价值流量，付费率不足。
  3. 智能体任务失败、权限越界或不可恢复，无法进入核心流程。
  4. 行业数据质量、权属、隐私和更新成本高于预期。
  5. 项目制集成和销售费用吞噬软件毛利。
  6. 客户有试点预算但无经常性预算，复购不足。
  7. 医疗、金融等高风险行业责任和监管限制自动执行。
  8. 内容标识、版权、数据安全和模型合规增加成本。
  9. 巨头捆绑云、模型和应用，挤压独立平台利润。
  10. 资本重复投入基础模型，应用、评测与交付资源不足。
