# 中国运载力产业完整分析报告

## 0. 基本信息

- **产业名称：** 中国运载力产业
- **产业类型：** 新兴产业 / 战略性数字基础设施
- **分析日期：** 2026-07-22
- **数据截止日期：** 2026-07-22；企业经营数据采用最近可得法定披露
- **地域范围：** 中国大陆，兼顾全球互联协议与器件供应链
- **分析范围：** 交换芯片 / SerDes、NIC / 智能网卡 / DPU、Scale-up 交换与连接 / 光电互联、Scale-out 以太网 / RoCE 交换设备、高速光模块 / 光器件、DCI / OTN / 骨干通道、网络遥测、路径 / 带宽编排与 DCI / OTN SLA；完整超节点、整机柜、集群部署与收入归“计算系统与集群”，任务放置、资源池调度、交易结算与平台收入归“资源运营与调度”
- **核心研究问题：** 运载力能否把分散的芯片、服务器、机架、集群和数据中心连接成“有效算力”；价值量与利润池分别落在芯片、器件、设备、网络资产还是平台软件？
- **主要数据来源：** 工业和信息化部、交易所及公司法定披露、运营商年报、企业官方技术资料

**关键口径与边界：**

- **运载力**不是单纯端口带宽，而是计算任务和数据在给定时间窗口内被接入、传输和交换，并获得可验证网络服务质量的能力。核心指标包括有效吞吐、端到端时延、丢包 / 重传、拥塞完成时间、可用性、能耗、路径可编排范围和跨域可达性；资源发现和任务放置不属于本报告收入边界。
- **Scale-up**互联位于超节点内部，服务张量 / 专家并行，追求纳秒级低时延、内存语义和高一致性；本报告只分析其交换芯片、SerDes、连接器 / 光电互联、协议与 NOS，不分析完整超节点、整机柜、集群部署或收入。**Scale-out**连接服务器、机架或集群，主要采用 InfiniBand 或以太网 / RoCE；**DCI / 骨干网**连接数据中心或地域，关注毫秒级时延、OTN 承载、路径可用性和 SLA。三者不能合并估值或比较端口价格。[中兴通讯对三域边界的说明](https://www.zte.com.cn/china/about/magazine/zte-technologies/2026/4/3/4.html)
- **芯片**向设备厂商出售 IC / IP，利润取决于研发摊销、授权和出货规模；**光模块 / 光器件**向云厂商或设备厂商出售收发器件，利润取决于速率代际、良率和客户结构；**交换设备**出售整机、网络操作系统和方案；**运营商网络**以重资产建设通道，按带宽、专线、云网套餐或 SLA 收费；**网络控制软件**提供遥测、路径和带宽编排。任务调度、资源交易和平台收入仅作外部需求旁证，不计入运载力收入。

## 1. 执行摘要

### 1.1 一句话结论

> 中国运载力正从“提高端口速率”进入“超节点、集群网络与跨域算力网分层升级”的产业中期：短期利润释放最强的是 800G / 1.6T 高速光模块，交换设备随 AI 集群订单增长但毛利受竞争和系统集成约束，交换芯片 / SerDes 与 DPU 仍处国产验证和研发摊销阶段，运营商 DCI / OTN 则由资本开支转化为较慢的带宽与云网服务回报；产业第一约束已从裸带宽转向端到端有效通信效率与开放互操作。

工信部计划到 2026 年形成较完备的算力互联互通标准、标识和规则体系，到 2028 年实现公共算力资源标准化互联，并明确节点内部高性能协议、网络直联、统一标识、算力感知和算力调度等层次。[《算力互联互通行动计划》](https://fjca.miit.gov.cn/zwgk/zcwj/wjfb/art/2025/art_25eea57dd6f840e680184fffb086883d.html) 对运载力而言，这意味着网络需要提供可观测、可选路、可保障 SLA 的跨域承载；政策中的资源发现、任务调度和交易结算属于“资源运营与调度”，只作为网络需求来源，不计入本产业收入。

### 1.3 核心结论

1. **长期方向：** 计算从单机走向超节点、万卡集群与跨域资源池，网络由附属连接升级为决定算力效率的系统组件。
2. **当前阶段：** Scale-out 以太网 / RoCE 与 800G 光互联处规模扩张，Scale-up 互联多路线竞争，网络遥测和路径 / 带宽编排处标准与服务验证期。
3. **当前供需周期：** AI 集群拉动高速交换机和光模块；运营商骨干、OTN 与专线稳步升级，节奏慢于云侧。
4. **利润传导：** 高速光模块最先兑现；设备订单随后；芯片和 DPU 需经历研发、认证、规模出货；网络服务最后依靠利用率兑现。
5. **第一约束：** 协议、拥塞控制、拓扑、光电链路和网络编排协同后的“任务完成时间”，不是单点最高速率。
6. **主要增量：** Scale-up 交换与光电互联、AI 后端网络、800G / 1.6T 光模块、数据中心互联、400/800G 骨干及跨域路径 / 带宽编排。
7. **最受益环节：** 已完成头部客户认证的高速光模块、具备交换芯片 / NOS 协同能力的设备商，以及可规模交付 DCI / OTN、网络遥测和 SLA 服务的主体。
8. **关键反证：** GPU 集群网络利用率不升、尾时延不降、互操作迟迟不成熟，或客户资本开支未转化为厂商回款。

**结论：**

> 运载力的战略变化是从“建设一张更快的网”转为“建立三层互联和一层网络控制”：节点内 Scale-up 互联、集群 Scale-out 网络、跨域承载，加上统一网络遥测和路径 / 带宽编排。完整超节点与计算任务调度分别由相邻子产业承接。

### 2.2 产业属性

| 产业属性 | 是否符合 | 核心逻辑 |
|---|---|---|
| 战略资源 | 是 | 决定异构算力是否可汇聚、可调用 |
| 支柱产业 | 是 | 连接芯片、服务器、云、运营商与 AI 应用 |
| 新兴产业 | 是 | Scale-up 互联、无损以太网、1.6T 光互联与网络编排仍快速演进 |

- **主要属性：** 战略性数字基础设施。
- **次要属性：** 高景气通信器件与新兴算网平台的组合产业。
- **跟踪原则：** 必须按产品 / 设备 / 服务分层，不用同一个“算力互联收入”口径覆盖全部公司。

```text
政策标准与网络接口
→ 计算节点和网络通道接入
→ 跨主体网络可达、选路与带宽编排
→ 真实流量、时延与 SLA 验证
→ 网络利用率、专线和云网服务收入
```

- **确定性较高：** 骨干、OTN、数据中心交换与光互联设备升级。
- **确定性中等：** 跨主体动态路径和带宽 SLA 形成可持续网络服务；仍需观察真实流量、定价和责任边界。
- **政策—收入传导边界：** 节点上线和带宽建成不等于有效流量或网络服务收入；资源目录、任务调度、交易结算和平台收入归“资源运营与调度”。

**边界说明：**

1. SerDes 是高速串行收发电路 / IP，不等于完整交换芯片；交换芯片也不等于交换机整机。
2. NIC 负责主机接入，DPU 进一步卸载虚拟交换、存储、安全和管理任务；具备 RoCE 功能的网卡不能自动定义为 DPU。
3. Scale-up 与 Scale-out 的协议、容错域、拓扑和客户不同。前者常随 AI 芯片 / 超节点成套，但完整超节点、整机、集群部署与收入归“计算系统与集群”；本报告只保留互联部件、协议和 NOS 能力旁证。后者更接近标准数据中心网络。
4. 光模块完成电—光—电转换，交换机完成报文转发与拥塞控制，OTN / 骨干网提供跨域承载，三者虽共同构成链路但收入不能重复计算。
5. 本报告中的编排仅指网络遥测、选路和带宽控制，并以真实流量、SLA 和网络服务收费验证；任务放置、资源池调度、统一计量、交易结算与平台收入归“资源运营与调度”，不得在此重复计算。

### 3.2 价值量与利润池

**价值量**指某环节在一套系统或一轮资本开支中的采购金额；**利润池**指收入扣除成本、研发、销售和资本成本后可持续留存的利润。高价值量不必然形成高利润池。

| 环节 | 价值量驱动 | 毛利 / 回报机制 | 当前利润位置 | 主要验证指标 |
|---|---|---|---|---|
| 交换芯片 / SerDes | 端口数、速率、交换容量、先进制程 | IP 与软件生态、高研发摊销 | 国产投入 / 规模爬坡期 | 高端端口出货、毛利、研发费用率、客户二供 |
| NIC / DPU | 每服务器 / 加速器连接数、卸载功能 | 芯片 + 板卡 + SDK，生态粘性 | 验证期 | 规模部署、兼容矩阵、卸载后 CPU / GPU 利用率 |
| Scale-up 交换与光电互联 | 端口 / 链路数量、SerDes 速率、拓扑密度 | 多随整机成套，协议与互联生态锁定能力强 | 早期扩张 | 独立部件订单、链路良率、互操作和通信加速比；不使用完整超节点收入 |
| Scale-out 交换设备 | 集群规模、叶脊层数、冗余与端口速率 | 整机毛利较芯片低，靠 NOS / 方案和服务 | 订单扩张 | 数据中心收入、毛利、应收、验收周期 |
| 高速光模块 / 器件 | 光口数、速率、单模 / 多模、距离 | 良率、代际先发、客户认证和规模制造 | 利润释放期 | 800G / 1.6T 占比、毛利、存货、客户集中 |
| DCI / OTN / 骨干 | 线路、带宽、设备、机房和维护 | 重资产折旧，靠利用率 / SLA / 长期合同 | 建设—运营期 | 端口利用率、专线收入、折旧、ROIC |

**当前利润池排序（仅产业判断）：**

1. 已获头部客户认证并进入 800G / 1.6T 放量的光模块。
2. 有规模份额、完整 NOS 和 AI 网络方案的数据中心交换设备。
3. 掌握高端交换 ASIC / SerDes 且能摊薄研发的芯片平台；国产主体尚未整体进入成熟利润期。
4. 高利用率 DCI / OTN 和云网专线服务；低利用率通道可能价值量高但回报低。
5. 网络遥测与路径 / 带宽编排软件；长期可形成运维和云网服务利润池，当前独立收入证据最弱。任务调度和交易平台不在本报告利润池内。

| 供应与客户集中 | 光模块和高端设备依赖少数客户 / 芯片 | 光模块、设备 | 中强 | 客户结构分散、二供成熟 |

### 4.2 核心约束排序

1. **端到端有效通信效率：** 单端口速率提升若不能降低集合通信时间、尾时延和重传，就不能转化为算力产出。
2. **开放互操作：** Scale-up 多协议、跨域多主体使设备兼容和责任边界成为扩张前提。
3. **高端芯片与光电能力：** 交换 ASIC、SerDes、DPU、光芯片和封装决定硬件天花板。
4. **网络运营闭环：** 统一网络接口、遥测、路径、带宽计费和 SLA 责任缺一不可；资源计量、任务调度与交易结算由“资源运营与调度”负责。
5. **资本回报：** 重资产通道必须由可持续流量、价格和 SLA 收入覆盖折旧。

**第一约束判断：**

> 2026 年第一约束不是“有没有 800G 端口”，而是芯片、NIC、交换、光链路、协议与网络路径控制能否共同提供稳定的有效吞吐；跨域场景还受制于网络接口、可达性、带宽计费和 SLA 责任。任务放置与交易结算是相邻产业约束，不计为运载力自身利润池。

## 6. 竞争格局与资本作用

### 6.1 主要参与者

| 环节 | 全球 / 国内格局 | 核心壁垒 | 国内位置 | 格局变化方向 |
|---|---|---|---|---|
| 交换芯片 / SerDes | 全球头部集中，国内追赶 | 架构、制程、SerDes、SDK、客户认证 | 中低端到高端逐步导入 | 国产二供和整机协同 |
| NIC / DPU | 芯片平台与软件生态主导 | 可编程核、I/O、虚拟化、SDK | 多主体产品验证 | 与服务器、云平台深度绑定 |
| Scale-up 交换与光电互联 | 专有方案与开放联盟并存 | 协议、内存语义、拓扑、SerDes 与光电链路 | 华为、中兴等推动自有路线 | 多路线共存后可能收敛；完整超节点竞争归相邻产业 |
| Scale-out 设备 | 国内设备商份额和交付能力较强 | NOS、拥塞控制、遥测、服务 | 华为 / 新华三 / 锐捷 / 中兴竞争 | AI 后端网络提高技术门槛 |
| 高速光模块 | 中国厂商全球竞争力较强 | 光电设计、良率、自动化、客户认证 | 龙头已进入利润释放 | 1.6T、LPO / CPO 带来重排 |
| DCI / OTN | 设备商与运营商双层集中 | 光系统、控制面、线路资源和运维 | 产业链较完整 | 从带宽建设转向云网协同 |
| 网络遥测与路径 / 带宽编排 | 运营商与设备商多控制域并行 | 拓扑可见性、路径算法、接口、SLA 和故障责任 | 早期 | 网络接口与 SLA 标准收敛；资源调度平台不计入 |

### 6.2 集中度与成本曲线

- **芯片层：** 固定研发和流片成本高，规模越大、生态越广，单位成本越低；高端 SerDes 和软件栈形成显著进入壁垒。
- **光模块层：** 代际早期由技术、良率和认证形成超额利润，成熟后价格下降；成本曲线取决于光芯片采购、封装自动化和良率。
- **设备层：** 芯片采购、硬件 BOM 与招标价格限制毛利，NOS、遥测、系统集成和服务可提高粘性，但不应把方案能力直接视作软件高毛利收入。
- **网络层：** 天然重资产和规模经济；固定成本由更多流量摊薄，但低价竞争可能使利用率提高而 ROIC 不升。
- **网络控制层：** 软件价值取决于遥测覆盖、跨域路径可用、带宽利用率和 SLA 责任闭环；资源目录、计量、交易结算及平台收入归相邻产业，不能借用其规模证明网络软件利润。

### 6.3 资本市场作用

| 资本作用 | 正向机制 | 可能扭曲 | 验证方式 |
|---|---|---|---|
| 支持芯片研发 | 覆盖长期研发、流片和生态投入 | 以技术路线叙事掩盖量产差距 | 高端收入、毛利、客户复购 |
| 支持光模块扩产 | 提升 800G / 1.6T 产能与自动化 | 扩产快于客户需求，形成库存 | 产能利用率、存货周转、现金流 |
| 支持设备周转 | 承担大型项目备货和应收 | 订单增长但回款恶化 | 合同负债、应收、经营现金流 |
| 支持运营商网络 | 降低长期基础设施融资成本 | 重复建设和低利用率 | 端口利用率、折旧、ROIC |
| 支持网络控制试点 | 建设遥测接口、路径控制和跨域网络节点 | 只追求接入数量，无真实流量和 SLA 闭环 | 路径成功率、带宽利用率、SLA 达标率、网络服务收入 |

**资本纪律：**

> 运载力投资应遵循“能力—订单—收入—现金流—利用率”五级证据链。产品发布只证明能力，合作协议只证明潜在订单，网络建成只证明供给；只有客户验收、回款和持续使用才证明利润池。

## 7. 综合产业判断

### 7.1 当前最重要的产业矛盾

> AI 集群和全国算力网要求网络从“尽力而为”升级为可验证 SLA 的确定性服务，但产业仍被分割在芯片、NIC、交换机、光链路、网络控制软件和运营商多个控制域；标称带宽快速提升，端到端有效吞吐、互操作、真实流量和网络服务付费却未同步成熟。

这一矛盾表现为三组错位：

1. **速率与效率错位：** 800G / 1.6T 端口并不自动消除拥塞、尾时延和集合通信等待。
2. **建设与利用错位：** 骨干、专线和节点可先建成，真实跨域任务和付费流量后置。
3. **产品与利润错位：** DPU、Scale-up 互联方案和网络控制软件可快速发布，但独立收入、毛利和现金流需要更长验证；完整超节点与任务调度平台不属于本报告收入边界。

### 7.2 主要增量来源

| 增量来源 | 直接受益 | 二阶受益 | 持续条件 | 反证 |
|---|---|---|---|---|
| 超节点卡数和带宽密度提升 | Scale-up 交换芯片、SerDes、连接器和光电互联 | 完整整机、冷却、电源属于相邻产业 | 模型并行效率改善且互联部件形成独立订单证据 | 软件无法利用、路线封闭；不得按完整超节点收入计量 |
| AI 集群从 400G 升级 800G / 1.6T | 光模块、交换设备 | 光器件、交换芯片、NIC | 客户资本开支与集群利用率 | 订单递延、降价超预期 |
| RoCE 规模部署 | NIC / DPU、交换机、遥测 | 网络软件和运维 | 尾时延与稳定性可控 | 运维成本抵消硬件节省 |
| 400/800G 骨干与 DCI | OTN、光模块、设备商 | 运营商专线和云网 | 通道利用率和 SLA 收入 | 重复建设、价格下降 |
| 全国算力网络互联 | DCI / OTN、网络接口、遥测和路径 / 带宽编排 | 专线、云网与 SLA 服务 | 形成真实流量、可核验 SLA 和网络服务付费 | 只有节点接入，没有流量与 SLA；任务调度和交易结算另行验证 |

### 7.3 最值得跟踪的产业环节

1. **高速光模块 / 光器件：** 当前业绩确定性最高，但需跟踪 1.6T 良率、客户集中、降价和扩产现金流。
2. **数据中心交换设备：** 观察 AI 后端网络收入、毛利与应收，而不是只看市场排名或产品速率。
3. **交换芯片 / SerDes：** 战略价值高，关注高端产品真实出货、客户二供和研发摊薄。
4. **NIC / DPU：** 用量产服务器平台、活跃部署和卸载收益验证，不以合作签约替代订单。
5. **Scale-up 互联：** 关注协议生态、交换芯片 / SerDes、连接器、跨机架光电方案及独立部件订单，不用完整超节点部署量或收入替代互联证据。
6. **DCI / OTN / 骨干：** 关注带宽利用率、专线 / 云网收入和 ROIC，而不只看资本开支。
7. **网络遥测与路径 / 带宽编排：** 关注真实流量、路径成功率、带宽利用率、SLA 与网络服务收入；任务调度成功率、交易额和结算归“资源运营与调度”。

### 7.4 情景分析

| 情景 | 关键假设 | 产业结果 | 受益环节 | 主要受损 / 承压 |
|---|---|---|---|---|
| 乐观 | AI 资本开支持续，1.6T 顺利量产，开放以太网与跨域网络标准成熟 | 运载力从硬件升级走向可编程网络与 SLA 服务，利用率和利润同步 | 光模块、交换芯片 / 设备、DPU、网络控制与云网服务 | 低速旧代际、封闭小生态 |
| 基准 | 云侧保持增长，Scale-up 互联多路线并存，跨域网络编排渐进 | 光模块先兑现，设备稳增，网络软件和 SLA 服务缓慢商业化 | 高纯度光互联和头部设备商 | 纯主题、收入不透明主体 |
| 悲观 | AI 资本开支放缓、客户去库存、协议割裂、网络利用率低 | 端口降价、扩产过剩、设备验收和网络服务收入推迟 | 高现金流、低成本、经常性网络服务 | 高估值扩产、单一客户、重资产低利用率 |

**转换信号：**

- 乐观上修：1.6T 规模交付、AI 交换收入与毛利同步、国产高端交换芯片放量、跨运营商网络路径与 SLA 服务形成付费闭环。
- 悲观下修：头部客户资本开支下调、光模块库存与应收显著快于收入、交换设备降价、跨域节点只有接入无真实流量和网络服务收入。

### 7.5 最终结论

> 运载力是算力从“设备资产”变成“可连接生产能力”的网络转换层。2026 年产业最清晰的景气来自 AI 集群内部的高速光互联和交换升级，最重要的国产化长期变量来自交换芯片 / SerDes、NIC / DPU 与 Scale-up 互联协议，最具基础设施属性的增量来自 DCI / OTN / 400—800G 骨干，最具软件期权但证据最弱的是网络遥测和路径 / 带宽编排。完整超节点、任务放置、资源池调度、交易结算与平台收入均归相邻子产业；本报告只保留它们对网络的需求旁证，不计收入。

**可执行跟踪框架：**

```text
需求：AI 集群卡数 / 运营商与云资本开支
→ 产品：交换芯片、NIC、交换机、光模块、OTN 代际
→ 效率：有效吞吐、尾时延、拥塞、功耗
→ 商业：订单、验收、收入、毛利、现金流
→ 运营：端口利用率、真实流量、路径成功率、SLA、网络服务收入
→ 反证：库存 / 应收上升、利用率下降、协议割裂、付费缺失
```

# 通用风险与证据附录

## 风险与反证

1. **AI 资本开支回落：** 云厂商削减集群建设会首先影响交换机和高速光模块订单。
2. **技术代际风险：** 800G、1.6T、LPO、CPO、OCS 路线切换可能造成设备与库存减值。
3. **协议生态风险：** Scale-up 多路线长期割裂，可能降低规模经济和二供能力。
4. **国产化验证风险：** 产品发布、实验室性能和合作协议不等于大规模商用或盈利。
5. **客户集中风险：** 高速光模块和定制设备对少数海外 / 云客户依赖较高。
6. **供应链风险：** 先进制程、EDA、高速 SerDes IP、DSP / 光芯片和关键材料可能受限。
7. **价格竞争风险：** 交换设备招标和光模块成熟代际降价可能使收入增长不转化为利润。
8. **网络利用率风险：** DCI / OTN / 骨干和地方算力网络若缺少真实流量，折旧压低回报。
9. **网络软件商业化风险：** 遥测和路径编排若未形成真实流量、SLA 与网络服务收费，只能证明能力或政策价值；资源交易平台商业化不在本报告收入边界。
10. **指标误用风险：** 端口带宽、交换容量、完整超节点部署数和资源平台接入算力规模均不能直接替代有效吞吐或运载力收入。
